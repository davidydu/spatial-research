# Chisel/RTL/HLS Immediate-Break Behavior

## Source Contract

Spatial's source-level early-exit hook is controller-local `breakWhen`, stored in the IR as `stopWhen`. `Pipe.apply(breakWhen)` stages a `UnitPipe` with `stopWhen = Some(breakWhen)`, while `Stream.apply(breakWhen)` and `Sequential.apply(breakWhen)` have explicit comments saying the controller will "break immediately" when the register is true and reset the register when the controller finishes (`src/spatial/lang/control/Control.scala:27`, `src/spatial/lang/control/Control.scala:41`, `src/spatial/lang/control/Control.scala:53`). The same field is carried by `UnitPipe`, `OpForeach`, `OpReduce`, `OpMemReduce`, `UnrolledForeach`, and `UnrolledReduce` (`src/spatial/node/Control.scala:33`, `src/spatial/node/Control.scala:45`, `src/spatial/node/Control.scala:57`, `src/spatial/node/Control.scala:78`, `src/spatial/node/Control.scala:121`, `src/spatial/node/Control.scala:133`). Metadata also treats this as a controller breaker, not a general host-language `break`: `breaker` extracts the `breakWhen`/`stopWhen` symbol for those controller nodes, and `Ctrl.stopWhen` recovers the same option from controller metadata (`src/spatial/metadata/control/package.scala:423`, `src/spatial/metadata/control/package.scala:952`).

## Scalagen Divergence

Scalagen documents a real backend difference rather than silently defining the semantic contract. For forever counters, a break-enabled unrolled loop emits a warning, then generates `while(hasItems_$lhs && !${stopWhen.get}.value)` (`src/spatial/codegen/scalagen/ScalaGenController.scala:75`, `src/spatial/codegen/scalagen/ScalaGenController.scala:77`). For bounded counters, it emits the same warning and lowers to `cchain.takeWhile(!stopWhen.value)` (`src/spatial/codegen/scalagen/ScalaGenController.scala:88`, `src/spatial/codegen/scalagen/ScalaGenController.scala:90`). The warning text is decisive: "scala break occurs at the end of the loop, while --synth break occurs immediately" (`src/spatial/codegen/scalagen/ScalaGenController.scala:76`, `src/spatial/codegen/scalagen/ScalaGenController.scala:89`). So Scala simulation is acknowledged as end-of-iteration behavior, while synthesis is acknowledged as immediate-break behavior.

## RTL Wiring

Chisel generation wires `stopWhen` straight into the controller state machine. `ChiselGenController` builds `myBreak` from the stop register's read port, connects the stop register reset to controller `done`, and assigns `sm.io.break := myBreak` (`src/spatial/codegen/chiselgen/ChiselGenController.scala:314`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:315`). The adjacent comment says stream transformation may make an immediate child the host of the counter chain, so parent break/reset propagation was considered there, though the parent-break implementation is commented out (`src/spatial/codegen/chiselgen/ChiselGenController.scala:312`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:313`).

In Fringe, the controller interface has a `break` input (`fringe/src/fringe/templates/Controllers.scala:21`). `OuterControl` uses it to reset active stages and set `iterDone`/`done` in pipelined, sequenced, fork-join, and streaming schedules (`fringe/src/fringe/templates/Controllers.scala:105`, `fringe/src/fringe/templates/Controllers.scala:129`, `fringe/src/fringe/templates/Controllers.scala:151`, `fringe/src/fringe/templates/Controllers.scala:166`). It also acknowledges children when either `iterDone` or `break` is true (`fringe/src/fringe/templates/Controllers.scala:212`). `InnerControl` similarly resets `active` and sets `done` from `io.break` (`fringe/src/fringe/templates/Controllers.scala:265`, `fringe/src/fringe/templates/Controllers.scala:269`). This is stronger than a loop-predicate check: the break input is part of the controller state machine's completion and child-ack protocol.

## Datapath And Streams

The break signal is also visible to generated datapath enables. Kernel configuration copies `sm.io.break` into `mySignalsIn.break` (`fringe/src/fringe/SpatialBlocks.scala:135`), and Chisel codegen exposes that as the common `break` expression (`src/spatial/codegen/chiselgen/ChiselGenCommon.scala:125`). Memory read and write enables are gated by `~break`, with writes also using `~break && backpressure` as flow enable (`src/spatial/codegen/chiselgen/ChiselGenMem.scala:40`, `src/spatial/codegen/chiselgen/ChiselGenMem.scala:46`). Struct dequeue readiness is likewise gated by `~break` (`src/spatial/codegen/chiselgen/ChiselGenStruct.scala:35`). This means synthesized break behavior suppresses generated memory/stream-like accesses once the break signal is visible to the RTL, not merely future counter increments.

The stream passes treat `stopWhen` as scheduling-sensitive. `StreamTransformer` only folds an outer stream `Foreach` into a `UnitPipe` when `stopWhen` is empty (`src/spatial/transform/StreamTransformer.scala:77`). Binding also refuses to bind across active memories that are controller breakers, making `breakWhen` a boundary in grouping decisions (`src/spatial/transform/BindingTransformer.scala:41`, `src/spatial/transform/BindingTransformer.scala:42`). In hierarchical-to-stream lowering, the pass creates a fresh `stopWhen` register, passes it to a streaming `UnitPipe`, and writes it in the release stage when no time map remains or the last token is reached (`src/spatial/transform/streamify/HierarchicalToStream.scala:782`, `src/spatial/transform/streamify/HierarchicalToStream.scala:808`, `src/spatial/transform/streamify/HierarchicalToStream.scala:664`, `src/spatial/transform/streamify/HierarchicalToStream.scala:669`). For HLS, this implies `breakWhen` should be modeled as a controller enable/kill condition that can affect same-controller effects, not as an ordinary source loop `break` after body execution (unverified). Stream HLS also needs an explicit policy for pending tokens, release/drain behavior, and side-effect visibility when break is asserted (unverified).

## D-11 Implications

The local evidence points to synthesized immediate-break as Spatial's hardware semantic target. The public API comments say immediate, ChiselGen wires `stopWhen` into `sm.io.break`, the Fringe state machine consumes `io.break` in done/ack/reset logic, and generated memory/struct accesses are gated by `~break`. Scalagen's end-of-iteration behavior is real, but its own warning labels it as a Scala/synthesis divergence.

For D-11, a Rust simulator that aims to predict RTL/HLS should match immediate-break behavior by default. Matching Scalagen would preserve compatibility with existing Scala-emulation goldens that depend on end-of-iteration effects, but it would intentionally reproduce a documented backend mismatch. A reasonable split is: immediate-break for the normative simulator/HLS mode, and a clearly named Scalagen-parity mode only if legacy test comparison requires it.

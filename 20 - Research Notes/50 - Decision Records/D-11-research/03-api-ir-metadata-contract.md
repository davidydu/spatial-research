# D-11 Research: API/IR and Metadata Contract for `breakWhen`/`stopWhen`

## User API Surface

`breakWhen` enters the public control API as an optional `Reg[Bit]` that becomes a controller-level `stopWhen`. `Pipe` carries `stopWhen` in its constructor and forwards it into `CtrlOpt`; its normal `apply` stages a unit controller with that value, while the special `apply(breakWhen)(func)` stages a `UnitPipe` with `Some(breakWhen)` directly (`src/spatial/lang/control/Control.scala:22`, `src/spatial/lang/control/Control.scala:25`, `src/spatial/lang/control/Control.scala:27`). The chainable pipe modifiers preserve the stored field when constructing derived `Pipe`s (`src/spatial/lang/control/Control.scala:29`).

`Stream` and `Sequential` expose the stronger user-facing form: `apply(breakWhen: Reg[Bit])` returns a new directive object whose `CtrlOpt` carries the stop register, so subsequent `Foreach`, `Reduce`, or unit `apply` calls inherit it (`src/spatial/lang/control/Control.scala:35`, `src/spatial/lang/control/Control.scala:46`, `src/spatial/lang/control/Control.scala:49`, `src/spatial/lang/control/Control.scala:56`). Their comments are also the clearest semantic contract: the controller breaks immediately when the register is true and resets that register when the controller finishes; `Stream` adds a warning that the behavior is discouraged because it can be tricky (`src/spatial/lang/control/Control.scala:41`, `src/spatial/lang/control/Control.scala:42`, `src/spatial/lang/control/Control.scala:43`, `src/spatial/lang/control/Control.scala:53`, `src/spatial/lang/control/Control.scala:54`).

## `CtrlOpt` and Staging

`CtrlOpt` stores `stopWhen`, but `set` deliberately applies only name, schedule, II, unroll directives, no-bind, and starvation metadata; it does not write `stopWhen` as metadata (`src/spatial/lang/control/CtrlOpt.scala:8`, `src/spatial/lang/control/CtrlOpt.scala:12`, `src/spatial/lang/control/CtrlOpt.scala:18`, `src/spatial/lang/control/CtrlOpt.scala:25`). The stop register is therefore an IR constructor argument, not a side metadata field.

That shows up in staging. `Directives.unit_pipe` stages `UnitPipe(..., stopWhen)` and then applies `options.set` (`src/spatial/lang/control/Control.scala:16`, `src/spatial/lang/control/Control.scala:18`). `ForeachClass` builds iterators/counterchains, then stages `OpForeach(..., opt.stopWhen)` (`src/spatial/lang/control/ForeachClass.scala:26`, `src/spatial/lang/control/ForeachClass.scala:30`). `ReduceAccum` stages `OpReduce(..., stopWhen)` after creating map/load/reduce/store blocks (`src/spatial/lang/control/ReduceClass.scala:38`, `src/spatial/lang/control/ReduceClass.scala:49`), and `ReduceClass` passes `opt.stopWhen` into both implicit and explicit accumulator builders (`src/spatial/lang/control/ReduceClass.scala:78`, `src/spatial/lang/control/ReduceClass.scala:87`). `MemReduceAccum` likewise carries `stopWhen` through to `OpMemReduce` (`src/spatial/lang/control/MemReduceClass.scala:11`, `src/spatial/lang/control/MemReduceClass.scala:74`, `src/spatial/lang/control/MemReduceClass.scala:88`).

## IR and Unrolling Shape

The core IR schema is uniform: `UnitPipe`, `OpForeach`, `OpReduce`, and `OpMemReduce` each contain `stopWhen: Option[Reg[Bit]]` (`src/spatial/node/Control.scala:33`, `src/spatial/node/Control.scala:45`, `src/spatial/node/Control.scala:68`, `src/spatial/node/Control.scala:92`). Post-unrolling forms preserve the same slot on `UnrolledForeach` and `UnrolledReduce` (`src/spatial/node/Control.scala:121`, `src/spatial/node/Control.scala:127`, `src/spatial/node/Control.scala:133`, `src/spatial/node/Control.scala:139`).

The unrollers treat the breaker register like a memory duplicate that must be remapped. Foreach unrolling rewrites `stopWhen` through `memories((stopWhen.get,0))`, then threads it into either fully unrolled `UnitPipe`s or partially unrolled `UnrolledForeach` nodes (`src/spatial/transform/unrolling/ForeachUnrolling.scala:13`, `src/spatial/transform/unrolling/ForeachUnrolling.scala:14`, `src/spatial/transform/unrolling/ForeachUnrolling.scala:35`, `src/spatial/transform/unrolling/ForeachUnrolling.scala:69`). Reduce unrolling does the same for `OpReduce`, with fully unrolled reductions becoming `UnitPipe(..., stopWhen)` and partial reductions becoming `UnrolledReduce(..., stopWhen)` (`src/spatial/transform/unrolling/ReduceUnrolling.scala:17`, `src/spatial/transform/unrolling/ReduceUnrolling.scala:23`, `src/spatial/transform/unrolling/ReduceUnrolling.scala:79`, `src/spatial/transform/unrolling/ReduceUnrolling.scala:123`). MemReduce follows the same pattern, although one internal red-side `UnrolledForeach` intentionally uses `None` while the enclosing generated controller carries the stop (`src/spatial/transform/unrolling/MemReduceUnrolling.scala:14`, `src/spatial/transform/unrolling/MemReduceUnrolling.scala:18`, `src/spatial/transform/unrolling/MemReduceUnrolling.scala:201`, `src/spatial/transform/unrolling/MemReduceUnrolling.scala:204`, `src/spatial/transform/unrolling/MemReduceUnrolling.scala:270`).

## Metadata and Backend Contract

Metadata offers two lookup paths. `CtrlHierarchyOps.breaker` returns the `breakWhen` symbol by pattern matching the controller op, including unrolled forms (`src/spatial/metadata/control/package.scala:423`, `src/spatial/metadata/control/package.scala:425`, `src/spatial/metadata/control/package.scala:430`). `CtrlOps.stopWhen` similarly returns the stop register from `UnrolledForeach`, `OpForeach`, `UnrolledReduce`, `OpReduce`, `OpMemReduce`, or `UnitPipe`; `SymControlOps.stopWhen` delegates controller symbols to that lookup (`src/spatial/metadata/control/package.scala:756`, `src/spatial/metadata/control/package.scala:952`, `src/spatial/metadata/control/package.scala:958`).

`UseAnalyzer` marks any referenced break register as `isBreaker` across the same node families, including `UnitPipe` and unrolled loops (`src/spatial/traversal/UseAnalyzer.scala:57`, `src/spatial/traversal/UseAnalyzer.scala:62`). The memory metadata defines that flag as “a break for some controller,” default false (`src/spatial/metadata/memory/MemoryData.scala:41`, `src/spatial/metadata/memory/MemoryData.scala:47`), and accessors expose `isBreaker`/setter (`src/spatial/metadata/memory/package.scala:487`, `src/spatial/metadata/memory/package.scala:488`). Backend reset is concrete in Chisel: if `lhs.stopWhen` exists, it reads the breaker register, connects reset to `$done`, and drives `sm.io.break`; a nearby comment mentions parent break/reset propagation but that code is disabled (`src/spatial/codegen/chiselgen/ChiselGenController.scala:312`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:314`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:315`).

## D-11 Implications

The stable contract should make `stopWhen` an explicit IR field on every control node that can terminate early, then derive metadata from that field. Passes must remap it wherever memories are duplicated, preserve it through unrolled forms, and mark it with `isBreaker` so it is not treated as unused control state. The reset contract should be documented as: backend reads the breaker as the break signal and resets it on controller done. D-11 should also avoid positional `CtrlOpt` calls for new APIs, because `stopWhen` and control flags are easy to mis-route, and should audit PIR/other backends for parity with Chisel.

# Downstream Consumers of Pipe Holders

## Pipeline Position

`PipeInserter` is not a late codegen convenience. It runs after memory dealiasing and before reg-read CSE, DCE, streamify, FIFO initialization, banking analysis, unrolling, accumulation rewrites, retiming, reports, and backend codegen (`src/spatial/Spatial.scala:187-203`, `src/spatial/Spatial.scala:208-246`). Its inserted values are therefore part of the compiler contract consumed by most of the middle end. For each escaping inner-stage value, it stages a generated `Pipe`, writes the escaped value into a holder, reads the holder afterward, and registers later uses to that read (`src/spatial/transform/PipeInserter.scala:189-201`). The holder choice is semantic: bit values under stream control become `FIFOReg`, other bit values become `Reg`, and non-bit values become `Var` (`src/spatial/transform/PipeInserter.scala:256-260`, `src/spatial/transform/PipeInserter.scala:271-314`).

## Analysis and Scheduling Consumers

The banking/scheduling stack observes these holders as memories, not as SSA temporaries. `AccessAnalyzer` treats `RegRead` specially for loop invariance by checking whether all register writers are outside the loop, and later records register reads through the generic access-pattern path (`src/spatial/traversal/AccessAnalyzer.scala:45-55`, `src/spatial/traversal/AccessAnalyzer.scala:303-319`). `MemoryAnalyzer` creates fully banked configurers for both `Reg` and `FIFOReg`, while `MemoryAllocator` deliberately keeps only SRAM/FIFO/LIFO in the SRAM-able set and assigns non-SRAM memories, including these scalar holders, to the fallback resource (`src/spatial/traversal/MemoryAnalyzer.scala:125-131`, `src/spatial/traversal/MemoryAllocator.scala:21-23`, `src/spatial/traversal/MemoryAllocator.scala:35-37`, `src/spatial/traversal/MemoryAllocator.scala:102-103`). Scheduling also depends on memory identity: `findAccumCycles` groups readers and writers by memory, and `pipeLatencies` adds preceding writes to a read's dependencies through `cur.readMem` (`src/spatial/util/modeling.scala:152-163`, `src/spatial/util/modeling.scala:257-269`). Early SSA would erase those RAW/WAR/WAW anchors.

## Streamify, FIFO Init, and Unrolling

Streamify contains direct register-holder logic. `HierarchicalToStream` treats `RegNew(init)` as an inferred initialization edge, substitutes `RegRead` from current token values, and updates token state while still restaging `RegWrite` (`src/spatial/transform/streamify/HierarchicalToStream.scala:187-190`, `src/spatial/transform/streamify/HierarchicalToStream.scala:220-225`, `src/spatial/transform/streamify/HierarchicalToStream.scala:556-573`). FIFO initialization is also sequenced through holders: the pipeline defines `fifoInitialization` as `FIFOInitializer`, then `PipeInserter`, then a metadata stripper, and runs it before banking analysis (`src/spatial/Spatial.scala:138`, `src/spatial/Spatial.scala:200-203`). `FIFOInitializer` stages an initialization `UnitPipe` around FIFO writes (`src/spatial/transform/streamify/FIFOInitializer.scala:12-39`), so the following `PipeInserter` pass can reestablish legal control/primitive boundaries. Unrolling then restages holder accesses explicitly: `RegRead`, `FIFORegDeq`, `RegWrite`, and `FIFORegEnq` each have cases in `MemoryUnrolling` (`src/spatial/transform/unrolling/MemoryUnrolling.scala:550-553`).

## Backend and Check Consumers

Backends lower these nodes by opcode. Chisel emits `FIFORegNew`, `FIFORegEnq`, `FIFORegDeq`, `RegNew`, `RegWrite`, and `RegRead`, registers FIFOReg activity lanes, and maps Reg/FIFOReg to concrete module interfaces (`src/spatial/codegen/chiselgen/ChiselGenMem.scala:202-205`, `src/spatial/codegen/chiselgen/ChiselGenMem.scala:217-280`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:354-361`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:412-423`). Scala simulation emits `Reg` as a `Ptr`, `FIFOReg` as a queue, and `Var` as a `Ptr` (`src/spatial/codegen/scalagen/ScalaGenReg.scala:15-36`, `src/spatial/codegen/scalagen/ScalaGenFIFO.scala:46-52`, `src/spatial/codegen/scalagen/ScalaGenVar.scala:15-18`). Reports and names also assume the opcodes: naming distinguishes holder allocations and accesses, and resource counting classifies `RegNew`/`FIFORegNew` under `"reg"` (`src/spatial/codegen/naming/NamedCodegen.scala:41-56`, `src/spatial/codegen/resourcegen/ResourceCountReporter.scala:96-99`). Compiler checks immediately after insertion verify defined inputs and forbid mixed outer-control primitive/control blocks once `PipeInserter` disables that allowance (`src/spatial/traversal/CompilerSanityChecks.scala:35-52`, `src/spatial/traversal/CompilerSanityChecks.scala:115-123`, `src/spatial/transform/PipeInserter.scala:317-320`).

## Breakage From Early SSA

Replacing holders with SSA before these consumers would change more than syntax. It would remove stream backpressure/forward-pressure evidence for `FIFOReg`, whose deq/enq are classified as stream-stage enablers/holders and later used by Chisel pressure logic (`src/spatial/metadata/access/package.scala:59-75`, `src/spatial/codegen/chiselgen/ChiselGenCommon.scala:247-253`, `src/spatial/codegen/chiselgen/ChiselGenCommon.scala:266-272`). It would hide scalar storage from banking reports, allocation fallback, resource reports, and emitted hardware/simulation state. It would also make scheduling see a pure data edge where the current compiler sees a mutable read/write boundary. Therefore D-13 should treat `Reg`/`FIFOReg` pipe holders as observable compiler state until a later, targeted optimization proves equivalence after streamify, banking, unrolling, scheduling, and backend-specific state emission no longer need the holder identity. `Var` is weaker for hardware, since Chisel emits placeholder strings for `Var` nodes, but it is still a mutable IR bridge for non-bit escapes and should not be collapsed before its uses are proven host/debug-only (`argon/src/argon/node/Var.scala:7-21`, `src/spatial/codegen/chiselgen/ChiselGenDebug.scala:18-20`).

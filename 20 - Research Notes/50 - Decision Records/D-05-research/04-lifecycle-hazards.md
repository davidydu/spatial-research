# D-05 Research Angle 4: Lifecycle Hazards

## Construction Chain

The current Chisel path encodes the top-level build as an ordering protocol spread across generated files. `Instantiator` is opened with a delayed `dut` closure (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:167`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:170`), then later fills in target-dependent width and argument counts before constructing `new SpatialIP(this.target, () => Module(new AccelUnit(...)))` (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:239`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:245`). `SpatialIP` immediately pattern-matches that string into a concrete `DeviceTarget`, writes it to `globals.target`, installs `makeIO`, and only then calls the `accelGen` closure (`fringe/src/fringe/SpatialIP.scala:18`, `fringe/src/fringe/SpatialIP.scala:19`, `fringe/src/fringe/SpatialIP.scala:36`, `fringe/src/fringe/SpatialIP.scala:37`). The generated `AccelUnit` mixes in `AccelWrapper` and calls `Main.main(this)` inside the class body (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:263`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:276`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:279`). D-05 should treat this as an implicit phase graph, not as a model to preserve.

## Globals Before IO

`AccelWrapper` mutates process-global state immediately before building the top IO. The base postlude computes arg counts, writes them into `globals`, copies stream descriptors and allocator count into `globals`, then constructs `IO(new CustomAccelInterface(... globals.LOAD_STREAMS ... globals.AXI_STREAMS_OUT ...))` (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:197`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:203`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:213`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:215`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:216`). Other postludes feed that moment: interface generation emits scalar counts and sorted memory stream lists before calling `super.emitPostMain` (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:121`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:125`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:134`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:152`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:189`), while controller generation mutates target latency fields and global flags before its own `super` call (`src/spatial/codegen/chiselgen/ChiselGenController.scala:563`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:566`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:572`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:580`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:585`). The builder replacement must make "all structural counts, descriptors, target knobs, and build flags are finalized before IO construction" an explicit precondition.

## Empty Defaults

Several current defaults are not neutral zeros. Argument inputs and outputs are forced to at least one before reaching `globals` and the IO (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:198`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:199`). `globals` itself initializes arg inputs and outputs to one, streams to `Nil`, AXI stream descriptors to a non-empty 64-bit default, and allocator count to zero (`fringe/src/fringe/globals.scala:47`, `fringe/src/fringe/globals.scala:53`, `fringe/src/fringe/globals.scala:57`, `fringe/src/fringe/globals.scala:60`). But the accessors replace empty load/store/gather/scatter lists with a synthetic `StreamParInfo(DATA_WIDTH, WORDS_PER_STREAM, 0)`, which depends on `globals.target` already being valid (`fringe/src/fringe/globals.scala:62`, `fringe/src/fringe/globals.scala:65`, `fringe/src/fringe/globals.scala:18`, `fringe/src/fringe/globals.scala:20`). Empty AXI lists are similarly mapped to a 256-bit default, distinct from the initial 64-bit singleton defaults (`fringe/src/fringe/globals.scala:57`, `fringe/src/fringe/globals.scala:67`, `fringe/src/fringe/globals.scala:68`). The manifest should distinguish "absent resource", "dummy one-lane interface required by downstream code", and "target-derived default" as named policies.

## Singleton Reuse and Target Strings

`globals` is a singleton object with mutable `target`, build flags, top parameters, stream descriptors, and a `PrintWriter` opened at object initialization (`fringe/src/fringe/globals.scala:8`, `fringe/src/fringe/globals.scala:10`, `fringe/src/fringe/globals.scala:26`, `fringe/src/fringe/globals.scala:33`, `fringe/src/fringe/globals.scala:47`, `fringe/src/fringe/globals.scala:53`). In repeated elaborations within one process, fields that are not overwritten on every path can leak across runs; the exact driver reuse modes are not established here (unverified). Target identity is also derived twice from strings: codegen bakes width choices from `spatialConfig.target.name`, the generated instantiator recomputes `w` from `this.target`, and `SpatialIP` separately maps the string to a target object (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:144`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:145`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:239`, `fringe/src/fringe/SpatialIP.scala:19`, `fringe/src/fringe/SpatialIP.scala:33`). Since the runtime `w` check only names lowercase `zcu` but uppercase `VCS` and `ASIC`, while `SpatialIP` accepts both lowercase and uppercase for VCS/ASIC, width derivation can diverge from target selection (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:239`, `fringe/src/fringe/SpatialIP.scala:21`, `fringe/src/fringe/SpatialIP.scala:31`).

## Builder Guarantees

The Rust/HLS manifest should make four ordering guarantees explicit. First, canonicalize the target id once, before deriving widths, stream defaults, latency knobs, or shell/fringe behavior (`fringe/src/fringe/SpatialIP.scala:19`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:239`). Second, build an immutable `ApplicationTop` record from IR-derived arg counts, loopback maps, memory streams, AXI streams, allocator count, instrumentation outputs, and breakpoint outputs before any top IO is created (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:125`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:559`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:240`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:245`). Third, apply target/build overlays as values, not mutations of a shared `DeviceTarget` (`src/spatial/codegen/chiselgen/ChiselGenController.scala:566`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:579`, `fringe/src/fringe/globals.scala:10`). Fourth, instantiate in a single direction: manifest validation, top IO shape, accelerator body wiring, then fringe/shell wrapping. That replaces the current hidden handshake among `Instantiator`, `SpatialIP`, `globals`, `AccelWrapper`, and `Main`.

# D-05 Research Angle 9: Precedent For Manifested Tops

## Local Problem

Spatial already constructs a manifest-like top boundary, but it is scattered across generated Scala and elaboration-time global mutation. `ChiselCodegen.emitPreMain` opens `AccelWrapper`, emits target-dependent `io_w` and `io_v`, starts `Instantiator`, and creates the generated `Args` object (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:129`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:143`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:148`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:190`). `emitEntry` then wraps the app as `Main.main(accelUnit)`, clears the IO, emits the application body, and finally runs `emitPostMain` (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:293`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:296`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:299`). The top is therefore an ordered construction protocol, not merely a module declaration.

`SpatialIP` is the integration root: it maps a target string to a concrete `DeviceTarget`, installs `makeIO`, builds the delayed accelerator, and asks the target to add Fringe and create final IP IO (`fringe/src/fringe/SpatialIP.scala:18`, `fringe/src/fringe/SpatialIP.scala:19`, `fringe/src/fringe/SpatialIP.scala:36`, `fringe/src/fringe/SpatialIP.scala:39`). D-05 should preserve that boundary, but make the contract explicit and immutable.

## What The Globals Carry

`fringe.globals` names the payload that a Rust/HLS manifest must replace. It holds the mutable `target`, plus target-derived accessors for BigIP, pipeline depth, address/data widths, stream geometry, external/target widths, and channel count (`fringe/src/fringe/globals.scala:8`, `fringe/src/fringe/globals.scala:10`, `fringe/src/fringe/globals.scala:13`, `fringe/src/fringe/globals.scala:18`, `fringe/src/fringe/globals.scala:24`). It also stores generation flags such as `retime`, `perpetual`, `enableModular`, debug/verbose settings, and channel assignment (`fringe/src/fringe/globals.scala:26`, `fringe/src/fringe/globals.scala:31`). The TCL writer and regression environment flag show that side-effecting build state is currently mixed into the same singleton (`fringe/src/fringe/globals.scala:34`, `fringe/src/fringe/globals.scala:43`).

The top-level ABI state is even more direct: comments say these parameters are set by the generated `Instantiator`, followed by mutable argument counts, loopback map, memory stream descriptors, AXI stream descriptors, and allocator count (`fringe/src/fringe/globals.scala:45`, `fringe/src/fringe/globals.scala:47`, `fringe/src/fringe/globals.scala:53`, `fringe/src/fringe/globals.scala:57`, `fringe/src/fringe/globals.scala:60`). Fallback accessors synthesize default stream and AXI descriptors when lists are empty, while derived counts expose the final ABI shape (`fringe/src/fringe/globals.scala:62`, `fringe/src/fringe/globals.scala:67`, `fringe/src/fringe/globals.scala:70`, `fringe/src/fringe/globals.scala:78`).

## Generation Already Resembles A Manifest

`ChiselGenInterface` emits ordered scalar argument counts and a loopback map, then emits load/store/gather/scatter stream lists and memory-argument count (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:125`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:131`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:134`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:138`). It repeats the same shape as `io_` values in `AccelWrapper` (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:141`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:148`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:152`). `ChiselGenController` emits app/build characteristics, instrumentation counts, breakpoint count, `max_latency`, arithmetic latency overrides, SRAM choices, retiming, perpetual-IP, channel-assignment, and modular-generation flags (`src/spatial/codegen/chiselgen/ChiselGenController.scala:551`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:559`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:564`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:566`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:572`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:581`).

`ChiselCodegen.emitPostMain` combines those values, writes them into `globals`, creates `CustomAccelInterface` from global accessors, and keeps local mutable maps for stream/buffer lane allocation (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:197`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:203`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:215`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:218`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:230`). It then passes the same derived payload to `SpatialIP` and `AccelUnit` constructors (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:239`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:245`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:263`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:279`).

## External Precedents And D-05 Shape

HLS flows commonly make the top-function signature the primary ABI declaration: scalar parameters, pointer or stream parameters, and return/control behavior become the host/kernel boundary (unverified). Vitis/Vivado HLS-style pragmas typically decorate that signature with AXI-lite control, `m_axi` memory, and AXI-stream protocol choices (unverified). Kernel metadata and IP-XACT-like packaging then provide the machine-readable SoC view: ports, bus interfaces, parameters, clocks, resets, address maps, and generated artifacts (unverified). JSON/TOML build manifests are also common for target part/board, source lists, kernel name, clock, constraints, and output paths (unverified).

The precedent is not one file format; it is the separation of declarative contract from implementation-time side effects. D-05 should define a Rust `TopManifest` as the canonical API, with JSON as the stable interchange and TOML only as a derived review/debug view (unverified). Required fields should include target identity and derived widths, host ABI ordering and loopbacks, memory and AXI stream descriptors, allocator descriptors/count, instrumentation and breakpoint counts, lane assignments, build flags, latency/SRAM assumptions, and artifact paths. HLS generation, host-stub generation, and SoC packaging should receive `&TopManifest` or a validated deserialized copy. Nothing should read a process-global singleton to discover the top-level contract.

# D-05 Research Angle 1: Source Global Surface

## Construction Surface

The Chisel path builds the top-level surface in generated Scala, then patches Fringe globals while elaborating it. `emitPreMain` opens `AccelWrapper`, emits target-dependent IO widths `io_w` and `io_v`, starts `Instantiator`, and creates `Args` (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:129`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:143`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:148`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:190`). `emitEntry` wraps the app as `Main.main(accelUnit)`, clears `accelUnit.io`, emits the app, then runs `emitPostMain` (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:293`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:296`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:299`). `SpatialIP` is the other root: it turns `targetName` into a concrete `DeviceTarget`, installs `makeIO`, creates the delayed `AccelUnit`, and asks the target to add Fringe and return final IO (`fringe/src/fringe/SpatialIP.scala:18`, `fringe/src/fringe/SpatialIP.scala:19`, `fringe/src/fringe/SpatialIP.scala:36`, `fringe/src/fringe/SpatialIP.scala:39`).

## Fringe Globals

`fringe.globals` contains the mutable state that the Rust/HLS manifest must replace. `target` is the primary mutable handle; derived accessors expose `bigIP`, pipeline depth, address/data widths, stream word geometry, external/target widths, and channel count from that target (`fringe/src/fringe/globals.scala:10`, `fringe/src/fringe/globals.scala:13`, `fringe/src/fringe/globals.scala:16`, `fringe/src/fringe/globals.scala:18`, `fringe/src/fringe/globals.scala:24`). Runtime/build flags are `retime`, `perpetual`, `enableModular`, `enableVerbose`, `enableDebugRegs`, and `channelAssignment` (`fringe/src/fringe/globals.scala:26`, `fringe/src/fringe/globals.scala:31`). Non-structural process state includes the `bigIP.tcl` `PrintWriter` plus setter and `regression_testing` from the environment (`fringe/src/fringe/globals.scala:34`, `fringe/src/fringe/globals.scala:40`, `fringe/src/fringe/globals.scala:43`). Application-top globals are `numArgIns`, `numArgOuts`, `numArgIOs`, `numArgInstrs`, `argOutLoopbacksMap`, four memory stream lists, two AXI stream lists, and `numAllocators` (`fringe/src/fringe/globals.scala:45`, `fringe/src/fringe/globals.scala:53`, `fringe/src/fringe/globals.scala:57`, `fringe/src/fringe/globals.scala:60`). Fallback accessors synthesize default stream descriptors and derived counts when lists are empty (`fringe/src/fringe/globals.scala:62`, `fringe/src/fringe/globals.scala:67`, `fringe/src/fringe/globals.scala:70`, `fringe/src/fringe/globals.scala:78`).

## Generated Manifest Values

`ChiselGenInterface` emits scalar counts and stream manifests twice: plain `numArgIns_reg`, `numArgOuts_reg`, `numArgIOs_reg`, `argOutLoopbacksMap`, `loadStreamInfo`, `storeStreamInfo`, `gatherStreamInfo`, `scatterStreamInfo`, and `numArgIns_mem` for `Instantiator`, plus `io_` variants for `AccelWrapper` (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:121`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:131`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:134`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:141`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:148`, `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:152`). It also emits mutable `outArgMuxMap` for wrapper-local output selection (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:153`). AXI stream descriptors are emitted as `io_axiStreamInsInfo`, `io_axiStreamOutsInfo`, `axiStreamInsInfo`, and `axiStreamOutsInfo` (`src/spatial/codegen/chiselgen/ChiselGenStream.scala:187`, `src/spatial/codegen/chiselgen/ChiselGenStream.scala:195`). DRAM codegen emits `io_numAllocators` and `numAllocators` (`src/spatial/codegen/chiselgen/ChiselGenDRAM.scala:74`, `src/spatial/codegen/chiselgen/ChiselGenDRAM.scala:81`). Controller codegen emits `numArgOuts_instr`, `numCtrls`, `numArgOuts_breakpts`, their `io_` wrapper variants, and `max_latency` (`src/spatial/codegen/chiselgen/ChiselGenController.scala:537`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:540`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:559`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:564`).

## Global Writes and Local Mutables

`ChiselCodegen.emitPostMain` combines scalar counts into `io_numArgIns`, `io_numArgOuts`, `io_numArgIOs`, `io_numArgInstrs`, and `io_numArgBreakpts`, then writes most of them into `globals` with stream descriptors and allocator count (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:197`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:203`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:207`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:213`). It builds `io` as `CustomAccelInterface` from global stream/count accessors (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:215`). Wrapper-local mutable lane allocators are `outStreamMuxMap`, `outBuffMuxMap`, and `inStreamMuxMap`, each consumed by a `get*Lane` method that increments per ID (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:218`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:224`, `src/spatial/codegen/chiselgen/ChiselCodegen.scala:230`). The generated `argOutLoopbacksMap` appears as local generated values, while `globals.argOutLoopbacksMap` exists separately; in the cited path I found no write connecting them (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:131`, `fringe/src/fringe/globals.scala:51`).

## Replacement Fields

The replacement manifest should carry immutable fields for target identity, target-derived widths, channel count, BigIP/shell binding, IO construction policy, scalar argument counts, loopback map, stream descriptors, AXI stream descriptors, allocator count, instrumentation outputs, controller count, breakpoint count, and wrapper lane allocation results. It also needs build-option overrides for arithmetic latencies, SRAM load/store latency, mux latency, cheap-SRAM mode, SRAM threshold, retiming, modular generation, perpetual IP, and channel assignment, because these are currently emitted as mutations on `globals.target` or `globals` (`src/spatial/codegen/chiselgen/ChiselGenController.scala:566`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:573`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:577`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:581`). `DeviceTarget` confirms those latency/SRAM knobs and `makeIO` are mutable target fields, not just constants (`fringe/src/fringe/targets/DeviceTarget.scala:20`, `fringe/src/fringe/targets/DeviceTarget.scala:31`, `fringe/src/fringe/targets/DeviceTarget.scala:62`). The manifest should also account for side outputs such as TCL generation and regression mode as explicit build artifacts/options rather than global process state (`fringe/src/fringe/globals.scala:34`, `fringe/src/fringe/globals.scala:43`).

# D-22 Angle 7: Implementation Cost and Migration Plan

## Policy Cost Matrix

D-22 is a project-wide simulator/HLS contract, not a local queue helper. The queue entry asks whether the Rust simulator and HLS semantics use ScalaGen elastic queues or bounded FIFO/LIFO/stream back-pressure with overflow checks (`Spatial Research/20 - Research Notes/40 - Decision Queue.md:95-97`). **Scalagen elastic only** is the cheapest migration: preserve `Queue`/`Stack`, advisory `isFull`, invalid-on-empty reads, and enabled writes that never test capacity (`src/spatial/codegen/scalagen/ScalaGenFIFO.scala:12-44`; `src/spatial/codegen/scalagen/ScalaGenLIFO.scala:12-44`; `src/spatial/codegen/scalagen/ScalaGenStream.scala:84-97`). Cost: low implementation, low test churn, high HLS divergence. **Bounded back-pressure only** is the most disruptive: Chisel pressure gates stream valid/FIFO non-empty/FIFO not-full and is II-sensitive (`src/spatial/codegen/chiselgen/ChiselGenCommon.scala:247-279`), while FIFO/LIFO templates are finite counter-backed memories (`fringe/src/fringe/templates/memory/MemPrimitives.scala:380-442`; `fringe/src/fringe/templates/memory/MemPrimitives.scala:447-541`). Cost: high implementation and migration, best liveness fidelity. **Dual-mode, elastic reference default plus bounded/cycle debug mode** is medium-high cost but best staged migration: keep value goldens, add bounded diagnostics for HLS/debug, and avoid making D-15/D-20/D-21 implicit (`D-15-research/10-recommendation-matrix.md:23-29`; `D-20-research/09-overlap-d15-d21-d22.md:16-18`). **HLS-native-only** is attractive for codegen but weak as a simulator plan: `hls::stream`/ring-buffer lowering can be correct only after local fixtures prove stalls and reports (`D-22-research/04-hls-stream-fifo-mapping.md:16-38`).

## Affected Implementation Surfaces

The Rust simulator needs a policy-tagged channel model: elastic queue/stack for compatibility, bounded FIFO ring, bounded LIFO stack, external stream endpoints, vector-lane atomicity, invalid generation, occupancy, high-water mark, and stall state. The existing Scala executor is useful readback but incomplete as a bounded template: `ScalaQueue` has capacity and throws on full/empty (`src/spatial/executor/scala/memories/ScalaQueue.scala:8-24`), `ExecPipeline.willStall` precomputes enq/deq counts against capacity (`src/spatial/executor/scala/ExecPipeline.scala:185-247`), but streams are often allocated as default unbounded queues and file streams bypass `enq` (`src/spatial/executor/scala/resolvers/MemoryResolver.scala:28-41`), LIFO pop is unimplemented (`src/spatial/executor/scala/ExecPipeline.scala:179-180`), and the top-level loop ticks until `Done` without a timeout (`src/spatial/executor/scala/ExecutorPass.scala:45-67`). Parallel execution also matters: `ParallelPipe` in ScalaGen is textual (`src/spatial/codegen/scalagen/ScalaGenController.scala:187-191`), while the executor ticks child executors under one stream/unit pipe (`src/spatial/executor/scala/ControlExecutor.scala:207-259`).

HLS lowering must split local and external cases. FIFO/LIFO depths come from IR dimensions (`src/spatial/node/FIFO.scala:7-19`; `src/spatial/node/LIFO.scala:7-9`) and `stagedSize`/width metadata (`src/spatial/metadata/memory/package.scala:278-331`). External AXI streams already have endpoint logic and current one-in/one-out limits (`src/spatial/codegen/chiselgen/ChiselGenStream.scala:17-186`). Local LIFO should not lower to FIFO-ordered streams; use an explicit bounded stack or reject in HLS mode (`D-22-research/04-hls-stream-fifo-mapping.md:34-36`).

## Schema and Diagnostics

Add sidecar records rather than overloading one result. Per channel: `channel_uid`, `kind`, `policy`, `depth_source`, `capacity`, `occupancy_trace`, `high_water`, `read_width`, `write_width`, `producer_controller_uid`, `consumer_controller_uid`, and `endpoint_kind`. Per event: `cycle`, `op`, `lane_mask`, `action = commit | stall | invalid | overflow | underflow`, `source_span`, and `evidence_ids`. D-21 already requires structured diagnostics with severity, code, field, reason, source span, evidence, and suggested action (`D-21-research/05-ii-reconciliation-schema.md:20-31`); D-22 should add queue codes for `write_full`, `read_empty`, `deadlock_timeout`, `elastic_growth`, `lifo_hls_unsupported`, and `mode_mismatch`. For cycle modes, include assumed II/provenance because FIFO pressure uses `.D(II-1)` in Chisel and D-20 requires the delay-line flow predicate to be named (`src/spatial/codegen/chiselgen/ChiselGenCommon.scala:249-293`; `D-20-research/09-overlap-d15-d21-d22.md:10-18`).

## Migration and Tests

Phase 0: label current behavior `compat_scalagen_elastic` and preserve file-backed stream parsing/dumping as compatibility only (`Spatial Research/10 - Spec/50 - Code Generation/20 - Scalagen/40 - FIFO LIFO Stream Simulation.md:140-151`). Phase 1: add schemas, mode flags, and diagnostics while still running elastic by default. Phase 2: implement bounded debug simulation behind an opt-in flag, with deterministic scheduling, timeout, and first-blocked-channel reports. Phase 3: add HLS lowering for bounded FIFO/ring, explicit LIFO stack/rejection, stream endpoint mapping, and D-21 II/report linkage. Phase 4: consider changing defaults only after fixtures pass.

Test migration should be explicit. Existing tests include elastic or ambiguous cases, such as `FIFOBanking2` overfilling depth 5 with 32 values, frame-to-FIFO depth mismatches, and apps that pad until `isFull` with timing comments (`D-22-research/03-tests-app-usage.md:9-15`). Missing fixtures are the real cost: intentional overflow, same-cycle `isFull`/enqueue, empty dequeue, small-depth stream back-pressure, LIFO overflow/underflow, `ParallelPipe` producer-consumer stalls, and HLS report round trips (`D-22-research/03-tests-app-usage.md:17-19`). The recommended migration is dual-mode: elastic remains the reference value oracle, bounded/cycle mode becomes the HLS/debug oracle, and HLS-native-only remains a backend validation lane, not the sole simulator story.

## Search scope

D-22's test/app surface is mostly in `test/spatial/tests`, not `apps/src`: the two top-level app examples use DRAM/SRAM and host IO only (`spatial/apps/src/HelloSpatial.scala:37-58`, `spatial/apps/src/TextFileIO.scala:14-18`). I searched tests/apps for `FIFO`, `LIFO`, `StreamIn`, `StreamOut`, `Parallel`, `enq`, `deq`, `isFull`, `isEmpty`, `numel`, and streaming kernels. The semantic baseline remains [[40 - FIFO LIFO Stream Simulation]]: Scalagen maps FIFO and StreamIn/StreamOut to `mutable.Queue`, LIFO to `mutable.Stack`, and enabled writes push without capacity checks (`spatial/src/spatial/codegen/scalagen/ScalaGenFIFO.scala:12-44`, `spatial/src/spatial/codegen/scalagen/ScalaGenLIFO.scala:12-44`, `spatial/src/spatial/codegen/scalagen/ScalaGenStream.scala:12-97`). That makes existing tests a mixed signal: many protect value order, while few prove bounded back-pressure.

## Value and status fixtures

The straightforward FIFO/LIFO tests mostly pass under either elastic or bounded semantics if scheduling never overfills the structure. `FIFOPushPop` enqueues one tile and immediately drains it, with depth twice the tile (`spatial/test/spatial/tests/feature/memories/fifo/FIFOPushPop.scala:18-24`); `FIFOLoad` loads exactly a `tileSize` FIFO then dequeues into SRAM (`spatial/test/spatial/tests/feature/memories/fifo/FIFOLoad.scala:21-30`); `MemCopyLIFO` asserts reverse order through a full-depth LIFO load/store (`spatial/test/spatial/tests/feature/transfers/MemCopyLIFO.scala:15-41`). Status coverage exists but is value-level: `FSMFIFOStack` drives FIFO/LIFO FSM state transitions from `isFull`, `isEmpty`, `isAlmostFull`, and `isAlmostEmpty` (`spatial/test/spatial/tests/feature/control/fsm/FSMFIFOStack.scala:21-82`) and checks sums/last values (`spatial/test/spatial/tests/feature/control/fsm/FSMFIFOStack.scala:87-128`). `Scan_fifofull` stores when `fifo.isFull`, samples `isFull`/`isEmpty`, and drains by `fifo.numel` (`spatial/test/spatial/tests/feature/control/Filter.scala:32-70`). These fixtures need status-reader compatibility, but they do not assert what happens on a write when full.

## Elastic or ambiguous dependencies

Some tests would break, hang, or become mode-dependent under a strictly bounded sequential simulator. The clearest elastic-only case is `FIFOBanking2`: it creates `FIFO[Int](5)`, enqueues `N = 32` values, then reduces 32 dequeues, and finally only `assert(true)` (`spatial/test/spatial/tests/feature/banking/FIFOBanking.scala:24-46`). That is not a bounded FIFO fixture; it is a compile/scheduling smoke test that Scalagen can run because enqueue is elastic. `FrameLoadStoreTest` is also ambiguous: a `Frame[U32](64)` is loaded into `FIFO[U32](8)`, transformed by 64 dequeues, and stored to a stream frame (`spatial/test/spatial/tests/feature/transfers/LoadStoreUnitTests.scala:51-72`). A bounded simulator must either model that load/dequeue/store as concurrent flow or reject the sequential fill. The Smith-Waterman and MachSuite alignment kernels show real app pressure around `isFull`: both pad FIFOs until full and include a comment that the FSM can see full too late (`spatial/test/spatial/tests/apps/SW.scala:146-156`, `spatial/test/spatial/tests/apps/MachSuite.scala:921-935`). `CompactingFIFO` avoids `isFull` entirely, saying it "turns on prematurely", and uses `numel == tileSize` instead (`spatial/test/spatial/tests/feature/memories/fifo/CompactingFIFO.scala:36-46`).

## Streaming and ParallelPipe overlap

The streaming tests are where bounded flow matters most, and this overlaps D-15. `ContinualStreaming` explicitly creates a slow producer that "causes starves" and a consumer that conditionally dequeues (`spatial/test/spatial/tests/compiler/ContinualStreaming.scala:18-42`). `StreamPipeFlush` uses unbounded stream loops, `priorityDeq`, flush FIFOs, an output FIFO, and in the complex case a second input FIFO of depth 1 (`spatial/test/spatial/tests/feature/dynamic/StreamPipeFlush.scala:13-48`, `spatial/test/spatial/tests/feature/dynamic/StreamPipeFlush.scala:69-110`). `PriorityDeq` has producer/consumer streams with done queues, payload-empty guards to avoid hangs, and comments about timing mismatches between enqueue and dequeue (`spatial/test/spatial/tests/feature/dynamic/PriorityDeq.scala:29-75`, `spatial/test/spatial/tests/feature/dynamic/PriorityDeq.scala:103-187`, `spatial/test/spatial/tests/feature/dynamic/PriorityDeq.scala:273-358`). `DotProduct` documents the D-15 interaction directly: two parallel DRAM loads must be serviced equally to avoid stalling the `Stream` controller (`spatial/test/spatial/tests/feature/dense/DotProduct.scala:109-128`). Systolic matmul similarly drains FIFO arrays inside a `Parallel` region and uses a FIFO done token (`spatial/test/spatial/tests/feature/dense/MatMult_systolic.scala:34-69`).

## Missing fixtures

The suite needs explicit D-22 fixtures before choosing bounded semantics as the default. Add a dual-mode test that intentionally overfills a small FIFO and records elastic growth versus bounded stall/error; add same-cycle `isFull`/enqueue and `isEmpty`/dequeue tests for FIFO and LIFO; add a bounded `FrameLoadStoreTest` variant with FIFO depth smaller than frame length and a slow sink/source; add a `StreamIn`/`StreamOut` ready/valid test with back-pressure rather than only `setFrame`/`getFrame` (`spatial/test/spatial/tests/feature/transfers/LoadStoreUnitTests.scala:8-99`). Also make disabled or commented streaming examples actionable: `HelloCXP` and `CXP1DFilter` both declare AXI streams but disable backends (`spatial/test/spatial/tests/feature/streaming/HelloCXP.scala:5-17`, `spatial/test/spatial/tests/feature/streaming/CXP1DFilter.scala:6-63`), while many video stream files are commented at the package line (`spatial/test/spatial/tests/feature/streaming/StreamingSobel.scala:1-25`, `spatial/test/spatial/tests/feature/streaming/ShiftReg.scala:1-21`). Without these, elastic Scalagen parity is well covered, but bounded back-pressure is mostly inferred.

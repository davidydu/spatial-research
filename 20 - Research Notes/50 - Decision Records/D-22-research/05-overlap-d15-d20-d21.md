# D-22 Angle 5: Overlap With D-15, D-20, And D-21

## Shared Boundary

D-22 cannot choose FIFO/LIFO/stream behavior as an isolated container policy. It is the boundary between ScalaGen value compatibility and any bounded timing model. ScalaGen maps FIFO and streams to `mutable.Queue`, LIFO to `mutable.Stack`, reports full from current size, but still enqueues/pushes when enabled without enforcing capacity (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFIFO.scala:12-44`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenLIFO.scala:12-44`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenStream.scala:12-97`). The Scalagen spec names this "silent elastic enqueue" and warns that overflow can pass simulation while failing synthesized hardware (`10 - Spec/50 - Code Generation/20 - Scalagen/40 - FIFO LIFO Stream Simulation.md:140-149`). [[D-15]] explicitly leaves bounded stalls and deadlock diagnostics to D-22 (`20 - Research Notes/50 - Decision Records/D-15.md:30-34`), [[D-20]] makes cycle delay lines depend on queue/backpressure policy (`20 - Research Notes/50 - Decision Records/D-20.md:54-61`), and [[D-21]] requires future queue traces to declare their II source (`20 - Research Notes/50 - Decision Records/D-21.md:86-95`).

## D-15 Tick Order

Queue semantics depend first on parent/child tick ordering. In ScalaGen, `ParallelPipe` just wraps the body, calls `gen(func)`, and marks done (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenController.scala:187-191`); block generation visits statements in source order (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaCodegen.scala:37-53`). The Scala executor is different: it constructs child executors, ticks them in deterministic order on each parent tick, and finishes when all child statuses are done, disabled, or indeterminate (`/Users/david/Documents/David_code/spatial/src/spatial/executor/scala/ControlExecutor.scala:207-246`). Hardware is different again: flow rules assign `ParallelPipe` to `ForkJoin`, and Fringe synchronizes all child `iterDone` bits under `io.backpressure` before parent progress (`/Users/david/Documents/David_code/spatial/src/spatial/flows/SpatialFlowRules.scala:318-333`, `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/Controllers.scala:141-154`). Therefore D-22 bounded mode must define whether same-cycle enqueue/dequeue pairs observe pre-state, post-state, or an explicit two-phase snapshot/commit rule. Without that, a producer child and consumer child connected by a FIFO can change behavior solely because D-15 chose source-order, cooperative, or fork/join stepping.

## D-20 Delay Policy

Delay-line simulation inherits the same stall predicate. ScalaGen elides `DelayLine` to a value alias (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenDelays.scala:7-15`), while Chisel emits delayed wires through `DL(...)` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenDelay.scala:16-30`). That helper advances retimed values only under `backpressure` and, for `haltIfStarved`, `forwardpressure` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:288-293`). [[D-20]] therefore rejects unlabelled cycle-aware delay behavior and says cycle mode needs tick policy, flow gating, reset/init, queue policy, and II source (`20 - Research Notes/50 - Decision Records/D-20.md:50-65`). D-22 must make bounded queues and delay lines advance from one shared "accepted controller step" event. If a child stalls on empty/full, its delay-line state must hold; if D-22 chooses elastic compatibility, D-20 should remain `compat_value_elided` except for clearly labelled debug traces.

## D-21 II Source

Back-pressure is also II-sensitive. Spatial currently stores selected `II`, optional `userII`, and `compilerII` separately (`/Users/david/Documents/David_code/spatial/src/spatial/metadata/control/package.scala:832-839`). `InitiationAnalyzer` computes outer `compilerII` from child II and inner `compilerII` from interval, latency, and iteration-distance evidence, then chooses effective `II` from user override or compiler policy (`/Users/david/Documents/David_code/spatial/src/spatial/traversal/InitiationAnalyzer.scala:14-41`). Chisel pressure expressions retime FIFO empty/full checks by `sym.s.get.II - 1` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:247-277`). [[D-21]] keeps user-requested, compiler, selected, requested-HLS, and accepted-HLS II as separate claims, never overwriting them (`20 - Research Notes/50 - Decision Records/D-21.md:46-55`). D-22 bounded traces must therefore record `ii_source` and `ii_value`: pre-report simulations may use selected/compiler II only as labelled estimates; post-report timing should prefer accepted II when mapping is trustworthy.

## Composition Constraints

D-22 should impose a mode matrix rather than one unqualified simulator behavior. `compat_elastic_scalagen` preserves unbounded queues, source-order invalid-on-empty behavior, and no deadlock claim. `bounded_backpressure` requires D-15 `hls_semantic` or an equivalent tick model, D-20 shared delay advance, and D-21 labelled II provenance. In bounded mode, D-22 must specify finite capacity, simultaneous multi-lane atomicity, empty/full stall versus diagnostic policy, ready/valid stream behavior, deterministic arbitration for same-cycle producers and consumers, high-water/occupancy reporting, timeout/deadlock classification, and how accepted-II mismatches invalidate or relabel older traces. That is the minimum contract for D-15, D-20, and D-21 to compose without silently promoting ScalaGen elasticity into HLS timing truth.

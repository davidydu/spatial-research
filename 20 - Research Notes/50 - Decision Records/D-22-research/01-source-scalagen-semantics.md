## Source Scope

The D-22 evidence points to generated Scala, not a cycle simulator. The vault note [[40 - FIFO LIFO Stream Simulation]] already identifies Scalagen's reference shape: FIFO and streams become `scala.collection.mutable.Queue`, LIFO becomes `scala.collection.mutable.Stack`, and enqueue is "silent elastic" (`10 - Spec/50 - Code Generation/20 - Scalagen/40 - FIFO LIFO Stream Simulation.md:21-30`, `10 - Spec/50 - Code Generation/20 - Scalagen/40 - FIFO LIFO Stream Simulation.md:138-145`). The source confirms that this is a value-level collection model: FIFO remaps to `mutable.Queue` and allocates an empty queue object (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFIFO.scala:12-18`); LIFO remaps to `mutable.Stack` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenLIFO.scala:12-18`); StreamIn and StreamOut remap to queues (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenStream.scala:12-16`).

## ScalaGen Queue Behavior

FIFO status is computed from current collection size. `isEmpty` is `fifo.isEmpty`, `isFull` is `fifo.size >= fifo.stagedSize`, and almost-empty/full use `readWidths`, `writeWidths`, and `stagedSize` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFIFO.scala:19-31`). LIFO mirrors this with stack size predicates (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenLIFO.scala:19-29`). These predicates are observational only. FIFO banked enqueue emits only `if (en) fifo.enqueue(data(i))` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFIFO.scala:41-44`), and FIFOReg enqueue is the same scalar pattern (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFIFO.scala:46-52`). LIFO banked push emits only `if (en) lifo.push(data(i))` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenLIFO.scala:39-44`). StreamOut banked write also just enqueues enabled lanes (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenStream.scala:92-97`). Therefore ScalaGen is unbounded in effect: it can report full and still accept more values.

## IR And Metadata Surface

The IR does carry bounded intent. `FIFONew(depth)` and `LIFONew(depth)` store `depth` as the one memory dimension (`/Users/david/Documents/David_code/spatial/src/spatial/node/FIFO.scala:7-19`, `/Users/david/Documents/David_code/spatial/src/spatial/node/LIFO.scala:7-9`), while StreamIn and StreamOut allocate a one-element dimension placeholder (`/Users/david/Documents/David_code/spatial/src/spatial/node/StreamIn.scala:7-9`, `/Users/david/Documents/David_code/spatial/src/spatial/node/StreamOut.scala:7-9`). FIFO and LIFO define enqueue/dequeue, peek, empty/full, almost-empty/full, numel, and banked variants as separate nodes (`/Users/david/Documents/David_code/spatial/src/spatial/node/FIFO.scala:22-71`, `/Users/david/Documents/David_code/spatial/src/spatial/node/LIFO.scala:11-34`). Stream nodes expose read and write directions only (`/Users/david/Documents/David_code/spatial/src/spatial/node/StreamIn.scala:11-20`, `/Users/david/Documents/David_code/spatial/src/spatial/node/StreamOut.scala:11-27`). Metadata turns the dimension into `stagedSize`, and derives read/write widths from unrolled accessors (`/Users/david/Documents/David_code/spatial/src/spatial/metadata/memory/package.scala:278-331`). It also classifies streams, FIFOs, LIFOs, and destructive reads (`/Users/david/Documents/David_code/spatial/src/spatial/metadata/memory/package.scala:385-413`). Access metadata marks dequeues and stream reads as stream-stage enablers, and enqueues, pushes, and stream writes as stream-stage holders (`/Users/david/Documents/David_code/spatial/src/spatial/metadata/access/package.scala:59-82`).

## Valid And Empty Behavior

ScalaGen does not stall on empty. FIFO peek returns `invalid(A)` when empty, and banked dequeue returns invalid per disabled or empty lane (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFIFO.scala:30-39`). LIFO peek/pop uses the same invalid fallback (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenLIFO.scala:28-37`). StreamIn banked read returns invalid per empty or disabled lane (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenStream.scala:84-90`). The invalid value is type-generated: fixed point maps to `FixedPoint.invalid`, float maps to `FloatPoint.invalid`, bit maps to `Bool(false,false)`, and structs recursively invalid each field (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:26-28`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFltPt.scala:32-34`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenBit.scala:21-23`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenStructs.scala:11-13`). This is value validity, not channel validity.

## Back-Pressure Decision

The omitted back-pressure is exactly the bounded ready/full contract: no producer stall when full, no write suppression from `full`, no consumer stall when empty, no atomic multi-lane transaction, no deadlock reason, and no per-cycle ready/valid handshake. ScalaGen has a limited liveness hack: streaming controls run while read streams/FIFOs are non-empty (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenController.scala:14-20`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenController.scala:31-35`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenController.scala:62-72`), but that is still value exhaustion, not back-pressure. D-22 should therefore choose between `compat_scalagen_elastic`, which preserves current goldens but allows synthesis divergence, and a bounded mode that shares one cycle policy with D-15 ParallelPipe, D-20 DelayLine advance, and D-21 II assumptions. The D-20 overlap note reaches the same dependency: unbounded queues make backpressure meaningless, while bounded mode must unify occupancy, invalid-on-empty, overflow, and deadlock diagnostics (`20 - Research Notes/50 - Decision Records/D-20-research/09-overlap-d15-d21-d22.md:16`).

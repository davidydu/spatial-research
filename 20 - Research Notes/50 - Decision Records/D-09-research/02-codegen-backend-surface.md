# D-09 Angle 2: Codegen/Backend Surface

## IR Surface

Unbiased fixed-point behavior enters the IR in two ways. The fixed-point API stages `*&` and `/&` as `UnbMul` and `UnbDiv`, with comments saying the operation probabilistically rounds up or down after multiply/divide (`argon/src/argon/lang/Fix.scala:75`, `argon/src/argon/lang/Fix.scala:80`, `argon/src/argon/lang/Fix.scala:83`, `argon/src/argon/lang/Fix.scala:87`). The saturating forms `*&!` and `/&!` stage `UnbSatMul` and `UnbSatDiv` (`argon/src/argon/lang/Fix.scala:115`, `argon/src/argon/lang/Fix.scala:122`, `argon/src/argon/lang/Fix.scala:129`). Cast syntax also has an unbiased surface: `CastFixToFix.unbiased` stages `FixToFixUnb`, and `unbsat` stages `FixToFixUnbSat` (`argon/src/argon/lang/api/Implicits.scala:204`, `argon/src/argon/lang/api/Implicits.scala:207`, `argon/src/argon/lang/api/Implicits.scala:208`). These nodes are distinct from `FixRandom`, which is its own simple-effect op (`argon/src/argon/node/Fix.scala:393`, `argon/src/argon/node/Fix.scala:394`).

## Scala And Executor

Scala codegen is the most complete host-simulation surface. `ScalaGenFixPt` emits unbiased arithmetic back to the emulation operators `*&`, `/&`, `*&!`, and `/&!` (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:96`, `src/spatial/codegen/scalagen/ScalaGenFixPt.scala:99`). For casts it emits direct calls to `FixedPoint.unbiased`, with `saturate = true` for the saturating cast (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:111`, `src/spatial/codegen/scalagen/ScalaGenFixPt.scala:114`). The same generator emits `FixRandom` through `FixedPoint.random` (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:126`, `src/spatial/codegen/scalagen/ScalaGenFixPt.scala:132`).

The executor path mirrors this. `FixResolver` handles unbiased casts by calling `FixedPoint.unbiased` (`src/spatial/executor/scala/resolvers/FixResolver.scala:45`, `src/spatial/executor/scala/resolvers/FixResolver.scala:51`) and handles `FixRandom` with `FixedPoint.random` (`src/spatial/executor/scala/resolvers/FixResolver.scala:68`, `src/spatial/executor/scala/resolvers/FixResolver.scala:74`). Arithmetic nodes such as `UnbMul` are generic `FixBinary` nodes carrying an unstaged emulation function (`argon/src/argon/node/Fix.scala:14`, `argon/src/argon/node/Fix.scala:18`, `argon/src/argon/node/Fix.scala:425`, `argon/src/argon/node/Fix.scala:444`), so `NaryResolver` runs them through `fb.unstaged(a, b)` (`src/spatial/executor/scala/resolvers/NaryResolver.scala:30`, `src/spatial/executor/scala/resolvers/NaryResolver.scala:35`). The emulation implementation is stochastic: `FixedPoint.unbiased` draws `scala.util.Random.nextFloat()` (`emul/src/emul/FixedPoint.scala:232`, `emul/src/emul/FixedPoint.scala:236`), while `FixedPoint.random` draws random bits with `scala.util.Random.nextBoolean()` (`emul/src/emul/FixedPoint.scala:247`, `emul/src/emul/FixedPoint.scala:249`). Separately, Scala bit random codegen uses `ThreadLocalRandom` (`src/spatial/codegen/scalagen/ScalaGenBit.scala:34`).

## C++, Rogue, PIR, Resources

The C++ and Rogue host generators do not preserve the full unbiased surface. C++ emits normal fixed multiply/divide at `FixMul`/`FixDiv` but has no `UnbMul`/`UnbDiv` cases in `CppGenMath`; unsupported nodes eventually reach the base no-rule failure when generation is enabled (`src/spatial/codegen/cppgen/CppGenMath.scala:28`, `src/spatial/codegen/cppgen/CppGenMath.scala:29`, `argon/src/argon/codegen/Codegen.scala:89`, `argon/src/argon/codegen/Codegen.scala:91`). C++ does recognize `FixToFixUnb`, but lowers it to the same plain cast as `FixToFix`; `FixToFixUnbSat` is just saturation then cast (`src/spatial/codegen/cppgen/CppGenMath.scala:120`, `src/spatial/codegen/cppgen/CppGenMath.scala:128`, `src/spatial/codegen/cppgen/CppGenMath.scala:129`, `src/spatial/codegen/cppgen/CppGenMath.scala:135`). Its random nodes use C `rand()` (`src/spatial/codegen/cppgen/CppGenMath.scala:57`, `src/spatial/codegen/cppgen/CppGenMath.scala:59`). Rogue is analogous: unbiased casts become assignment or saturation, while random nodes use Python `random` (`src/spatial/codegen/roguegen/RogueGenMath.scala:47`, `src/spatial/codegen/roguegen/RogueGenMath.scala:49`, `src/spatial/codegen/roguegen/RogueGenMath.scala:87`, `src/spatial/codegen/roguegen/RogueGenMath.scala:94`).

PIRGen preserves node identity instead of choosing an RNG policy. It emits `UnbMul`/`UnbDiv`/saturating forms with `genOp`, and it preserves `FixToFixUnb` with explicit input trimming (`src/spatial/codegen/pirgen/PIRGenFixPt.scala:39`, `src/spatial/codegen/pirgen/PIRGenFixPt.scala:49`). `genOp` records the operation class name and inputs (`src/spatial/codegen/pirgen/PIRGenHelper.scala:78`, `src/spatial/codegen/pirgen/PIRGenHelper.scala:84`). ResourceCountReporter only counts these as fixed ops, including random fixed nodes, without any RNG semantics (`src/spatial/codegen/resourcegen/ResourceCountReporter.scala:124`, `src/spatial/codegen/resourcegen/ResourceCountReporter.scala:127`, `src/spatial/codegen/resourcegen/ResourceCountReporter.scala:135`, `src/spatial/codegen/resourcegen/ResourceCountReporter.scala:136`).

## HLS Lowering

The hardware path is not "deterministic rounding." ChiselGen passes `Unbiased` into `Math.mul`, `Math.div`, and `Math.fix2fix` for unbiased arithmetic and casts (`src/spatial/codegen/chiselgen/ChiselGenMath.scala:34`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:40`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:77`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:78`). Fringe defines `Unbiased` as LFSR-based stochastic rounding (`fringe/src/fringe/templates/math/RoundingMode.scala:4`, `fringe/src/fringe/templates/math/RoundingMode.scala:8`). The converter implements that by instantiating `PRNG(scala.math.abs(scala.util.Random.nextInt))` and adding low PRNG bits before shaving fractional bits (`fringe/src/fringe/templates/math/Converter.scala:37`, `fringe/src/fringe/templates/math/Converter.scala:43`); the PRNG itself is deterministic from its seed (`fringe/src/fringe/templates/math/PRNG.scala:6`, `fringe/src/fringe/templates/math/PRNG.scala:16`).

## Implication

Rust simulator parity should mean "same stochastic rounding law with controllable replay," not bit-for-bit imitation of current JVM nondeterminism. Matching Scala's global `scala.util.Random` would reproduce a weak host artifact and still diverge from C++/Rogue, which mostly ignore or lack unbiased semantics. Deterministic rounding would be worse for HLS parity because Chisel/Fringe explicitly lower unbiased nodes to PRNG/LFSR rounding. The best D-09 answer is a seedable deterministic RNG in Rust, ideally with an explicit default seed for tests and an opt-in nondeterministic seed for fuzz/simulation runs. That keeps Rust reproducible while remaining semantically aligned with both the Scala executor's probabilistic behavior and the hardware lowering's PRNG-based implementation.

# Current FixedPoint.unbiased Algorithm and Call Surface

## Decision Frame

D-09 / Q-118 is the Rust simulator determinism question: should Rust reproduce the current JVM simulator's stochastic unbiased rounding, expose seedable stochastic rounding, or move to deterministic rounding (`20 - Research Notes/40 - Decision Queue.md:43-45`, `20 - Research Notes/20 - Open Questions.md:1590-1602`). D-17 / Q-144 overlaps at the canonical-semantics layer, so this note treats cross-backend policy as D-17 territory and focuses on what Rust would need to match in today's JVM/emul behavior (`20 - Research Notes/40 - Decision Queue.md:75-77`, `20 - Research Notes/20 - Open Questions.md:2052-2062`).

## Algorithm and Signed Behavior

`FixedPoint.unbiased(bits, valid, fmt, saturate = false)` says its `bits` argument already has four extra fractional bits beyond the normal fixed-point representation, and its only policy knob is the `saturate` Boolean; there is no RNG or seed parameter in the call signature (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:224-232`). The implementation drops those four guard bits into `biased = bits >> 4`, recovers a 1/16-granularity fractional residue with `(bits & 0xF).toFloat / 16.0f`, draws `scala.util.Random.nextFloat()`, and adds the draw to the residue (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:233-237`). If that sum is at least one, nonnegative `biased` values become `biased + 1`, negative `biased` values become `biased - 1`, and all other cases keep `biased` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:238`). The final value is wrapped through `FixedPoint.clamped` unless `saturate` is true, in which case it goes through `FixedPoint.saturating` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:239-240`).

The signed range comes from `FixFormat`: signed formats use `[-2^(ibits+fbits-1), 2^(ibits+fbits-1)-1]`, while unsigned formats use `[0, 2^(ibits+fbits)-1]` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixFormat.scala:10-11`). The wrapping path sign-extends when the format is signed and the format-width sign bit is set; otherwise it masks with `MAX_VALUE`, and the constructed `FixedPoint` preserves the passed `valid` bit (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:197-208`). The saturating path compares the rounded integer against `fmt.MIN_VALUE` and `fmt.MAX_VALUE`, returns the prebuilt min/max fixed-point constants on overflow, and only delegates to `clamped` when the value is in range (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:211-221`). For negative rounded bases, the random branch explicitly steps farther negative (`biased - 1`) rather than toward zero, while the residue calculation itself is still the same low-nibble calculation used for all signs (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:234-238`).

## Extra Fractional Bits

The unbiased multiply operators manufacture the four guard bits by shifting each operand left by two before multiplying, then shifting the product back by `fmt.fbits`; the non-saturating form calls `FixedPoint.unbiased(..., fmt)` and the saturating form calls the same helper with `saturate = true` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:52-60`). The unbiased divide operators instead shift the numerator by `fmt.fbits + 4` before division, again passing either the default wrapping mode or `saturate = true` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:55-63`). Both divide forms are wrapped in `valueOrX`, whose catch-all fallback returns `FixedPoint.invalid(fmt)`, so divide exceptions such as divide-by-zero become X values on these paths (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:55-63`, `/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:102-104`). By contrast, fixed-to-fixed unbiased casts pass the source `.value` directly into `FixedPoint.unbiased`; they do not locally add guard bits in Scalagen or the executor resolver (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:111-114`, `/Users/david/Documents/David_code/spatial/src/spatial/executor/scala/resolvers/FixResolver.scala:45-51`).

## Call Surface

At the user DSL layer, `*&` and `/&` stage `UnbMul` and `UnbDiv`, while `*&!` and `/&!` stage `UnbSatMul` and `UnbSatDiv` (`/Users/david/Documents/David_code/spatial/argon/src/argon/lang/Fix.scala:75-87`, `/Users/david/Documents/David_code/spatial/argon/src/argon/lang/Fix.scala:115-129`). Those IR nodes are `FixBinary` nodes backed by the emul operators `_*&_`, `_/&_`, `_*&!_`, and `_/&!_`; `FixBinary` stores an unstaged `(FixedPoint, FixedPoint) => FixedPoint` implementation, and generic binary rewriting can call it for constant operands (`/Users/david/Documents/David_code/spatial/argon/src/argon/node/Fix.scala:14-17`, `/Users/david/Documents/David_code/spatial/argon/src/argon/node/Fix.scala:425-444`, `/Users/david/Documents/David_code/spatial/argon/src/argon/Binary.scala:34-35`). Scalagen emits generated Scala that calls exactly `$x *& $y`, `$x /& $y`, `$x *&! $y`, and `$x /&! $y` for the four arithmetic nodes (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:96-99`). The Scala executor's generic `NaryResolver` evaluates binary nodes by fetching both inputs and calling `fb.unstaged(a, b)`, so these arithmetic nodes reach the same emul operators outside generated Scala (`/Users/david/Documents/David_code/spatial/src/spatial/executor/scala/resolvers/NaryResolver.scala:30-35`). Casts enter through `toUnbiased` and `toUnbSat`, which dispatch to `cast.unbiased` and `cast.unbsat`; the fixed-to-fixed cast implementation stages `FixToFixUnb` and `FixToFixUnbSat` (`/Users/david/Documents/David_code/spatial/argon/src/argon/lang/api/Implicits.scala:250-254`, `/Users/david/Documents/David_code/spatial/argon/src/argon/lang/api/Implicits.scala:204-208`). One caveat: constant `FixToFixUnb` and `FixToFixUnbSat` rewrites use deterministic `c.toFixedPoint(...)`, not `FixedPoint.unbiased(...)` (`/Users/david/Documents/David_code/spatial/argon/src/argon/node/Fix.scala:343-357`).

## D-09 Implications

The current JVM surface is seedless stochastic rounding at every runtime call to `FixedPoint.unbiased`, because the helper draws from `scala.util.Random.nextFloat()` internally and accepts no RNG state (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:232-237`). Exact Rust parity would therefore mean reproducing the current distribution and signed branch behavior, including the four-guard-bit contract and negative `biased - 1` step, but not necessarily a reproducible cross-run sequence unless Rust intentionally mirrors JVM global RNG state. A seedable Rust helper would preserve stochastic semantics while making simulator tests reproducible. Deterministic rounding would be a semantic change beyond Rust ergonomics, so it should be decided under D-17/Q-144 and then implemented consistently in Rust, JVM emul, and generated-reference tests.

# ExhaustiveBanking Search Mechanics

## Entry Path and Directive Order

`ExhaustiveBanking.bankAccesses` is the strategy entrypoint: it receives grouped reads/writes plus precomputed `BankingOptions`, rewrites dephased accesses, separates host reads, records mappings back to original `AccessMatrix` values, and then calls `findSchemes` on sparse-matrix groups (`src/spatial/traversal/banking/ExhaustiveBanking.scala:54`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:327`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:331`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:335`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:356`). The candidate directive order is built one layer up: `MemoryConfigurer` forms the Cartesian product of banking views, `N` strictness, alpha strictness, and duplication/regroup options, sorts by each component priority, filters illegal flat/regroup mixes, then moves non-relaxed same-priority choices ahead of other non-relaxed choices and relaxed choices (`src/spatial/traversal/banking/MemoryConfigurer.scala:615`, `src/spatial/traversal/banking/MemoryConfigurer.scala:619`, `src/spatial/traversal/banking/MemoryConfigurer.scala:620`, `src/spatial/traversal/banking/MemoryConfigurer.scala:628`, `src/spatial/traversal/banking/MemoryConfigurer.scala:630`). By default, non-explicit banking tries flat and hierarchical views, `NBestGuess` before `NRelaxed`, and `AlphaBestGuess` before `AlphaRelaxed` (`src/spatial/traversal/banking/MemoryConfigurer.scala:25`, `src/spatial/traversal/banking/MemoryConfigurer.scala:34`, `src/spatial/traversal/banking/MemoryConfigurer.scala:38`).

## Projection, Regrouping, and Cache

For each directive, a flat view searches one axes list containing every rank dimension, while hierarchical banking expands into one search per selected dimension (`src/spatial/metadata/memory/BankingData.scala:518`, `src/spatial/metadata/memory/BankingData.scala:520`, `src/spatial/metadata/memory/BankingData.scala:523`, `src/spatial/metadata/memory/BankingData.scala:525`). Each per-axis search slices full access matrices into an `AccessView`, then, for partial-rank views, `repackageGroup` may split projections when complement dimensions already disambiguate conflicts (`src/spatial/traversal/banking/ExhaustiveBanking.scala:93`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:105`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:119`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:143`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:255`). Reads can be omitted from the actual conflict set when the axes are entirely in the regroup/duplication dimensions, which means that strategy searches can intentionally produce duplicates rather than one shared read banking (`src/spatial/traversal/banking/ExhaustiveBanking.scala:257`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:279`).

The per-axis answer is memoized in `solutionCache` by regrouped access sets, `N` strictness, alpha strictness, and axes (`src/spatial/traversal/banking/ExhaustiveBanking.scala:45`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:269`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:270`). That speeds repeated patterns, but the key does not include memory flags such as `onlyBlockCyclic`, `noBlockCyclic`, `blockCyclicBs`, or staged dimensions; if a strategy instance is reused across memories with the same regrouped access shape but different banking constraints, stale reuse is a plausible risk (unverified). There are also unit shortcuts: singleton memories, groups with fewer than two accesses, and simple line-buffer cases can bypass the exhaustive loop (`src/spatial/traversal/banking/ExhaustiveBanking.scala:202`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:206`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:207`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:271`).

## Inner Search Loop

`findBanking` first handles forced full banking of a dimension: if `fullyBankDims` contains the axis head, it returns a full-dimension `ModBanking` immediately, without checking conflicts (`src/spatial/traversal/banking/ExhaustiveBanking.scala:367`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:371`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:372`). Otherwise, `Nmin` is the larger of write-group width and read-group width, with read width halved only on the first dual-ported read search; `Ncap` is the product of staged dimensions, at least `Nmin` (`src/spatial/traversal/banking/ExhaustiveBanking.scala:374`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:376`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:377`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:378`). `NStrictness.expand` then supplies the ordered `N` list: best-guess uses access-count factors, staged-dimension factors, powers of two, and Mersenne-like values; relaxed enumerates the full range (`src/spatial/metadata/memory/BankingData.scala:560`, `src/spatial/metadata/memory/BankingData.scala:562`, `src/spatial/metadata/memory/BankingData.scala:568`).

The loop order is `N`, then alpha vectors, then optionally block size `B` (`src/spatial/traversal/banking/ExhaustiveBanking.scala:392`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:393`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:394`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:405`). Alpha best-guess builds vectors from factors of `N`, products of staged dimensions, coprimes, and easy powers/sums of powers, then sorts distinct candidates by sum; relaxed enumerates all residues modulo `N` (`src/spatial/metadata/memory/BankingData.scala:618`, `src/spatial/metadata/memory/BankingData.scala:623`, `src/spatial/metadata/memory/BankingData.scala:624`, `src/spatial/metadata/memory/BankingData.scala:631`). Search stops after `numSchemesPerRegion` successes or `bankingTimeout` alpha attempts, whose defaults are two schemes and 50,000 attempts (`src/spatial/traversal/banking/ExhaustiveBanking.scala:390`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:391`, `src/spatial/SpatialConfig.scala:36`, `src/spatial/SpatialConfig.scala:37`).

## Conflict Gates and Candidate Counts

The cyclic gate is tried before block-cyclic when allowed. For ordinary groups, `checkCyclic` requires every access pair to have an empty polyhedral domain for equal-bank conflicts; for first dual-ported reader search, it checks triplets and accepts if at least one pair in the triplet is separable (`src/spatial/traversal/banking/ExhaustiveBanking.scala:409`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:443`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:447`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:454`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:456`). Block-cyclic search is gated by `noBlockCyclic`/explicit `B`, scans `explicitBs` or default `blockCyclicBs`, and validates pairs through an emptiness check over four inequalities (`src/spatial/traversal/banking/ExhaustiveBanking.scala:416`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:417`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:462`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:464`, `src/spatial/metadata/memory/package.scala:61`). `numChecks` is a diagnostic/product-cost proxy based on pair counts, or triplet counts for dual-ported reads (`src/spatial/traversal/banking/ExhaustiveBanking.scala:382`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:404`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:431`).

`computeP` is not a conflict gate; it computes legal offset-region periods after a candidate passes a cyclic or block-cyclic gate (`src/spatial/traversal/banking/ExhaustiveBanking.scala:411`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:421`). The helper derives raw periods from `N*B/gcd(N*B, alpha_i)`, expands by divisors and sometimes staged dimensions, then filters combinations by volume and by whether the candidate spans all banks (`utils/src/utils/math/package.scala:134`, `utils/src/utils/math/package.scala:193`, `utils/src/utils/math/package.scala:194`, `utils/src/utils/math/package.scala:195`, `utils/src/utils/math/package.scala:198`, `utils/src/utils/math/package.scala:202`). This matters for HLS-style pruning because accepting only partition forms is not enough; the offset period still has to exist for Spatial's `ModBanking` representation.

## Failure Modes for D-07

Effort gates can prune whole directive regions after earlier successes: effort 0 stops after any found scheme, effort 1 allows only one solution per exact `(view, regroup)` and only no-regroup/full-regroup forms, and effort 2 allows at most two per `(view, regroup)` (`src/spatial/traversal/banking/ExhaustiveBanking.scala:181`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:188`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:189`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:190`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:194`). The final validation hook is effectively disabled: all `fullOpts` are partitioned with `case opt => true`, despite the nearby `isValidBanking` helper (`src/spatial/traversal/banking/ExhaustiveBanking.scala:298`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:302`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:360`). Forced explicit banking also bypasses normal rewriting and safety search, returning explicit schemes split by dispatch, with an error for host reads plus multiple explicit duplicates (`src/spatial/traversal/banking/ExhaustiveBanking.scala:64`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:211`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:212`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:221`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:231`). For D-07, reusing Spatial's search means inheriting these alpha/`N`/`B` mechanics; constraining it to HLS partition forms would likely mean cutting candidate generation and directive order while preserving the conflict and `computeP` gates.

# D-07 research angle 6: tests/app usage

## Scope

The test/app evidence separates three obligations. Semantic correctness is where a directive or conflict waiver is needed for the program to return its gold value. Performance expectation is where users constrain the banking search to get an affordable or findable physical plan. Legacy Chisel/Spatial diagnostic behavior is where tests assert exact compiler artifacts such as SRAM counts, mux ports, or search reports. The source API is explicitly an affine banking model: `ModBanking` is documented as `(alpha*A / B) mod N`, with `B == 1` printed as cyclic and `B > 1` as block-cyclic (`src/spatial/metadata/memory/BankingData.scala:49`, `src/spatial/metadata/memory/BankingData.scala:62`, `src/spatial/metadata/memory/BankingData.scala:67`). HLS block/cyclic/complete partition forms are a narrower public target model (unverified), so D-07 should treat tests as evidence about promises, not as a demand to preserve every Spatial search degree of freedom.

## Directive Surface

The source-level directives are user-facing. `hierarchical` and `flat` select the banking view (`src/spatial/lang/SRAM.scala:61`, `src/spatial/lang/SRAM.scala:63`); `fullfission`, `axesfission`, and `nofission` constrain duplication (`src/spatial/lang/SRAM.scala:83`, `src/spatial/lang/SRAM.scala:90`, `src/spatial/lang/SRAM.scala:93`). `noblockcyclic`, `onlyblockcyclic`, and `blockcyclic_Bs` directly expose block/cyclic search choices (`src/spatial/lang/SRAM.scala:104`, `src/spatial/lang/SRAM.scala:106`, `src/spatial/lang/SRAM.scala:108`). `effort(e)` is a performance/search-budget knob, mirrored by CLI `--bankingEffort`, `--numSchemesPerRegion`, and `--bankingTimeout` (`src/spatial/lang/SRAM.scala:110`, `src/spatial/Spatial.scala:372`, `src/spatial/Spatial.scala:382`, `src/spatial/Spatial.scala:400`). `bank` is checked explicit banking, while `forcebank` sets `forceExplicitBanking` and skips safety rejection by design (`src/spatial/lang/SRAM.scala:117`, `src/spatial/lang/SRAM.scala:119`, `src/spatial/metadata/memory/package.scala:121`). `fullybankdim` and `fullybanked` encode complete-style banking; the latter says it is intended for correctness debugging (`src/spatial/lang/SRAM.scala:122`, `src/spatial/lang/SRAM.scala:125`).

## Correctness Cases

Several tests make banking part of correctness. `Bank1D` compares checked `.bank` with unchecked `.forcebank` and still asserts the numerical gold, so explicit banking syntax is semantic at the DSL boundary even if the exact backend implementation can change (`test/spatial/tests/feature/banking/Bank1D.scala:39`, `test/spatial/tests/feature/banking/Bank1D.scala:40`, `test/spatial/tests/feature/banking/Bank1D.scala:57`). `Cyclic1D` forces one SRAM through block-cyclic search and another through non-block-cyclic search, then requires both to produce the same reduction result (`test/spatial/tests/feature/banking/Bank1D.scala:94`, `test/spatial/tests/feature/banking/Bank1D.scala:97`, `test/spatial/tests/feature/banking/Bank1D.scala:106`). `ManualConflictDephasing` is the clearest unbankable waiver: `.conflictable` is annotated “Should be unbankable without this flag,” and the test says the user-promised sleep/dephasing is required for the right answer (`test/spatial/tests/feature/banking/ManualConflictDephasing.scala:11`, `test/spatial/tests/feature/banking/ManualConflictDephasing.scala:16`, `test/spatial/tests/feature/banking/ManualConflictDephasing.scala:27`). `ContinualStreaming` uses `.fullybanked` and deliberately issues two reads on alternate cycles to force banking before checking gold output (`test/spatial/tests/compiler/ContinualStreaming.scala:13`, `test/spatial/tests/compiler/ContinualStreaming.scala:37`, `test/spatial/tests/compiler/ContinualStreaming.scala:50`).

## Diagnostics And Search Expectations

Some tests are Spatial diagnostics rather than HLS semantic contracts. `ModuloIndexing` has real value checks for modulo/divide indexing, but its `checkIR` additionally requires exactly four `SRAMNew`s “if banking is all correct”; that exact count is legacy IR behavior (`test/spatial/tests/feature/banking/ModuloIndexing.scala:20`, `test/spatial/tests/feature/banking/ModuloIndexing.scala:55`, `test/spatial/tests/feature/banking/ModuloIndexing.scala:63`). `BankNonbuffered` similarly checks output values, then inspects instance depth and read mux ports (`test/spatial/tests/feature/banking/BankNonbuffered.scala:26`, `test/spatial/tests/feature/banking/BankNonbuffered.scala:38`, `test/spatial/tests/feature/banking/BankNonbuffered.scala:44`). `Bankapalooza` is explicitly a directive sampler, covering hierarchical, flat, high effort, full fission, axes fission, and block-cyclic variants before one aggregate gold check (`test/spatial/tests/feature/banking/Bank4DHard.scala:35`, `test/spatial/tests/feature/banking/Bank4DHard.scala:87`, `test/spatial/tests/feature/banking/Bank4DHard.scala:95`, `test/spatial/tests/feature/banking/Bank4DHard.scala:119`, `test/spatial/tests/feature/banking/Bank4DHard.scala:145`).

## App Implications

Apps show why multi-bank memories must remain expressible. `HelloSpatial` uses parallel DRAM load/store and gather/scatter over SRAM aliases (`apps/src/HelloSpatial.scala:43`, `apps/src/HelloSpatial.scala:46`, `apps/src/HelloSpatial.scala:49`, `apps/src/HelloSpatial.scala:51`). `Inference` marks convolution kernel and local window SRAMs hierarchical, and later rewrites local access “to make banking happy and easy” (`test/spatial/tests/apps/Inference.scala:86`, `test/spatial/tests/apps/Inference.scala:98`, `test/spatial/tests/apps/Inference.scala:475`). MachSuite uses effort as tuning: AES requests `RegFile(...).buffer.effort(2)`, molecular dynamics uses hierarchical 4D/3D SRAMs, and FFT chooses `effort(0)` for work buffers (`test/spatial/tests/apps/MachSuite.scala:90`, `test/spatial/tests/apps/MachSuite.scala:1170`, `test/spatial/tests/apps/MachSuite.scala:2742`). Recommendation: preserve semantic directives, conflict diagnostics, and enough block/cyclic/complete planning to satisfy app bandwidth; treat full Spatial alpha/N/B exhaustive search and exact Chisel artifact checks as legacy diagnostics unless HLS gains a matching partition representation.

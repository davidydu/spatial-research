# Banking Metadata Surface

## Core Address Model

D-07 / Q-112 is choosing whether an HLS banking-search strategy should reuse Spatial's existing alpha/N/B search, constrain it to HLS partition forms, or replace it. The core local abstraction is deliberately broader than a directive surface: every `Banking` records `nBanks`, `stride`, selected `axes`, `alphas`, `Ps`, diagnostic volumes, and a generated `bankSelect` function (`src/spatial/metadata/memory/BankingData.scala:17`, `src/spatial/metadata/memory/BankingData.scala:19`, `src/spatial/metadata/memory/BankingData.scala:27`). The concrete implementation is `ModBanking(N, B, alpha, axes, P)`, documented as `(alpha*A / B) mod N` and implemented by summing `alpha_i * addr_i`, dividing by `B`, then reducing modulo `N` (`src/spatial/metadata/memory/BankingData.scala:49`, `src/spatial/metadata/memory/BankingData.scala:50`, `src/spatial/metadata/memory/BankingData.scala:62`, `src/spatial/metadata/memory/BankingData.scala:64`).

This means Spatial can express cyclic banking when `B == 1` and block-cyclic banking when `B > 1`; the `toString` labels those cases directly (`src/spatial/metadata/memory/BankingData.scala:66`, `src/spatial/metadata/memory/BankingData.scala:67`, `src/spatial/metadata/memory/BankingData.scala:68`). `P` is not just decorative: it contributes to hidden-volume estimation and offset layout, while padding is stored on `Memory` alongside banking, depth, and accumulator type (`src/spatial/metadata/memory/BankingData.scala:55`, `src/spatial/metadata/memory/BankingData.scala:57`, `src/spatial/metadata/memory/BankingData.scala:180`, `src/spatial/metadata/memory/BankingData.scala:183`).

## Views And Offsets

The metadata already distinguishes flat and hierarchical banking. A physical `Memory` carries a `Seq[Banking]`; printing calls one scheme "Flat" and multiple schemes "Hierarchical" (`src/spatial/metadata/memory/BankingData.scala:159`, `src/spatial/metadata/memory/BankingData.scala:181`). `Flat(rank)` expands all dimensions into one grouped address, while `Hierarchical(rank)` expands each dimension separately unless given a selected view (`src/spatial/metadata/memory/BankingData.scala:510`, `src/spatial/metadata/memory/BankingData.scala:518`, `src/spatial/metadata/memory/BankingData.scala:520`, `src/spatial/metadata/memory/BankingData.scala:523`, `src/spatial/metadata/memory/BankingData.scala:526`). Offset generation reinforces that only two layouts are first-class: one flat scheme over all dimensions, or one scheme per sparse dimension. Mixed arbitrary dimension groupings throw an exception (`src/spatial/metadata/memory/BankingData.scala:208`, `src/spatial/metadata/memory/BankingData.scala:212`, `src/spatial/metadata/memory/BankingData.scala:226`, `src/spatial/metadata/memory/BankingData.scala:250`, `src/spatial/metadata/memory/BankingData.scala:252`).

## Search Option Space

`BankingOptions` is a four-axis search descriptor: `view`, `N`, `alpha`, and `regroup`; options are marked undesirable when either N or alpha uses relaxed search (`src/spatial/metadata/memory/BankingData.scala:493`, `src/spatial/metadata/memory/BankingData.scala:494`, `src/spatial/metadata/memory/BankingData.scala:495`). `NStrictness` can be user-defined, powers of two, a best-guess set derived from access counts, staged dimensions, powers, and Mersenne-adjacent values, or fully relaxed over `min to max` (`src/spatial/metadata/memory/BankingData.scala:532`, `src/spatial/metadata/memory/BankingData.scala:537`, `src/spatial/metadata/memory/BankingData.scala:542`, `src/spatial/metadata/memory/BankingData.scala:547`, `src/spatial/metadata/memory/BankingData.scala:560`, `src/spatial/metadata/memory/BankingData.scala:568`).

`AlphaStrictness` is similarly tiered: user-defined vectors, powers-of-two-like candidates, best-guess candidates from N factors, staged-dimension products, coprimes, and easy sums, or relaxed modulo candidates (`src/spatial/metadata/memory/BankingData.scala:571`, `src/spatial/metadata/memory/BankingData.scala:581`, `src/spatial/metadata/memory/BankingData.scala:593`, `src/spatial/metadata/memory/BankingData.scala:598`, `src/spatial/metadata/memory/BankingData.scala:606`, `src/spatial/metadata/memory/BankingData.scala:619`, `src/spatial/metadata/memory/BankingData.scala:628`). Candidate alpha vectors are filtered by coprimality (`src/spatial/metadata/memory/BankingData.scala:583`, `src/spatial/metadata/memory/BankingData.scala:586`, `src/spatial/metadata/memory/BankingData.scala:588`). `RegroupDims` adds an access-grouping/duplication dimension, with helpers for all subsets, all dimensions, or none (`src/spatial/metadata/memory/BankingData.scala:498`, `src/spatial/metadata/memory/BankingData.scala:501`, `src/spatial/metadata/memory/BankingData.scala:502`, `src/spatial/metadata/memory/BankingData.scala:504`, `src/spatial/metadata/memory/BankingData.scala:506`).

## User Controls

The metadata surface can narrow the search without changing the representation. Users can restrict N and alpha families, disable or require block-cyclic search, and provide the candidate `B` list for block-cyclic banking (`src/spatial/metadata/memory/BankingData.scala:353`, `src/spatial/metadata/memory/BankingData.scala:361`, `src/spatial/metadata/memory/BankingData.scala:363`, `src/spatial/metadata/memory/BankingData.scala:369`, `src/spatial/metadata/memory/BankingData.scala:371`, `src/spatial/metadata/memory/BankingData.scala:377`, `src/spatial/metadata/memory/BankingData.scala:379`, `src/spatial/metadata/memory/BankingData.scala:385`, `src/spatial/metadata/memory/BankingData.scala:387`, `src/spatial/metadata/memory/BankingData.scala:393`). Explicit schemes can directly specify `Ns`, `Bs`, `Alphas`, and optional `Ps`, and `forceExplicitBanking` can request using an explicit scheme even if unsafe (`src/spatial/metadata/memory/BankingData.scala:637`, `src/spatial/metadata/memory/BankingData.scala:644`, `src/spatial/metadata/memory/BankingData.scala:645`, `src/spatial/metadata/memory/BankingData.scala:656`, `src/spatial/metadata/memory/BankingData.scala:662`). Complete-style intent is represented separately as `FullyBankDims` (`src/spatial/metadata/memory/BankingData.scala:647`, `src/spatial/metadata/memory/BankingData.scala:654`). Some flag comments/names should be checked before exposing them directly: for example, `NoHierarchicalBank`'s comment says it disables flattened banking, while `NoFlatBank`'s comment says it disables banking (`src/spatial/metadata/memory/BankingData.scala:343`, `src/spatial/metadata/memory/BankingData.scala:351`, `src/spatial/metadata/memory/BankingData.scala:424`, `src/spatial/metadata/memory/BankingData.scala:432`).

## HLS Fit

Standard HLS partition surfaces are usually dimension, kind, and factor choices such as complete, block, or cyclic (unverified). The natural subset is therefore hierarchical, per-dimension banking with simple `N` factors, `B == 1` for cyclic, unit or trivial alpha, and `FullyBankDims` for complete partition intent (`src/spatial/metadata/memory/BankingData.scala:523`, `src/spatial/metadata/memory/BankingData.scala:526`, `src/spatial/metadata/memory/BankingData.scala:537`, `src/spatial/metadata/memory/BankingData.scala:542`, `src/spatial/metadata/memory/BankingData.scala:647`, `src/spatial/metadata/memory/BankingData.scala:654`). Flat banking can map if the generated HLS memory is physically flattened first, but it is less natural for source-level multidimensional array partition directives (unverified) (`src/spatial/metadata/memory/BankingData.scala:518`, `src/spatial/metadata/memory/BankingData.scala:520`). Non-unit alpha vectors, block-cyclic `B > 1`, arbitrary `P`, and regroup-driven duplication are the Spatial-specific superset: they are valuable search expressivity, but they do not look like portable HLS partition directives without either custom address generation or replicated arrays (unverified) (`src/spatial/metadata/memory/BankingData.scala:50`, `src/spatial/metadata/memory/BankingData.scala:64`, `src/spatial/metadata/memory/BankingData.scala:67`, `src/spatial/metadata/memory/BankingData.scala:393`, `src/spatial/metadata/memory/BankingData.scala:506`).

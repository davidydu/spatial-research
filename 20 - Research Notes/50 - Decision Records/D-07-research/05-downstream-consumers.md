# Downstream Consumers of Banking Choices

## 1. Minimum Planner Output

Downstream passes do not consume a bare partition factor. They consume a physical-memory plan: one or more `Memory` duplicates, each with `Seq[Banking]`, buffer `depth`, `padding`, and accumulator type (`src/spatial/metadata/memory/BankingData.scala:180`, `src/spatial/metadata/memory/BankingData.scala:184`). Each `Banking` exposes `nBanks`, `stride`, `axes`, `alphas`, `Ps`, and `bankSelect`; `ModBanking` implements `(alpha dot addr / B) mod N` (`src/spatial/metadata/memory/BankingData.scala:17`, `src/spatial/metadata/memory/BankingData.scala:27`, `src/spatial/metadata/memory/BankingData.scala:49`, `src/spatial/metadata/memory/BankingData.scala:64`). `Memory.bankSelects` and `bankOffset` only fully support flat one-scheme banking or one scheme per sparse dimension; arbitrary mixed groupings throw (`src/spatial/metadata/memory/BankingData.scala:202`, `src/spatial/metadata/memory/BankingData.scala:212`, `src/spatial/metadata/memory/BankingData.scala:226`, `src/spatial/metadata/memory/BankingData.scala:250`). Therefore an HLS planner reusing current lowering must emit those shapes, or replace every bank/offset consumer.

The planner must also produce access metadata. `Dispatch` maps unrolled uids to duplicate ids, with writers allowed multiple destinations and readers intended to have at most one (`src/spatial/metadata/memory/BankingData.scala:289`, `src/spatial/metadata/memory/BankingData.scala:293`). `Ports` maps dispatch id and uid to `Port(bufferPort, muxPort, muxOfs, castgroup, broadcast)` (`src/spatial/metadata/memory/BankingData.scala:99`, `src/spatial/metadata/memory/BankingData.scala:323`). `GroupId` records banking-analysis group identity (`src/spatial/metadata/memory/BankingData.scala:306`, `src/spatial/metadata/memory/BankingData.scala:310`). The spec frames HLS lowering as mapping `Memory` duplicates and `ModBanking` choices into array partitioning, banking, and buffering pragmas rather than rediscovering layout (`10 - Spec/20 - Semantics/40 - Memory Semantics.md:67`, `10 - Spec/20 - Semantics/40 - Memory Semantics.md:69`).

## 2. Transform Consumers

`MemoryUnrolling` is the first hard consumer. It duplicates local memories from `mem.duplicates`, installs each as `mem2.instance`, and registers original-to-duplicate mapping by dispatch id (`src/spatial/transform/unrolling/MemoryUnrolling.scala:67`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:73`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:83`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:87`). For each access it reads `dispatch`, `gid`, and `port`, rejects loads with more than one dispatch, groups lanes by duplicate and `(bufferPort, muxPort)`, then forms vector chunks from contiguous `muxOfs` values (`src/spatial/transform/unrolling/MemoryUnrolling.scala:416`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:420`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:425`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:456`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:460`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:464`). It computes bank and offset vectors through the selected `Memory` before staging banked access nodes (`src/spatial/transform/unrolling/MemoryUnrolling.scala:236`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:237`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:509`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:544`).

Two repair/reshaping passes also assume this metadata is editable and uid-stable. `EarlyUnroller` filters and rewrites dispatches, ports, and group ids when it clones accesses under a lane prefix (`src/spatial/transform/streamify/EarlyUnroller.scala:250`, `src/spatial/transform/streamify/EarlyUnroller.scala:256`, `src/spatial/transform/streamify/EarlyUnroller.scala:259`, `src/spatial/transform/streamify/EarlyUnroller.scala:265`). `BufferRecompute` recomputes buffer ports, updates duplicate depths, and bumps conflicting mux ports after later transforms (`src/spatial/traversal/BufferRecompute.scala:34`, `src/spatial/traversal/BufferRecompute.scala:38`, `src/spatial/traversal/BufferRecompute.scala:42`, `src/spatial/traversal/BufferRecompute.scala:46`). These are compiler-level consumers, not Chisel-specific.

## 3. Cost, Area, DSE, Reports

The current planner's cost model depends on banking equations, not only bank counts. `MemoryConfigurer.cost` builds read/write mux-width histograms using `bankMuxWidth(nBanks, stride, alpha)`, passes `B`, `N`, `alpha`, `P`, depth, and histograms into the area estimator, and adds auxiliary arithmetic costs for SRAM address calculations (`src/spatial/traversal/banking/MemoryConfigurer.scala:448`, `src/spatial/traversal/banking/MemoryConfigurer.scala:451`, `src/spatial/traversal/banking/MemoryConfigurer.scala:461`, `src/spatial/traversal/banking/MemoryConfigurer.scala:467`, `src/spatial/traversal/banking/MemoryConfigurer.scala:469`, `src/spatial/traversal/banking/MemoryConfigurer.scala:479`). `AreaEstimator`'s crude fallback partitions direct versus crossbar accesses from that histogram, penalizes non-power-of-two offset, bank, mux, and size terms, and its ML path pads the same `B/N/alpha/P/hist/depth` feature vector (`models/src/models/AreaEstimator.scala:81`, `models/src/models/AreaEstimator.scala:89`, `models/src/models/AreaEstimator.scala:93`, `models/src/models/AreaEstimator.scala:97`, `models/src/models/AreaEstimator.scala:127`, `models/src/models/AreaEstimator.scala:156`, `models/src/models/AreaEstimator.scala:170`).

Area and DSE consumers rely on duplicate and bank depth. `MemoryAllocator` iterates duplicates, computes raw resource area and bank depth, and assigns `resourceType` when capacity and minimum-depth constraints pass (`src/spatial/traversal/MemoryAllocator.scala:38`, `src/spatial/traversal/MemoryAllocator.scala:62`, `src/spatial/traversal/MemoryAllocator.scala:83`, `src/spatial/traversal/MemoryAllocator.scala:85`, `src/spatial/traversal/MemoryAllocator.scala:87`). `AreaModel.rawMemoryArea` multiplies resources per bank by total banks and buffer depth (`src/spatial/targets/AreaModel.scala:145`, `src/spatial/targets/AreaModel.scala:146`, `src/spatial/targets/AreaModel.scala:147`, `src/spatial/targets/AreaModel.scala:149`, `src/spatial/targets/AreaModel.scala:153`). DSE validates points by comparing evaluated area to capacity (`src/spatial/dse/DSEThread.scala:119`, `src/spatial/dse/DSEThread.scala:120`). Reports consume the same facts: `MemoryReporter` prints duplicates, banking, padding, ports, direct-banking status, and broadcast metadata (`src/spatial/report/MemoryReporter.scala:44`, `src/spatial/report/MemoryReporter.scala:53`, `src/spatial/report/MemoryReporter.scala:61`, `src/spatial/report/MemoryReporter.scala:70`, `src/spatial/report/MemoryReporter.scala:81`, `src/spatial/report/MemoryReporter.scala:82`). HLS can replace the cost function with HLS-native partition estimates (unverified), but still needs equivalent depth, duplicate, port, and reportable partition facts if these consumers remain.

## 4. Codegen Consumers

Scalagen consumes bank/offset expressions directly. Initialized memories are grouped by `inst.bankSelects` and filled by `inst.bankOffset`; generated loads and stores call `mem.apply`/`mem.update` with bank vectors and offsets (`src/spatial/codegen/scalagen/ScalaGenMemories.scala:133`, `src/spatial/codegen/scalagen/ScalaGenMemories.scala:134`, `src/spatial/codegen/scalagen/ScalaGenMemories.scala:135`, `src/spatial/codegen/scalagen/ScalaGenMemories.scala:164`, `src/spatial/codegen/scalagen/ScalaGenMemories.scala:169`, `src/spatial/codegen/scalagen/ScalaGenMemories.scala:179`, `src/spatial/codegen/scalagen/ScalaGenMemories.scala:185`). PIR consumes a smaller but still explicit contract: memory state includes depth, padded dims, and bank counts, while banked reads/writes emit `.bank`, `.offset`, `.port`, `.muxPort`, `.broadcast`, and `.castgroup` (`src/spatial/codegen/pirgen/PIRGenHelper.scala:30`, `src/spatial/codegen/pirgen/PIRGenHelper.scala:31`, `src/spatial/codegen/pirgen/PIRGenHelper.scala:32`, `src/spatial/codegen/pirgen/PIRGenHelper.scala:60`, `src/spatial/codegen/pirgen/PIRGenHelper.scala:71`, `src/spatial/codegen/pirgen/PIRGenHelper.scala:72`, `src/spatial/codegen/pirgen/PIRGenSRAM.scala:16`, `src/spatial/codegen/pirgen/PIRGenSRAM.scala:17`). These are not Chisel-specific, although PIR has Plasticine legality constraints and vector-size assertions (`src/spatial/traversal/banking/MemoryConfigurer.scala:165`, `src/spatial/traversal/banking/MemoryConfigurer.scala:174`, `src/spatial/codegen/pirgen/PIRGenHelper.scala:14`, `src/spatial/codegen/pirgen/PIRGenHelper.scala:18`).

## 5. Chisel-Specific Boundary

The Chisel backend and Fringe memory templates are the safest Chisel-specific classification. `ChiselGenMem` converts each duplicate into `MemParams`: padded dimensions, `inst.nBanks`, `inst.Bs`, `inst.Ps`, widths, and per-access `Access` objects containing hash, `muxPort`, `muxOfs`, `castgroup`, `broadcast`, `bufferPort`, access width, and residual generators (`src/spatial/codegen/chiselgen/ChiselGenMem.scala:162`, `src/spatial/codegen/chiselgen/ChiselGenMem.scala:163`, `src/spatial/codegen/chiselgen/ChiselGenMem.scala:164`, `src/spatial/codegen/chiselgen/ChiselGenMem.scala:165`, `src/spatial/codegen/chiselgen/ChiselGenMem.scala:170`, `src/spatial/codegen/chiselgen/ChiselGenMem.scala:176`, `src/spatial/codegen/chiselgen/ChiselGenMem.scala:182`). Fringe instantiates Chisel `Bundle`s and modules from those params: `MemParams` owns `Ns/Bs/Ps/WMapping/RMapping`, interfaces create `R_Port`/`W_Port` bundles, and `BankedSRAM` routes ports to banks using visible-bank residual generators and Chisel muxes (`fringe/src/fringe/templates/memory/MemParams.scala:7`, `fringe/src/fringe/templates/memory/MemParams.scala:11`, `fringe/src/fringe/templates/memory/MemParams.scala:14`, `fringe/src/fringe/templates/memory/MemInterfaceType.scala:57`, `fringe/src/fringe/templates/memory/MemInterfaceType.scala:80`, `fringe/src/fringe/templates/memory/MemPrimitives.scala:51`, `fringe/src/fringe/templates/memory/MemPrimitives.scala:75`, `fringe/src/fringe/templates/memory/MemPrimitives.scala:93`, `fringe/src/fringe/templates/memory/MemPrimitives.scala:138`). `NBufMem` is likewise Chisel buffer-depth and `bufferPort` routing (`fringe/src/fringe/templates/memory/NBuffers.scala:82`, `fringe/src/fringe/templates/memory/NBuffers.scala:94`, `fringe/src/fringe/templates/memory/NBuffers.scala:103`, `fringe/src/fringe/templates/memory/NBuffers.scala:113`). HLS codegen should preserve semantic planner output, but need not preserve these Chisel bundle/module shapes unless maintaining Chisel.

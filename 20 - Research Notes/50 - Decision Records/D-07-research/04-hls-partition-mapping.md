# HLS Partition/Pragma Mapping Constraints

## Local Banking Shape

Spatial's existing banking object is more general than a normal HLS array partition. `ModBanking` selects a bank with `(alpha dot addr / B) % N`, carries the chosen axes and padding vector `P`, and prints `B == 1` as "Cyclic" and `B != 1` as "Block Cyclic" (`src/spatial/metadata/memory/BankingData.scala:49`, `src/spatial/metadata/memory/BankingData.scala:62`, `src/spatial/metadata/memory/BankingData.scala:67`). The search can either flatten all dimensions into one bank address or treat dimensions hierarchically (`src/spatial/metadata/memory/BankingData.scala:510`, `src/spatial/metadata/memory/BankingData.scala:518`, `src/spatial/metadata/memory/BankingData.scala:523`). Its default `NBestGuess` does not just pick powers of two: it adds factors of concurrent access counts, factors of staged dimension products, powers of two, Mersennes, and near-Mersennes (`src/spatial/metadata/memory/BankingData.scala:547`, `src/spatial/metadata/memory/BankingData.scala:560`, `src/spatial/metadata/memory/BankingData.scala:562`).

## Current Search Contract

`MemoryConfigurer.bankGroups` builds a cross product of view, `N`, alpha, and duplication/regroup directives, sorts them by priority, partitions preferred and undesired cases, then calls `strategy.bankAccesses` (`src/spatial/traversal/banking/MemoryConfigurer.scala:610`, `src/spatial/traversal/banking/MemoryConfigurer.scala:616`, `src/spatial/traversal/banking/MemoryConfigurer.scala:619`, `src/spatial/traversal/banking/MemoryConfigurer.scala:633`). `ExhaustiveBanking.findBanking` then iterates candidate `N`s and alpha vectors until it finds enough schemes or hits the banking timeout (`src/spatial/traversal/banking/ExhaustiveBanking.scala:392`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:394`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:430`). For each alpha, it first tries cyclic `B = 1`; if permitted, it also tries configured block-cyclic `B`s and materializes `ModBanking(N, B, alpha, axes, P)` when the Presburger conflict checks pass (`src/spatial/traversal/banking/ExhaustiveBanking.scala:409`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:411`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:417`, `src/spatial/traversal/banking/ExhaustiveBanking.scala:422`). This contract assumes Spatial owns the bank-selection formula.

## Natural HLS Surface

The HLS surface should be treated as a smaller directive language. The local banking spec records the HLS assumption as rework: Vitis-like `array_partition` supports `cyclic`, `block`, and `complete` modes per dimension, and a Rust+HLS path would likely emit pragmas from a banking choice rather than re-create the full physical bank network (`10 - Spec/40 - Compiler Passes/70 - Banking.md:295`, `10 - Spec/40 - Compiler Passes/70 - Banking.md:297`). Specific vendor spellings remain external and are therefore (unverified), but the natural forms are: complete partition of a dimension, cyclic partition with a factor, block partition with a factor, array reshape/widening, explicit duplication by creating multiple arrays or instances, and storage binding to BRAM/URAM. Spatial has local analogues for the last two: regroup/dimension duplication is already part of the candidate grid (`src/spatial/traversal/banking/MemoryConfigurer.scala:616`), and Xilinx-style targets distinguish URAM, BRAM, URAM-overflow, and LUT memories with depth thresholds (`src/spatial/targets/xilinx/XilinxDevice.scala:13`, `src/spatial/targets/xilinx/XilinxDevice.scala:18`, `src/spatial/targets/xilinx/XilinxDevice.scala:23`, `src/spatial/targets/xilinx/XilinxDevice.scala:33`).

## Mapping Limits

Only a subset of `ModBanking` maps cleanly. A hierarchical, single-axis, `B = 1`, unit-alpha scheme can become cyclic partition factor `N`; full per-element banking can become complete partition; a simple contiguous-block case may become HLS block partition if the vendor's block semantics match the required address mapping (unverified). A reshape directive is not a conflict-proof bank mapping by itself: it packs adjacent elements into wider words and changes port granularity (unverified). Storage binding also does not solve banking conflicts; Spatial currently allocates resources after duplicates exist, first trying non-fallback resources and only assigning a duplicate when capacity and depth thresholds pass (`src/spatial/traversal/MemoryAllocator.scala:56`, `src/spatial/traversal/MemoryAllocator.scala:59`, `src/spatial/traversal/MemoryAllocator.scala:85`, `src/spatial/traversal/MemoryAllocator.scala:102`).

The hard mismatch is arbitrary alpha/modular banking. Spatial can search alpha vectors across flattened dimensions, including factor-, dimension-, coprime-, and power-of-two-derived values (`src/spatial/metadata/memory/BankingData.scala:571`, `src/spatial/metadata/memory/BankingData.scala:619`, `src/spatial/metadata/memory/BankingData.scala:623`, `src/spatial/metadata/memory/BankingData.scala:624`). A per-dimension HLS partition pragma cannot naturally express `bank = (18*i + j + 3*k + 9*l) % 36` or a Spatial block-cyclic repetition over a multidimensional alpha vector (unverified). Emitting such a scheme would require manual array-of-banks lowering or index rewriting, at which point the backend is no longer relying on HLS partition planning (unverified).

## D-07 Implication

For HLS, replace the alpha/N/B search core with an HLS-form planner, while reusing Spatial's evidence and candidate heuristics. The planner should consume the same access groups, affine conflict tests, staging dimensions, buffer depth, and duplication options from `MemoryConfigurer.bankGroups`, but its search space should be `partition_kind x factor x dimension x reshape x duplicate_count x binding_preference`, not arbitrary alpha vectors (`src/spatial/traversal/banking/MemoryConfigurer.scala:610`, `src/spatial/traversal/banking/MemoryConfigurer.scala:616`, `src/spatial/traversal/banking/MemoryConfigurer.scala:631`, `src/spatial/traversal/banking/MemoryConfigurer.scala:633`). `NBestGuess` is still useful as a factor-priority seed, especially because it already biases toward access-count and dimension-product factors (`src/spatial/metadata/memory/BankingData.scala:560`, `src/spatial/metadata/memory/BankingData.scala:562`). But the emitted plan should be constrained before solving, not filtered after the full exhaustive search, because the full search can find valid Spatial schemes that HLS pragmas cannot state. The decision posture is therefore: new HLS planner, shared local analysis, no promise to preserve arbitrary modular alpha banking.

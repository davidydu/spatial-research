# D-15 Angle 10: Recommendation Matrix for ParallelPipe Simulation

## Decision Frame

D-15 asks whether Rust simulation of `ParallelPipe` should follow ScalaGen's sequential source-order behavior or model parallel interleaving/back-pressure (`20 - Research Notes/40 - Decision Queue.md:67-69`; `20 - Research Notes/20 - Open Questions.md:2019-2024`). The key evidence is split by purpose. ScalaGen emits `ParallelPipe` by wrapping the body and calling `gen(func)` before a no-op done hook, so generated Scala visits statements textually (`spatial/src/spatial/codegen/scalagen/ScalaGenController.scala:187-191`; [[01-scalagen-parallelpipe-semantics]]; [[50 - Controller Emission]]). The Scala executor is closer to cooperative simulation: `ParallelPipe` uses `StreamUnitPipeExecutor`, creates child executors, ticks them in source order each parent tick, and completes when all children are finished/disabled/indeterminate (`src/spatial/executor/scala/ControlExecutor.scala:207-249`; [[03-scala-executor-simulator]]). Hardware and D-14 point the other way: `ParallelPipe` is assigned `ForkJoin`, whose Fringe controller activates enabled children under one parent window and joins on all child `iterDone` bits (`fringe/src/fringe/templates/Controllers.scala:141-154`; [[02-hardware-forkjoin-contrast]]; [[D-14]]).

## Cross-Decision Coupling

D-15 should not decide D-22 by implication. A peer/barrier simulator can model the `ForkJoin` control contract without claiming bounded-channel fidelity; a bounded cycle simulator cannot be honest until FIFO/LIFO/stream depth, full/empty, invalid-on-empty, and deadlock behavior are specified. Likewise, DSE should distinguish value compatibility, analytic runtime estimates, semantic simulation, bounded dataflow simulation, and backend reports rather than collapsing them into one unlabeled `Cycles` number ([[06-runtime-model-latency]]; [[D-08]]). This is also why randomized interleaving belongs in validation, not as the default user-visible reference.

## Preliminary Recommendation

Preliminarily choose **dual-mode/compare with `hls_semantic` deterministic peer/barrier as the HLS-facing simulation default**. Keep `compat_scalagen_serial` as an explicit compatibility mode for ScalaGen regression and old goldens. Define the default `ParallelPipe` rule as peer child issue plus all-child barrier over enabled, non-masked children, with deterministic polling/tie-breaks and mode provenance in reports. Defer full bounded cycle/back-pressure to D-22, then add it as a stricter `bounded_dataflow`/`cycle_backpressure` mode. Use seeded randomized interleavings only as a fuzz/testing layer, never as the semantics.

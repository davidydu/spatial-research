# D-25 Angle 1: Source IR And Scalagen Semantics

## IR surface and rewrites

The public surface is three separate APIs, not one overloaded selector policy: `mux` stages `Mux`, `oneHotMux` stages `OneHotMux`, and `priorityMux` stages `PriorityMux` (`src/spatial/lang/api/MuxAPI.scala:10-31`). `Mux` is the ordinary 2-way primitive; its rewrite folds literal true, literal false, and equal arms (`src/spatial/node/Mux.scala:9-16`). `OneHotMux` has a more semantic rewrite: one selector returns the only value, any statically literal-true selector returns the first true value, and multiple literal true selectors only warn before still selecting the first true lane (`src/spatial/node/Mux.scala:19-35`). `PriorityMux` intentionally does not do that literal-true selection, because the analogous block is commented out; it only collapses one selector or identical values (`src/spatial/node/Mux.scala:39-56`). There is no API-level length check in `oneHotMux`/`priorityMux`; both pass `sels` and boxed `vals` directly to the node (`src/spatial/lang/api/MuxAPI.scala:26-31`).

## Static checks and producers

Static checking is mostly warning, not enforcement. `SwitchOptimizer` drops statically false cases, records statically true cases, inlines exactly one true case, but if more than one true case or no selector remains it emits warnings and continues (`src/spatial/transform/SwitchOptimizer.scala:45-68`). For motion-safe bit results, it lowers two cases to `mux` and more than two to `oneHotMux` (`src/spatial/transform/SwitchOptimizer.scala:70-79`). `FlatteningTransformer` has the same bit-switch rule when `flattenSwitch` is active (`src/spatial/transform/FlatteningTransformer.scala:85-96`). `MemoryDealiasing` is another major producer: alias reads do all candidate reads under `ens + c` and return `oneHotMux(conds, reads)`, while metadata queries also use `oneHotMux` for addresses, dimensions, lengths, origins, par, and status reads (`src/spatial/transform/MemoryDealiasing.scala:22-35`, `src/spatial/transform/MemoryDealiasing.scala:90-127`, `src/spatial/transform/MemoryDealiasing.scala:150-159`). These sites assume exclusivity by construction; they do not prove it locally.

## Scalagen and other lowerings

Scalagen is the sharpest evidence for current runtime meaning. `Mux` lowers to Scala `if (sel) a else b` (`src/spatial/codegen/scalagen/ScalaGenBits.scala:29`). `OneHotMux` lowers to a `List((sel,d), ...)`, filters true selectors, then calls `.reduce{_|_}` (`src/spatial/codegen/scalagen/ScalaGenBits.scala:30-37`). Therefore dynamic multi-true means OR-reduce, while dynamic zero-true has no invalid fallback and will reduce an empty collection. `PriorityMux` lowers to ordered `if` / `else if` and ends with `else { invalid(R) }` (`src/spatial/codegen/scalagen/ScalaGenBits.scala:39-45`). Chiselgen preserves the intended distinction by emitting `Mux1H` for `OneHotMux` and Chisel `PriorityMux` for `PriorityMux` (`src/spatial/codegen/chiselgen/ChiselGenMath.scala:226-232`). PIR records one-hot arity as `OneHotMuxN`, while the area model treats `Mux`, `OneHotMux`, and `PriorityMux` as separate cost keys (`src/spatial/codegen/pirgen/PIRGenBits.scala:11-14`, `src/spatial/targets/NodeParams.scala:43-45`).

## OR behavior, invalids, and spec readback

The OR-reduce is type-sensitive. `Bool.|` is logical OR with validity conjunction (`emul/src/emul/Bool.scala:5-8`). `FixedPoint.|` is bitwise OR of stored scaled integers, also with validity conjunction (`emul/src/emul/FixedPoint.scala:23-25`). `FloatPoint` exposes arithmetic and comparisons but no `|` operator in its runtime class (`emul/src/emul/FloatPoint.scala:144-172`), so Scalagen float `OneHotMux` would not compile unless rewritten away. Structs are `Bits` when fields are bits-like, and `asBits`/recast operate fieldwise (`argon/src/argon/lang/types/Bits.scala:71-82`, `argon/src/argon/lang/types/Bits.scala:148-165`), but Scalagen emits structs as plain case classes with mutable fields and no OR method (`src/spatial/codegen/scalagen/ScalaGenStructs.scala:18-24`, `src/spatial/codegen/scalagen/ScalaGenStructs.scala:39-42`). Invalid fallback exists for `PriorityMux`: bit invalid is `Bool(false,false)`, fixed invalid is `FixedPoint.invalid`, float invalid is `FloatPoint.invalid`, and struct invalid recurses over fields (`src/spatial/codegen/scalagen/ScalaGenBit.scala:21-23`, `src/spatial/codegen/scalagen/ScalaGenFixPt.scala:26-28`, `src/spatial/codegen/scalagen/ScalaGenFltPt.scala:32-34`, `src/spatial/codegen/scalagen/ScalaGenStructs.scala:11-13`). The vault spec already records Scalagen as reference, with `OneHotMux` filter-and-OR and `PriorityMux` invalid fallback ([[60 - Counters and Primitives]], lines 58-65), and D-25 frames the open choice as OR-reduce parity, one-hot assertion, or priority semantics (`20 - Research Notes/40 - Decision Queue.md:107-109`). The source survey favors naming these as distinct modes rather than treating "one hot" as already enforced.

# D-17 Angle 5: Tests/Apps and Statistical Expectations for Unbiased Rounding

## Search Result

In `test` plus `apps`, the checked surface is much narrower than the API surface. The direct operator hits are `*&`, mostly `*&!`, and one `/&`; I found no `/&!`, no `FixToFixUnb`/`FixToFixUnbSat`, and no test/app references to the IR names `UnbMul` or `UnbDiv`. The missing cast coverage matters because ScalaGen and the Scala executor do implement unbiased fixed casts through `FixedPoint.unbiased`, while ChiselGen lowers `FixToFixUnb` and `FixToFixUnbSat` to `Math.fix2fix(..., Unbiased, Wrapping/Saturating, ...)` (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:107-114`, `src/spatial/executor/scala/resolvers/FixResolver.scala:37-51`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:75-78`). Arithmetic lowering also distinguishes all four IR nodes: `UnbMul`, `UnbDiv`, `UnbSatMul`, and `UnbSatDiv` map to `*&`, `/&`, `*&!`, and `/&!` in ScalaGen (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:92-99`).

## Focused Statistical Test

`SpecialMath` is the only focused test of unbiased rounding as a statistical property. It sets `N = 256`, repeats the same unsigned and signed `*&` products, and comments that the mean should be close to the exact product (`test/spatial/tests/feature/math/SpecialMath.scala:36-53`). The assertion is explicitly distributional, not sequence-based: it computes the returned means, compares them to exact host products, and accepts error below one LSB for the `_4` fractional format, `2^-4` (`test/spatial/tests/feature/math/SpecialMath.scala:75-88`). The same test checks saturation endpoints for `*&!`: unsigned upper endpoint `15.9375`, signed lower endpoint `-8`, and signed upper endpoint `7.9375` (`test/spatial/tests/feature/math/SpecialMath.scala:59-61`, `test/spatial/tests/feature/math/SpecialMath.scala:81-93`). Its closing print still says division needs checking, which matches the search gap: there is no focused `/&` or `/&!` statistical test (`test/spatial/tests/feature/math/SpecialMath.scala:106-107`).

## App-Level Expectations

The apps use unbiased rounding as noise-tolerant low-precision arithmetic, not as a bit-exact observable. `SGD` uses `*&!` in the prediction reduction and asserts only that final squared distance from `W_gold` is below a configured threshold (`test/spatial/tests/apps/SGD.scala:137-143`, `test/spatial/tests/apps/SGD.scala:179-198`). `SVRG` uses several `*&!` sites in full-gradient and per-example update paths, then checks Cartesian distance against a threshold rather than exact weights (`test/spatial/tests/apps/SVRG.scala:138-179`, `test/spatial/tests/apps/SVRG.scala:219-239`). `LP_SVRG` is the one `/&` hit, normalizing a gradient contribution by `N`, while later `*&!` sites participate in the stochastic update; its only final assertion is also distance below threshold (`test/spatial/tests/apps/LP_SVRG.scala:203-208`, `test/spatial/tests/apps/LP_SVRG.scala:238-256`, `test/spatial/tests/apps/LP_SVRG.scala:302-323`). `SmallTypeTransfers` uses `*&`, but only for exactly representable half-step values and exact array equality, so it does not constrain stochastic behavior (`test/spatial/tests/feature/transfers/SmallTypeTransfers.scala:20-35`, `test/spatial/tests/feature/transfers/SmallTypeTransfers.scala:60-69`).

## Seeds and Flakiness

I found no explicit seed, `setSeed`, `flaky`, or `flake` marker in `test`/`apps`. That is awkward because current emulation consumes `scala.util.Random.nextFloat()` inside `FixedPoint.unbiased`, after deriving a four-extra-fractional-bit remainder (`emul/src/emul/FixedPoint.scala:232-240`). Several app tests independently use `random[...]` for data or sampling; for example, `SGD` chooses a random point each epoch, and low-precision SGD uses random perturbations in cast-like helpers (`test/spatial/tests/apps/SGD.scala:120-143`, `test/spatial/tests/apps/LP_SGD.scala:47-56`, `test/spatial/tests/apps/LP_SGD.scala:185-207`). Current tests therefore implicitly tolerate nondeterminism by using means or convergence thresholds, but they do not provide replay evidence for a failed stochastic run.

## Canonical Test Policy

The canonical policy should make the probability law observable and the RNG sequence non-canonical unless D-17 intentionally chooses sequence compatibility. Existing evidence supports "statistically unbiased with exact saturation endpoints": preserve the four-guard-bit threshold rule used by `*&`, `/&`, `*&!`, and `/&!`, then clamp or saturate as requested (`emul/src/emul/FixedPoint.scala:52-62`, `emul/src/emul/FixedPoint.scala:224-240`). Required tests should include fixed-seed replay, changed-seed variation allowance, and order/lane independence if stable per-event RNG is adopted. Property/statistical tests should cover signed and unsigned `*&`, `/&`, `*&!`, `/&!`, plus `FixToFixUnb` and `FixToFixUnbSat`; each should check expected mean within a justified confidence bound, endpoint saturation, negative-value behavior, divide-by-zero invalid propagation, and exact replay metadata. App tests can remain convergence-style, but canonical regression tests need small deterministic fixtures and seeded statistical fixtures so failures are debuggable rather than merely "try again."

# Migration and Update Cost

## Baseline Cost Surface

[[D-17]] is not just a simulator switch. The queue asks whether canonical unbiased rounding keeps nondeterministic Spatial behavior, becomes seedable, or is replaced by deterministic rounding (`20 - Research Notes/40 - Decision Queue.md:75-77`). The current source is compact but widely exposed: `FixedPoint.unbiased` consumes four guard bits, draws `scala.util.Random.nextFloat()`, steps `biased + 1` for nonnegative values or `biased - 1` for negative values when the stochastic threshold fires, then wraps or saturates (`emul/src/emul/FixedPoint.scala:232-240`). Arithmetic reaches it through `*&`, `/&`, `*&!`, and `/&!` (`emul/src/emul/FixedPoint.scala:52-63`), Scalagen emits both those operators and fixed-to-fixed unbiased casts (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:96-114`), and the Scala executor resolves cast nodes through the same helper (`src/spatial/executor/scala/resolvers/FixResolver.scala:45-51`). The specs already call this nondeterministic reference behavior and say Rust+HLS must resolve it rather than inherit it accidentally (`10 - Spec/50 - Code Generation/20 - Scalagen/20 - Numeric Reference Semantics.md:41-47`; `10 - Spec/20 - Semantics/50 - Data Types.md:66`).

## Keep Ambient Nondeterminism

Keeping ambient Scala behavior has the lowest Scala-emul edit cost: leave the helper signature and generated Scala unchanged. But it has the highest cross-backend carrying cost. Rust would need either a Scala/JVM-compatible global stream or a consciously non-replayable simulator mode, which conflicts with [[D-09]]'s reproducibility handoff. HLS cannot depend on host `scala.util.Random`; existing hardware is already a different stochastic source, because ChiselGen passes `Unbiased` into `Math.mul`, `Math.div`, and `Math.fix2fix` (`src/spatial/codegen/chiselgen/ChiselGenMath.scala:33-40`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:75-78`), while Fringe defines `Unbiased` as LFSR stochastic rounding (`fringe/src/fringe/templates/math/RoundingMode.scala:3-8`) and seeds a `PRNG` during elaboration (`fringe/src/fringe/templates/math/Converter.scala:36-44`; `fringe/src/fringe/templates/math/PRNG.scala:6-16`). Tests remain mostly statistical, not reproducible: `SpecialMath` averages 256 samples and accepts within one LSB, rather than checking exact bit sequences (`test/spatial/tests/feature/math/SpecialMath.scala:51-88`). This option preserves legacy surprise instead of paying migration cost.

## Seeded Stable Stochastic

Seeded stable stochastic is medium implementation cost and lowest semantic churn. Scala emul needs a context-aware overload or policy object around `FixedPoint.unbiased`, while keeping the guard-bit rule and legacy negative branch for compatibility. Scalagen must emit or sidecar stable event IDs for arithmetic and casts, because current one-line emission exposes no RNG identity (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:96-114`). Rust simulator work is real but localized: implement a pure threshold from `(seed, algorithm, static op, dynamic instance, lane)` as recommended by [[06-seedable-rng-policy]] and formalized by [[07-cross-backend-provenance-schema]]. Existing config/CLI patterns can carry simulator knobs, since `SpatialConfig` already tracks backend/simulator flags and copies them into worker states (`src/spatial/SpatialConfig.scala:21-23`, `src/spatial/SpatialConfig.scala:95-120`), and CLI backend switches already live beside `--sim` and `--synth` (`src/spatial/Spatial.scala:281-293`). HLS lowering must implement a seeded hash/LFSR primitive with recorded advance policy; vendor fixed-point libraries generally make deterministic rounding easier than portable stochastic rounding (unverified). Specs change from "nondeterministic" to "seeded stochastic with provenance," and tests gain exact same-seed replay plus statistical changed-seed checks.

## Deterministic Rounding

Deterministic rounding is lowest backend mechanics cost and highest semantic update cost. Rust gets ordinary integer rounding with no RNG state. HLS can likely delegate round-to-nearest-even or truncation to arbitrary-precision fixed-point types (unverified), or implement compare/add logic directly. Scala emul changes are straightforward but invasive: `FixedPoint.unbiased` must stop calling `Random.nextFloat()` and the staged API comments promising probabilistic rounding after `*&` and `/&` become wrong (`argon/src/argon/lang/Fix.scala:75-87`, `argon/src/argon/lang/Fix.scala:115-129`). Chisel/Fringe `Unbiased` must either be renamed, redefined, or split from deterministic modes, because today's source explicitly separates `Truncate` from stochastic `Unbiased` (`fringe/src/fringe/templates/math/RoundingMode.scala:3-8`). Tests must replace mean-based stochastic expectations with exact edge fixtures and review convergence-tolerance apps; [[05-tests-apps-statistical-expectations]] shows division and cast coverage gaps that would become mandatory migration fixtures.

## Ranking

For migration cost, ambient nondeterminism looks cheap only if Scala emul is the whole product; across Rust, HLS, specs, and CI it is the most expensive to keep explaining. Deterministic rounding is cheapest to implement in new backends but expensive to justify, rename, and retest because it changes the meaning recorded in [[20 - Numeric Reference Semantics]]. Seeded stable stochastic is the best cost balance: more upfront plumbing than deterministic rounding, but it preserves the current stochastic intent, makes [[D-09]] replayable, gives HLS an explicit primitive instead of hidden host randomness, and confines spec churn to provenance and seed policy rather than a semantic rewrite.

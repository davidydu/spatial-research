# D-18 Research: Backend Codegen Variation

## Reference Surface

D-18 should treat backend float behavior as a split contract, not one uniform implementation. The IR-level `Flt` type records `mbits` and `ebits`, then lowers to emulation `FltFormat(mbits - 1, ebits)` (`argon/src/argon/lang/Flt.scala:9-16`, `argon/src/argon/lang/Flt.scala:25-32`). [[20 - Numeric Reference Semantics]] therefore has to distinguish the reference simulator from generated C++ and RTL. In `emul`, `FloatPoint` is a tagged value (`NaN`, signed infinity, signed zero, or `Value(BigDecimal)`), and arithmetic operators route through `FloatPoint.clamped` after every neg/add/sub/mul/div/mod result (`emul/src/emul/FloatPoint.scala:144-165`). Packing also calls clamp: `Value.bits` calls `FloatPoint.clamp`, then emits mantissa, exponent, and sign bits (`emul/src/emul/FloatPoint.scala:120-126`). The clamp algorithm itself is only defined under `emul`: it decomposes `BigDecimal` values, handles the `x > 1.9` subnormal guard, emits infinity/normal/subnormal/zero encodings, then canonicalizes through `convertBackToValue` in `clamped` (`emul/src/emul/FloatPoint.scala:318-433`). A repository search found `FloatPoint.clamp`/`clamped` use only in `emul` numeric helpers, not in cppgen, chiselgen, or fringe codegen (`emul/src/emul/Number.scala:116-155`, `emul/src/emul/FloatPoint.scala:120-433`).

## Scala Emission and Executor

Scalagen is closest to [[10 - Overview]]'s reference semantics. It remaps every `FltPtType` to `FloatPoint`, preserves special constants through `FloatPoint("NaN"|"Inf"|"-Inf"|"-0.0", FltFormat(...))`, and emits invalid values as `FloatPoint.invalid` (`src/spatial/codegen/scalagen/ScalaGenFltPt.scala:12-34`). Arithmetic is emitted as overloaded `FloatPoint` operations, so clamp happens indirectly inside `emul`; `FltToFlt` emits `.toFloatPoint(...)`, `FltToFix` emits `.toFixedPoint(...)`, and `TextToFlt` constructs a new `FloatPoint` (`src/spatial/codegen/scalagen/ScalaGenFltPt.scala:63-91`). Scalagen packing is also semantic rather than native: `DataAsBits` emits `$a.bits`, `BitsAsData` emits `FloatPoint.fromBits`, `DataAsVec` groups float bits into fixed elements, and `VecAsData` flattens element bits back into `FloatPoint.fromBits` (`src/spatial/codegen/scalagen/ScalaGenBits.scala:47-72`; `emul/src/emul/FloatPoint.scala:435-466`). The Scala executor is narrower: `FltResolver` handles predicates, `FltToFix`, `FltToFlt`, text, and random, while generic unary/binary/comparison float ops execute through the `unstaged` functions on `Flt` nodes (`src/spatial/executor/scala/resolvers/FltResolver.scala:8-42`; `src/spatial/executor/scala/resolvers/NaryResolver.scala:22-47`; `argon/src/argon/node/Flt.scala:22-39`). One exception matters for [[D-18]] and D-19: `FltFMA` has no unstaged function and no executor resolver, while Scalagen emits `($m1 * $m2) + $add`, causing two ordinary `FloatPoint` round/clamp points rather than a fused operation (`argon/src/argon/node/Flt.scala:26`; `src/spatial/codegen/scalagen/ScalaGenFltPt.scala:120`; `src/spatial/executor/scala/resolvers/OpResolver.scala:35-39`).

## Cppgen Variation

Cppgen is not clamp-compatible for custom formats. It maps `FltPtType(g,e)` to native `float`, maps `FloatType` to `float` and `DoubleType` to `double`, and prints `FltPtType` constants as raw literals (`src/spatial/codegen/cppgen/CppGenCommon.scala:75-99`, `src/spatial/codegen/cppgen/CppGenCommon.scala:125-128`). Float arithmetic and predicates lower to native C/C++ expressions or libm calls: `+`, `-`, `*`, `/`, `1.0 / y`, `fabs`, `sqrt`, `exp`, `pow`, and ternary min/max (`src/spatial/codegen/cppgen/CppGenMath.scala:23-58`, `src/spatial/codegen/cppgen/CppGenMath.scala:95-117`). Casts are plain C casts for `FixToFlt`, `FltToFix`, and `FltToFlt`, with no `FltFormat`, no subnormal heuristic, and no valid-bit model (`src/spatial/codegen/cppgen/CppGenMath.scala:136-138`). Packing is partial and native-width: `DataAsBits`/`DataAsVec` for floats reinterpret a `float*` as `int32_t*`, but `BitsAsData` and `VecAsData` throw "Bit-wise operations not supported on floating point values yet" (`src/spatial/codegen/cppgen/CppGenArray.scala:111-147`). Thus [[10 - Cppgen]] is useful evidence for host approximation, not for bit-identical D-18 packing.

## Chisel and Fringe Variation

Chiselgen keeps the custom width but not the Scala clamp algorithm. It maps `FltPtType(m,e)` to `new FloatingPoint(m, e)` and float constants to `.FlP(m,e)` (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:315-326`; `fringe/src/fringe/utils/implicits.scala:180-198`). Arithmetic and casts delegate to fringe math: `FltAdd/Sub/Mul/Div/FMA/Sqrt/Recip` emit `Math.fadd`, `Math.fsub`, `Math.fmul`, `Math.fdiv`, `Math.fma`, and conversions emit `Math.fix2flt`, `Math.flt2fix`, `Math.flt2flt` (`src/spatial/codegen/chiselgen/ChiselGenMath.scala:51-65`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:92-102`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:185-247`). Chisel packing is raw hardware wiring: `DataAsBits` indexes the data bits, `DataAsVec` slices bit ranges, and `VecAsData`/`BitsAsData` concatenate raw bits (`src/spatial/codegen/chiselgen/ChiselGenMath.scala:259-267`).

Fringe defines `FloatingPoint` as a `UInt(m + e)` payload with raw accessors; its unimplemented `cast` method confirms that format conversion is centralized elsewhere (`fringe/src/fringe/templates/math/FloatingPoint.scala:8-29`). Literals call `emul.FloatPoint(...).bits` only to create constant bit patterns, so `FloatPoint.clamp` can affect literal packing but not runtime RTL arithmetic (`fringe/src/fringe/templates/math/FloatingPoint.scala:91-124`). Runtime operations delegate to `globals.bigIP`, and `BigIP` makes float add/sub/mul/div/compare/convert target hooks (`fringe/src/fringe/templates/math/Math.scala:548-678`, `fringe/src/fringe/templates/math/Math.scala:680-799`; `fringe/src/fringe/BigIP.scala:33-104`). Sim targets use HardFloat-style recFN modules for core ops and fixed/float conversions, while Zynq-style targets instantiate named blackboxes including `Float2Float` (`fringe/src/fringe/targets/BigIPSim.scala:136-173`, `fringe/src/fringe/targets/BigIPSim.scala:275-291`; `fringe/src/fringe/targets/zynq/BigIPZynq.scala:97-125`, `fringe/src/fringe/targets/zynq/BigIPZynq.scala:211-264`). The decision implication is clear: D-18 bit compatibility must be specified against Scalagen/emul; C++ and Chisel/fringe are divergent backend precedents, not proof that `FloatPoint.clamp` is optional for the reference simulator.

# Format Edge Cases

## Format Fields And Bit Order

Read this with [[D-18]], [[20 - Numeric Reference Semantics]], and [[50 - Data Types]]. Spatial's staged `Flt[M,E]` calls the first parameter `mbits`, includes the hidden significand bit in that count, and converts to emul as `FltFormat(mbits - 1, ebits)` (`/Users/david/Documents/David_code/spatial/argon/src/argon/lang/Flt.scala:9-16`, `/Users/david/Documents/David_code/spatial/argon/src/argon/lang/Flt.scala:29-31`). Thus `Float` is `FltPtType(24,8)` at the DSL layer but `FltFormat(23,8)` in emul (`/Users/david/Documents/David_code/spatial/argon/src/argon/lang/Flt.scala:144-153`, `/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:260-262`). `FltFormat` defines total bits as `sbits + ebits + 1`, with `bias = 2^(ebits - 1) - 1`, `MIN_E = -bias + 1`, `MAX_E = bias`, and `SUB_E = MIN_E - sbits` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FltFormat.scala:3-8`). Packing is little-endian by field: mantissa bits first, exponent bits next, sign last (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:120-126`, `/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:435-455`).

## Special Values

The value model is explicit: `NaN`, signed `Inf`, signed `Zero`, or finite `Value(BigDecimal)` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:84-112`). NaN packs as all exponent bits one with a canonical top mantissa bit; any incoming all-one exponent with nonzero mantissa unpacks to this payload-less `NaN` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:88-92`, `/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:440-443`). Infinities pack as all-one exponent and zero mantissa; zeros pack as all-zero exponent and mantissa with sign retained (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:94-110`). Float and Double constructors preserve negative zero with a reciprocal sign check, but `BigDecimal(0)` enters `clamp` and becomes positive zero (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:129-141`, `/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:318-322`). Arithmetic uses custom rules: NaN contaminates, opposite-signed infinities in addition become NaN, `0/0` and `inf/inf` become NaN, division by zero becomes signed infinity, and both zeros compare equal (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:12-47`, `/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:71-80`). [inference] The zero predicate names appear swapped: `Zero(true)` prints `-0.0`, but `isPosZero` checks `Zero(true)` and `isNegZero` checks `Zero(false)` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:103-110`, `/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:220-224`).

## Rounding And Subnormals

Finite construction re-enters `FloatPoint.clamp`, so packing is also runtime quantization (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:417-433`). Normal numbers encode `mantissa = round(((x - 1) * 2^(sbits + 1)) / 2)` and exponent `y + bias`; the implementation keeps one guard bit and rounds up when that bit is one (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:358-370`). Subnormals encode exponent bits zero, compute `x * 2^(sbits + 1)`, shift by `MIN_E - y + 1`, and again round from one discarded bit; a zero shifted mantissa underflows to signed zero (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:372-395`). [inference] This is not IEEE round-to-nearest-even because there is no sticky bit or even-tie check. Hardware is not a single mirror: BigIPSim uses HardFloat recoding for div/sqrt and int conversions with rounding mode `0` and tininess false, while add/sub/mul instantiate `MulAddRecFN` without visibly assigning those inputs after `DontCare` (`/Users/david/Documents/David_code/spatial/fringe/src/fringe/targets/BigIPSim.scala:125-173`, `/Users/david/Documents/David_code/spatial/fringe/src/fringe/targets/BigIPSim.scala:275-291`, `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/hardfloat/MulAddRecFN.scala:302-345`). [unverified] That may make some hardware boundary behavior target/tool dependent.

## `x > 1.9` Boundary Effect

`clamp` decomposes nonzero finite values into `y = floor(log2(abs(value)))` and `x = abs(value) / 2^y`; before normal `x >= 2` repair, it checks `y < SUB_E && x > 1.9`, then increments `y` and forces `x = 1` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:318-343`). [inference] For `y = SUB_E - 1`, this snaps values in roughly `(1.9 * 2^(SUB_E - 1), 2^SUB_E)` to the minimum subnormal path instead of final zero underflow. Values in the same exponent bucket with `x <= 1.9` still underflow, so the threshold is a narrow rescue band, not a principled half-ulp rule (`/Users/david/Documents/David_code/spatial/emul/src/emul/FloatPoint.scala:386-395`). [unverified] The best reading is compatibility debt: preserve it for bit-identical Scalagen/Rust tests, but do not cite it as a mathematical custom-float spec without D-18 explicitly deciding that.

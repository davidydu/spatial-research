# D-14 Angle 9: Risk Analysis and Implementation Cost

## Risk frame

The policy choice matters because these three tags are structural, not cosmetic. Spatial enumerates `Fork`, `ForkJoin`, and `PrimitiveBox` alongside ordinary schedules (`src/spatial/metadata/control/ControlData.scala:7-14`), and flow assigns them categorically: `ParallelPipe` to `ForkJoin`, `Switch`/`IfThenElse` to `Fork`, `SpatialBlackboxImpl` to `PrimitiveBox`, and `SwitchCase` to `Sequenced` (`src/spatial/flows/SpatialFlowRules.scala:317-339`). A Rust/HLS scheduler can therefore fail in five distinct ways: execute the wrong branch, omit a join barrier, create deadlock or back-pressure mismatch, mis-model blackbox latency/effects, or feed DSE a latency model that prices a different machine than the generated HLS.

## Preserve Chisel/Fringe semantics exactly

This is the strongest compatibility option, but also the most expensive. The upside is semantic continuity with hardware: ChiselGen instantiates controller state machines from `lhs.level` and `lhs.rawSchedule` (`src/spatial/codegen/chiselgen/ChiselGenController.scala:333-358`). Fringe `ForkJoin` synchronizes with an AND of all child `iterDone` bits and advances the counter only at that barrier (`fringe/src/fringe/templates/Controllers.scala:141-154`), while `Fork` gates active children by `selectsIn(i)` and synchronizes with OR over child done (`fringe/src/fringe/templates/Controllers.scala:173-187`). The model also exposes parent enable, child done/mask, acknowledgements, and backpressure as first-class controller ports (`fringe/src/fringe/templates/Controllers.scala:9-43`), with shared enable/ack logic after the schedule match (`fringe/src/fringe/templates/Controllers.scala:212-215`).

The cost is that "exact" means rebuilding a Chisel-style handshake in Rust and mapping it onto HLS task, dataflow, and pipeline constructs. That is high implementation and verification cost, especially for masked children, break, child acknowledgements, and bounded FIFO behavior. Failure modes are severe but detectable: a `Fork` implemented as parallel fork can execute multiple branches; a `ForkJoin` implemented as serial or fire-and-forget can miss the all-child barrier; an incomplete back-pressure model can deadlock or hide a deadlock. `PrimitiveBox` is the awkward edge: Fringe has no `PrimitiveBox` schedule object (`fringe/src/fringe/templates/Controllers.scala:62-67`), so exact preservation still requires a non-controller special case.

## Use ScalaGen serial simulation semantics

This is the cheapest implementation and the easiest golden-test story. ScalaGen emits block statements in order through `javaStyleChunk` (`src/spatial/codegen/scalagen/ScalaCodegen.scala:37-53`), emits `ParallelPipe` by generating the body textually and then marking done (`src/spatial/codegen/scalagen/ScalaGenController.scala:187-191`), and emits switches as an ordered `if` / `else if` chain (`src/spatial/codegen/scalagen/ScalaGenController.scala:209-220`). That makes branch choice deterministic and naturally avoids many concurrency races in Rust simulation.

Its risk is that it is a simulation compatibility policy, not an HLS schedule policy. Serializing `ForkJoin` erases the join-barrier contract rather than modeling it, so memory conflicts and stream back-pressure can appear safe in Rust while becoming cyclic or throughput-limited in HLS. It also contradicts the runtime model, which prices `ForkJoin` as max-child parallel cost rather than sum-child serial cost (`models/src/models/RuntimeModel.scala:321`). DSE then optimizes for a different execution model than the simulator. This option is useful as an explicit `compat_scalagen_serial` mode, but dangerous as the canonical HLS semantics.

## Define HLS-native semantics

This is medium-to-high implementation cost with the best long-term risk profile. The contract would be semantic rather than waveform-exact: `Fork` is one selected branch per iteration, with multi-true selectors rejected or treated by a documented priority rule; `ForkJoin` launches all enabled children in an HLS dataflow/task region and joins after every non-masked child completes; `PrimitiveBox` is an atomic function/IP boundary with declared latency, II, effects, and optional reference model. This preserves the key source facts while letting HLS reports become the authority for accepted II and latency.

The main risk is underspecification. If the HLS-native contract does not define selector exclusivity, branch priority, child masks, and effect visibility, it can still produce wrong branch execution or lost barriers. Blackboxes need particular care: primitive blackbox codegen wires `bbox.io.done` as child done and a parent mask as child mask (`src/spatial/codegen/chiselgen/ChiselGenBlackbox.scala:147-160`), while Chisel implementation emission builds module wrappers rather than schedule machines (`src/spatial/codegen/chiselgen/ChiselGenBlackbox.scala:166-238`). DSE cost is moderate: the current model lacks `PrimitiveBox` in its schedule enum (`models/src/models/RuntimeModel.scala:29-40`), models `Fork` by branch duty-cycle weights (`models/src/models/RuntimeModel.scala:325-327`), and already feeds `Cycles` through DSE/HyperMapper consumers (`src/spatial/dse/DSEAnalyzer.scala:96-128`; `src/spatial/dse/HyperMapperDSE.scala:70-88`). HLS-native semantics should therefore add provenance, not silently reuse stale equations.

## Leave as opaque metadata

Opaque metadata is lowest short-term cost and highest semantic risk. It preserves parsing and diagnostics but delegates the real policy to later passes. That virtually guarantees inconsistent lowering: one consumer may serialize `ForkJoin`, another may assume max-child parallelism, and DSE may keep pricing legacy schedules while HLS emits something else. It also leaves `PrimitiveBox` latency/effects underdefined, despite existing blackbox metadata carrying latency and pipeline factor (`src/spatial/metadata/blackbox/BlackboxData.scala:6-17`). Cost ranking: opaque metadata is cheapest but unsafe; ScalaGen serial is cheap but compatibility-only; Chisel parity is safest for legacy hardware but costly and brittle; HLS-native is the preferred middle path if paired with explicit mismatch checks and report-calibrated DSE.

# D-14 Angle 7: Tests and Apps Evidence

## Scope and Signal Strength

The tests/apps corpus rarely names `ForkJoin` directly. The source-level evidence is mostly `Parallel { ... }`, branch/Switch behavior, and blackbox APIs; `ParallelPipeInsertion` is the only test name containing `ParallelPipe` (`test/spatial/tests/feature/control/ParallelPipeInsertion.scala:5`). I treat assertions and goldens as source-proven behavior, and comments or disabled checks as semantic pressure. This distinction matters because the tests can prove final observable results, but usually do not prove cycle-level simultaneity, barrier timing, or HLS lowering shape.

## Parallel / ForkJoin Evidence

Source-proven behavior: `Parallel` is used when sibling effects must all be visible after the region. `StridedLoad` splits even and odd row loads into two `Parallel` branches, then stores the combined SRAM and asserts the full matrix equals the original source (`test/spatial/tests/feature/transfers/StridedLoad.scala:21-27`, `test/spatial/tests/feature/transfers/StridedLoad.scala:30-45`). `ScatterInterference` puts two independent scatter pipelines in `Parallel`, writing addresses `0` and `65`, and asserts both updates land in DRAM (`test/spatial/tests/feature/transfers/ScatterInterference.scala:27-45`, `test/spatial/tests/feature/transfers/ScatterInterference.scala:48-52`). The Smith-Waterman app similarly uses `Parallel` for two aligned-output stores after the backtrack FSM (`test/spatial/tests/apps/SW.scala:154-157`).

Inferred pressure: these tests require all branches to be scheduled as peers and joined before downstream observation, but they do not prove hardware-level concurrent issue. `ParallelPipeInsertion` is stronger structurally because it places a conditional load and an unconditional slice load in one `Parallel` block inside a loop (`test/spatial/tests/feature/control/ParallelPipeInsertion.scala:22-33`), but its final assert is commented out (`test/spatial/tests/feature/control/ParallelPipeInsertion.scala:36-43`). Treat it as a regression sketch, not executable proof.

## Branches / Fork Evidence

Source-proven behavior: branch tests expect exactly selected effects. `OuterSwitch` leaves DRAM all-zero when the outer condition is false, otherwise writes only the guarded prefix (`test/spatial/tests/feature/control/OuterSwitch.scala:18-24`, `test/spatial/tests/feature/control/OuterSwitch.scala:27-29`). `NestedIfs` checks nested then/else paths with mutually exclusive results `10` or `20` (`test/spatial/tests/feature/control/NestedIfs.scala:17-30`, `test/spatial/tests/feature/control/NestedIfs.scala:35-40`). `InlineSwitch` covers an if/else ladder as a value expression and asserts the scalar branch result (`test/spatial/tests/compiler/InlineSwitch.scala:15-22`). `FIFOBranch` is especially operational: the reduce body dequeues from `fifo1` until empty, then from `fifo2`, and checks the sum of both FIFO sequences (`test/spatial/tests/feature/control/FIFOBranch.scala:21-27`, `test/spatial/tests/feature/control/FIFOBranch.scala:30-35`).

Inferred pressure: `Fork` should mean selected branch execution, not all-child join. The corpus does not expose multi-true `Switch` selectors, so one-hot or priority behavior remains a spec question rather than a tests-proven fact.

## Branch Dephasing and Synchronization

`SynchronizedStartsTest` is the clearest tests/apps evidence that branch structure changes scheduling legality. Comments say an "unknown Fork" injected into the hierarchy makes controllers unsynchronized for lane-dependent branches (`test/spatial/tests/feature/synchronization/SynchronizedStartsTest.scala:78-86`, `test/spatial/tests/feature/synchronization/SynchronizedStartsTest.scala:100-109`). The same file contrasts a forked-iterator-invariant branch, where all controllers stay synchronized (`test/spatial/tests/feature/synchronization/SynchronizedStartsTest.scala:89-97`). Its `checkIR` then requires different SRAM duplication counts: branch/fork cases such as `sram6`, `sram8`, `sram9`, and `sram10` require two or four duplicates, while invariant cases stay at one (`test/spatial/tests/feature/synchronization/SynchronizedStartsTest.scala:148-169`).

Source-proven behavior here is compiler-analysis expectation, not user-visible output. The inferred Rust/HLS pressure is that branch-dependent runtime variance can dephase lanes after a fork, so a scheduler cannot treat all conditional regions as lockstep unless branch predicates are proved invariant.

## Blackboxes / PrimitiveBox Evidence

Primitive blackbox tests prove input-output replacement behavior. `VerilogPrimitive` declares latency and pipeline factor in comments, calls adder/multiplier/divmod/delay boxes inside a parallelized loop, and asserts each DRAM result equals the inline arithmetic goldens (`test/spatial/tests/syntax/Blackboxes.scala:5-8`, `test/spatial/tests/syntax/Blackboxes.scala:35-49`, `test/spatial/tests/syntax/Blackboxes.scala:58-86`). `SpatialPrimitive` declares the blackbox outside `Accel`, uses it in a `Foreach`, and checks max/min outputs against scalar goldens (`test/spatial/tests/syntax/Blackboxes.scala:143-148`, `test/spatial/tests/syntax/Blackboxes.scala:156-190`, `test/spatial/tests/syntax/Blackboxes.scala:198-220`). XGBoost puts each decision tree in a `Blackbox.SpatialPrimitive` and compares blackboxed predictions to a numeric golden (`test/spatial/tests/apps/XGBoost.scala:117-124`, `test/spatial/tests/apps/XGBoost.scala:143-158`).

Controller blackboxes are different evidence. `SpatialBBoxParams` uses `Blackbox.SpatialController` with FIFO stream interfaces inside `Stream.Foreach` and asserts DRAM streams match parameterized reductions (`test/spatial/tests/syntax/Blackboxes.scala:265-303`, `test/spatial/tests/syntax/Blackboxes.scala:320-337`). `FilterStream1D` comments that BBoxes disable stream-counter optimization, then instantiates a controller blackbox as a streaming processing stage (`test/spatial/tests/feature/streaming/FilterStream1D.scala:608-622`). Inferred pressure: `PrimitiveBox` can be an atomic latency-bearing primitive boundary, but `CtrlBlackbox` semantics require stream handshakes and controller scheduling rather than primitive substitution.

# API/IR Origins of `Fork`, `ForkJoin`, and `PrimitiveBox`

## Schedule Assignment Is Structural

`CtrlSchedule` merely enumerates `Sequenced`, `Pipelined`, `Streaming`, `ForkJoin`, `Fork`, and `PrimitiveBox`; the useful meaning comes from where flow assigns them (`src/spatial/metadata/control/ControlData.scala:7-14`). The assignment rules are categorical: `ParallelPipe` is always `ForkJoin`; `Switch` and `IfThenElse` are always `Fork`; `SpatialBlackboxImpl` is `PrimitiveBox`; `SpatialCtrlBlackboxImpl` and `SwitchCase` are `Sequenced` (`src/spatial/flows/SpatialFlowRules.scala:317-337`). By contrast, the user-facing `Pipe`, `Stream`, and `Sequential` directives all stage `UnitPipe` through `unit_pipe`; they differ by the `CtrlOpt` schedule they carry, with `Pipe` selecting `Pipelined`, `Stream` selecting `Streaming`, and `Sequential` selecting `Sequenced` (`src/spatial/lang/control/Control.scala:16-24`, `src/spatial/lang/control/Control.scala:35-51`). The schedule accessor preserves the special schedules even for single-iteration controllers, while only `Pipelined` collapses to `Sequenced` in single-control cases (`src/spatial/metadata/control/package.scala:193-215`). Therefore `Fork`, `ForkJoin`, and `PrimitiveBox` are compiler-recognized structural shapes, not normal user schedule directives.

## `ForkJoin`: Parallel Region With Barrier Join

The public origin is `Parallel { ... }`, which stages the body and emits `ParallelPipe(Set.empty, block)` (`src/spatial/lang/control/Parallel.scala:9-13`). `ParallelPipe` is a `Pipeline[Void]` with no counters and one pseudo-stage, like `UnitPipe`, but it lacks `stopWhen` and is assigned `ForkJoin` rather than inheriting a user schedule (`src/spatial/node/Control.scala:33-43`; `src/spatial/flows/SpatialFlowRules.scala:331-333`). It is also synthesized by compiler transforms: independent child-controller bundles become `ParallelPipe`, and multi-lane unrolling can wrap duplicated controller copies in `ParallelPipe` (`src/spatial/transform/BindingTransformer.scala:80-83`; `src/spatial/transform/unrolling/UnrollingBase.scala:147-165`; `src/spatial/transform/unrolling/ForeachUnrolling.scala:40-49`).

Operationally, `ForkJoin` means "start all enabled child stages for the same logical iteration and advance only after all have finished." The Chisel `OuterControl` implementation sets `synchronize` to the AND of all `iterDone` bits, drives every child active under the same enable/backpressure condition, and increments the counter only when that barrier fires (`fringe/src/fringe/templates/Controllers.scala:141-154`). The runtime model also prices `ForkJoin` as `maxChild * cchainIters`, not a sum of children (`models/src/models/RuntimeModel.scala:321`). Unverified inference: Rust/HLS should lower this as a dataflow/parallel child region with an AND barrier, not as source-order serial execution, even though Scala/C++ codegen visits the `ParallelPipe` block textually (`src/spatial/codegen/scalagen/ScalaGenController.scala:187-190`; `src/spatial/codegen/cppgen/CppGenAccel.scala:122-126`).

## `Fork`: Branch Fanout With OR Completion

`Fork` originates in conditionals. Non-literal staged `if` creates an `IfThenElse(cond, thenBlk, elseBlk)` (`src/spatial/lang/api/SpatialVirtualization.scala:93-100`; `argon/src/argon/node/IfThenElse.scala:7-8`). In hardware, `SwitchTransformer` rewrites this into `Switch(selects, body)` plus `SwitchCase` nodes, recursively flattening else-if chains and path-gating selectors as `cond2 & prevCond` or `!cond2 & prevCond` (`src/spatial/transform/SwitchTransformer.scala:52-84`). The `Switch` body uses a custom scheduler that keeps only `SwitchCase` nodes inside the switch and motions other statements out (`src/spatial/node/Switch.scala:8-29`, `src/spatial/node/Switch.scala:62-69`).

The likely operational definition is "enable the selected branch case(s), and consider the fork complete when any enabled case completes." Chisel `Fork` sets child activity with `selectsIn(i)`, computes `synchronize` as OR of child `doneIn`, and sets all `done(i)` from that OR (`fringe/src/fringe/templates/Controllers.scala:173-187`). The Scala executor is stricter/priority-like: it finds the first true selector and executes only that `SwitchCase` (`src/spatial/executor/scala/ControlExecutor.scala:961-972`). Because `SwitchTransformer` generates mutually exclusive path conditions, ordinary user `if/else` should behave one-hot. Unverified inference: HLS should document `Fork` as one-hot branch selection or explicitly reject multi-true selectors, since existing hardware and executor behavior diverge if multiple cases are true.

## `PrimitiveBox`: Primitive Module Boundary, Not Child Control

`PrimitiveBox` comes from `Blackbox.SpatialPrimitive`, which captures a lambda and stages `SpatialBlackboxImpl`; `Blackbox.SpatialController` instead stages `SpatialCtrlBlackboxImpl` (`src/spatial/lang/Blackbox.scala:40-60`). The distinction is semantic in the IR: primitive blackbox uses are `Primitive`s, while control blackbox uses are `EnControl`s with mutable effects; both implementation nodes are allocations binding the lambda input (`src/spatial/node/Blackbox.scala:37-58`). Flow assigns `SpatialBlackboxImpl` to `PrimitiveBox`, but assigns `SpatialCtrlBlackboxImpl` to `Sequenced`; the control-blackbox path also requires contained controllers and marks the implementation outer-level (`src/spatial/flows/SpatialFlowRules.scala:77-81`, `src/spatial/flows/SpatialFlowRules.scala:331-335`).

Chisel generation reinforces this reading. `SpatialBlackboxImpl` emits a standalone module class with struct input/output ports, generates the lambda body inside it, and wires the lambda result to output fields; `SpatialBlackboxUse` instantiates that module and concatenates outputs (`src/spatial/codegen/chiselgen/ChiselGenBlackbox.scala:166-238`). Retiming treats the implementation as an accel-like region and records `bodyLatency = func.result.fullDelay` (`src/spatial/traversal/RetimingAnalyzer.scala:112-115`). There is no corresponding Fringe controller schedule named `PrimitiveBox` next to `Fork`/`ForkJoin` (`fringe/src/fringe/templates/Controllers.scala:62-68`). Unverified inference: HLS should treat `PrimitiveBox` as a module/function implementation boundary with latency, not as a runtime controller with children or a fork/join protocol.

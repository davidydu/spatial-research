# D-14 Angle 1: Control Metadata and Effective Schedule Semantics

## Bottom line

For the Rust/HLS scheduler, keep `Fork`, `ForkJoin`, and `PrimitiveBox` as distinct `CtrlSchedule` variants in metadata, but do not treat all three as the same kind of executable controller policy. Spatial's source enum makes them peers of `Sequenced`, `Pipelined`, and `Streaming` (`src/spatial/metadata/control/ControlData.scala:7-14`), and the effective `schedule` helper preserves all three under every `level`/`looping` combination (`src/spatial/metadata/control/package.scala:193-219`). Operationally, though, `ForkJoin` is a parallel barrier region, `Fork` is a mutually exclusive branch region, and `PrimitiveBox` is an atomic primitive-blackbox boundary whose schedule value is metadata, not a child-stage ordering rule.

This should be linked back to [[10 - Control]]: that note already records the three-axis control model of schedule, level, and looping, and the `Ctrl.Node(..., -1)` master convention. D-14 should refine that into a Rust contract: raw metadata records what classification pass assigned; effective metadata is what scheduler predicates and most analyses ask.

## Raw and effective schedule

`ControlSchedule` is compiler-set metadata, while `UserScheduleDirective` is user-set metadata (`src/spatial/metadata/control/ControlData.scala:57-74`). The symbol helpers expose `getRawSchedule`, throwing `rawSchedule`, scratchpad-writing `rawSchedule_=`, and metadata-writing `finalizeRawSchedule`; reads prefer the scratchpad value when present (`src/spatial/metadata/control/package.scala:785-789`). The Rust port should model that as an explicit staged assignment/finalization path, not as an ordinary immutable field, because flow rules can normalize schedules before later consumers run.

The effective `schedule` helper is the canonical query. If no raw schedule exists, it returns `Sequenced` (`src/spatial/metadata/control/package.scala:196-198`). If the control is `Looped`, it returns the raw schedule unchanged for both `Inner` and `Outer` levels (`src/spatial/metadata/control/package.scala:198-200`). If the control is `Single`, it collapses only `Pipelined` to `Sequenced`; `Streaming`, `ForkJoin`, `Fork`, and `PrimitiveBox` survive unchanged for both inner and outer controls (`src/spatial/metadata/control/package.scala:201-216`). The separate flow rule also mutates raw `Pipelined` to `Sequenced` for single controls, single-child outers without primitives, and copied `Fork` metadata on `UnitPipe` (`src/spatial/flows/SpatialFlowRules.scala:364-368`). Do not revive an inferred pipeline after either collapse.

## ForkJoin and Fork

`ForkJoin` is assigned to `ParallelPipe` before generic control scheduling (`src/spatial/flows/SpatialFlowRules.scala:331-333`). Hardware semantics match a fork-barrier: all stages can become active together, synchronization is the AND of all per-stage `iterDone`, and the parent counter increments only at that synchronization point (`fringe/src/fringe/templates/Controllers.scala:141-153`). The runtime model likewise charges `ForkJoin` as startup/shutdown plus the maximum child cost per iteration, not the sum (`models/src/models/RuntimeModel.scala:321`). Rust/HLS should therefore lower it as parallel child launch plus per-iteration join, with latency dominated by the slowest child.

`Fork` is assigned to `Switch` and `IfThenElse`, while `SwitchCase` itself is sequenced (`src/spatial/flows/SpatialFlowRules.scala:333-337`). Its controller synchronization is OR over child `doneIn`, and each child only activates when its `selectsIn(i)` is asserted (`fringe/src/fringe/templates/Controllers.scala:173-187`). The runtime model uses branch duty-cycle weights over children, reinforcing that branches are alternatives rather than parallel workers (`models/src/models/RuntimeModel.scala:325-327`). Rust/HLS should lower `Fork` as selected-control execution: issue one branch/case for an iteration, complete the parent when the selected branch completes, and never require an all-child barrier.

## PrimitiveBox

`PrimitiveBox` is assigned only to `SpatialBlackboxImpl` in the flow schedule rule (`src/spatial/flows/SpatialFlowRules.scala:331-335`). The control hierarchy maps blackbox implementations to `Ctrl.SpatialBlackbox`, whose `mayBeOuterBlock` is false (`src/spatial/metadata/control/Ctrl.scala:32-35`), and `toCtrl` special-cases blackbox implementations to that control identity (`src/spatial/metadata/control/package.scala:752-754`). Effective level therefore treats spatial blackboxes as inner, regardless of raw outer metadata (`src/spatial/metadata/control/package.scala:161-167`).

Primitive blackboxes do not expose ordinary controller children: `Ctrl.SpatialBlackbox` returns `Seq()` for `SpatialBlackboxImpl`, while control blackboxes can return raw children (`src/spatial/metadata/control/package.scala:907-910`). Chisel primitive blackbox codegen emits a module wrapper and the user function body, not an `OuterControl` using `PrimitiveBox` (`src/spatial/codegen/chiselgen/ChiselGenBlackbox.scala:166-210`). So in Rust/HLS, `PrimitiveBox` should mean "opaque primitive module with declared ports/effects/latency," not "run a PrimitiveBox schedule machine." It should be schedulable as an atomic node inside its parent.

## Scheduler predicates and metadata dependencies

`isCtrl` composes `isControl`, effective `looping`, effective `level`, and effective `schedule`; all schedule predicates built on it observe effective schedules, not raw ones (`src/spatial/metadata/control/package.scala:221-238`). There is a named `isForkJoinControl`, but no matching named `isForkControl` or `isPrimitiveBoxControl`; raw helpers only provide `isForkJoin` and `isFork` over `rawSchedule` (`src/spatial/metadata/control/package.scala:257-265`, `src/spatial/metadata/control/package.scala:790-794`). Rust should avoid scattering raw enum checks where an effective predicate is intended.

Parent/child metadata is part of the schedule contract. `Children` stores controller children; `ParentCtrl` defaults to `Host` and records the containing controller (`src/spatial/metadata/control/ControlData.scala:77-99`, `src/spatial/metadata/control/package.scala:807-814`). Master controllers expand body stages, flatten pseudostages into scoped raw children, and Host exposes all accel scopes (`src/spatial/metadata/control/package.scala:886-905`). Because `looping` treats only non-fully-unrolled loops as `Looped` (`src/spatial/metadata/control/package.scala:25-34`, `src/spatial/metadata/control/package.scala:188-192`), the Rust scheduler should compute schedule semantics after hierarchy, level, and unroll facts are known.

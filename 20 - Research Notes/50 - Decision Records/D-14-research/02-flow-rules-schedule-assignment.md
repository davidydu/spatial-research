**Assignment Surface.**

Spatial's schedule domain is a small closed enum: `Sequenced`, `Pipelined`, `Streaming`, `ForkJoin`, `Fork`, and `PrimitiveBox` (`src/spatial/metadata/control/ControlData.scala:7`). `controlSchedule` is not a pure defaulting pass; it first hard-codes several node classes, then falls back to user/compiler/default schedules (`src/spatial/flows/SpatialFlowRules.scala:317`). The hard assignments are: `ParallelPipe` -> `ForkJoin`, `Switch` -> `Fork`, `SpatialBlackboxImpl` -> `PrimitiveBox`, `SpatialCtrlBlackboxImpl` -> `Sequenced`, `IfThenElse` -> `Fork`, `SwitchCase` -> `Sequenced`, and both dense and sparse transfers -> `Pipelined` (`src/spatial/flows/SpatialFlowRules.scala:331`). Only ordinary `Control[_]` nodes enter the fallback ladder: user schedule wins if no compiler schedule exists; otherwise UnitPipe/Accel default to `Sequenced`, and other controls default to `Pipelined` (`src/spatial/flows/SpatialFlowRules.scala:346`, `src/spatial/flows/SpatialFlowRules.scala:350`).

**Fork.**

`Fork` is the branch schedule. It is attached both to raw `IfThenElse` and to the structured `Switch` node (`src/spatial/flows/SpatialFlowRules.scala:333`, `src/spatial/flows/SpatialFlowRules.scala:336`). Hardware `IfThenElse` is commonly rewritten into `Switch`/`SwitchCase`: the transformer creates case bodies under condition enables, flattens else-if chains when motion is legal, and finally stages `Switch.op_switch(selects, cases)` (`src/spatial/transform/SwitchTransformer.scala:44`, `src/spatial/transform/SwitchTransformer.scala:59`, `src/spatial/transform/SwitchTransformer.scala:84`). The `Switch` scheduler keeps only `SwitchCase` statements in the switch body and motions other operations out, so the branch controller owns cases rather than arbitrary condition-computation logic (`src/spatial/node/Switch.scala:8`, `src/spatial/node/Switch.scala:23`). In the Chisel template, outer `Fork` sets a child active only when its `selectsIn(i)` is true, synchronizes on OR of child done signals, and marks all cases done from that single synchronize event (`fringe/src/fringe/templates/Controllers.scala:173`, `fringe/src/fringe/templates/Controllers.scala:178`, `fringe/src/fringe/templates/Controllers.scala:182`). Inner `Fork` similarly declares done when any child is done (`fringe/src/fringe/templates/Controllers.scala:269`). The runtime model mirrors this as a branch mix: it asks for duty cycles for all but the last branch and computes a weighted child runtime (`models/src/models/RuntimeModel.scala:325`).

**ForkJoin.**

`ForkJoin` comes from `ParallelPipe`, whose source-level constructor is `Parallel { ... }`: it stage-blocks the body and stages `ParallelPipe(Set.empty, block)` directly (`src/spatial/lang/control/Parallel.scala:9`, `src/spatial/lang/control/Parallel.scala:12`). It is also produced by compiler transformations, especially lane/control duplication. The generic unroller wraps multiple duplicated controllers in a `ParallelPipe` when the lane count is greater than one (`src/spatial/transform/unrolling/UnrollingBase.scala:147`, `src/spatial/transform/unrolling/UnrollingBase.scala:164`). Foreach unrolling also wraps outer non-MoP lanes in `ParallelPipe` for both fully and partially unrolled forms (`src/spatial/transform/unrolling/ForeachUnrolling.scala:39`, `src/spatial/transform/unrolling/ForeachUnrolling.scala:48`, `src/spatial/transform/unrolling/ForeachUnrolling.scala:72`, `src/spatial/transform/unrolling/ForeachUnrolling.scala:83`). Parallel binding can likewise bundle several sibling controllers under a generated `ParallelPipe` (`src/spatial/transform/BindingTransformer.scala:80`, `src/spatial/transform/BindingTransformer.scala:82`). Operationally, the outer Chisel `ForkJoin` starts every unfinished child stage, waits until all `iterDone` bits are set, and increments the parent counter only at that all-child synchronize point (`fringe/src/fringe/templates/Controllers.scala:141`, `fringe/src/fringe/templates/Controllers.scala:146`, `fringe/src/fringe/templates/Controllers.scala:149`). The performance model treats it as max-child latency per parent iteration plus synchronization, not sum-child latency (`models/src/models/RuntimeModel.scala:321`).

**PrimitiveBox, Transfers, and Collapses.**

`PrimitiveBox` is produced by `Blackbox.SpatialPrimitive`, which stages a `SpatialBlackboxImpl` from a one-argument lambda (`src/spatial/lang/Blackbox.scala:40`, `src/spatial/lang/Blackbox.scala:46`). That implementation is represented as a blackbox control-like scope, but it has no child controllers (`src/spatial/metadata/control/package.scala:753`, `src/spatial/metadata/control/package.scala:908`). Its Chisel path emits a standalone module from the lambda body and blackbox uses instantiate that module, rather than constructing an `OuterControl(PrimitiveBox, ...)` state machine (`src/spatial/codegen/chiselgen/ChiselGenBlackbox.scala:166`, `src/spatial/codegen/chiselgen/ChiselGenBlackbox.scala:191`, `src/spatial/codegen/chiselgen/ChiselGenBlackbox.scala:229`). By contrast, `Blackbox.SpatialController` stages `SpatialCtrlBlackboxImpl`, requires at least one child in flow-level control analysis, and is scheduled `Sequenced` (`src/spatial/lang/Blackbox.scala:54`, `src/spatial/flows/SpatialFlowRules.scala:77`, `src/spatial/flows/SpatialFlowRules.scala:335`).

Dense and sparse transfers are early blackboxes structurally, but the schedule rule classifies them as `Pipelined`, not `PrimitiveBox` (`src/spatial/node/DenseTransfer.scala:34`, `src/spatial/node/DenseTransfer.scala:44`, `src/spatial/node/SparseTransfer.scala:27`, `src/spatial/node/SparseTransfer.scala:35`, `src/spatial/flows/SpatialFlowRules.scala:338`). The main correction rules then collapse illegal or unhelpful raw schedules: any single control with raw `Pipelined` becomes `Sequenced`; an outer controller with exactly one child and no non-transient primitive work also becomes `Sequenced`; and a `UnitPipe` that inherited `Fork` metadata from switch pipe insertion is reset to `Sequenced` (`src/spatial/flows/SpatialFlowRules.scala:359`, `src/spatial/flows/SpatialFlowRules.scala:364`, `src/spatial/flows/SpatialFlowRules.scala:367`, `src/spatial/flows/SpatialFlowRules.scala:368`). The accessor-level effective schedule repeats the key single-control collapse by mapping `Pipelined` to `Sequenced` for both inner and outer `Single` controls while leaving `Fork`, `ForkJoin`, and `PrimitiveBox` intact (`src/spatial/metadata/control/package.scala:193`, `src/spatial/metadata/control/package.scala:201`, `src/spatial/metadata/control/package.scala:209`).

# D-22 - FIFO/LIFO/Stream Back-Pressure Semantics

## Question

Choose FIFO/LIFO/stream elastic simulation versus bounded back-pressure semantics.

Source: `spatial/src/spatial/codegen/scalagen/ScalaGenFIFO.scala:41-44`; `spatial/src/spatial/codegen/scalagen/ScalaGenLIFO.scala:39-44`; `spatial/src/spatial/codegen/scalagen/ScalaGenStream.scala:92-97`; [[40 - FIFO LIFO Stream Simulation]]

Decision criteria: User decision: simulator and HLS semantics use Scalagen elastic queues or bounded FIFO/LIFO/stream back-pressure with overflow checks.

## Research findings

- ScalaGen is an elastic value simulator. FIFO and streams lower to `mutable.Queue`, LIFO lowers to `mutable.Stack`, full/empty are observable size predicates, and enabled enqueue/push/write does not enforce capacity. Empty reads return type-specific `invalid` values instead of stalling. See [[01-source-scalagen-semantics]].

- RTL/Chisel is bounded and pressure-driven. FIFO/LIFO templates are finite memories with counters and empty/full signals; streams use ready/valid; controller forward/back pressure gates progress; FIFO pressure is delayed by controller II. LIFO has bounded state, but the visible pressure filters are less complete than FIFO/stream. See [[02-chisel-rtl-backpressure]].

- The test corpus mostly protects final values and status readers, not overflow or liveness. Some tests are elastic or ambiguous, such as pushing 32 values into a depth-5 FIFO or loading a larger frame through a smaller FIFO without an explicit concurrent flow model. Streaming and `ParallelPipe` tests expose the need for bounded stalls, but not as a settled fixture suite. See [[03-tests-app-usage]].

- HLS mappings should distinguish local queues from external streams. Local FIFO can lower to bounded stream/dataflow channels or ring buffers; LIFO should lower to an explicit bounded stack or be rejected in HLS mode, not silently mapped to FIFO-ordered streams. AXI streams are external endpoint mappings, not a replacement for local FIFO/LIFO semantics. See [[04-hls-stream-fifo-mapping]].

- D-22 composes directly with D-15, D-20, and D-21. Bounded queues need a parent/child tick and same-cycle rule from D-15, delay-line advance must share the accepted controller step from D-20, and bounded traces must name their II source under D-21. See [[05-overlap-d15-d20-d21]].

- Elastic-only has low migration cost but hides overflow and deadlock. Bounded-only gives better hardware truth but risks breaking legacy goldens and inventing timing precision before D-15/D-20/D-21 are fully settled. HLS-report-only arrives too late for source-level liveness debugging. See [[06-risk-failure-modes]].

- Migration should be staged. Label current behavior first, add schemas and diagnostics, implement bounded simulation behind an opt-in flag, then add HLS bounded lowering and report linkage. Existing Scala executor code is useful but incomplete: it has capacity checks and stall logic for some queues, but unbounded streams and missing LIFO pop behavior remain. See [[07-implementation-cost-migration]].

- Validation needs explicit policy fixtures: overflow, underflow, depth-one same-cycle transfer, multi-lane atomicity, faster producer, faster consumer, cyclic dataflow deadlock, LIFO stack ordering, and stream host protocol stalls. Diagnostics should report queue policy, scheduler policy, occupancy, blocked reason, controller path, and II source. See [[08-validation-fixtures-diagnostics]].

- The policy contract should define claim boundaries. `compat_scalagen_elastic` may claim ScalaGen value parity, but not boundedness, hardware liveness, cycle accuracy, or HLS timing. `bounded_backpressure` may claim finite-capacity protocol behavior only with declared scheduler, same-cycle rule, max-cycle/deadlock policy, and II source. See [[09-policy-options-contract]].

- The recommendation matrix selects dual mode. It preserves the useful legacy oracle while adding a bounded lane for HLS/RTL validation and deadlock visibility. It rejects elastic-only, bounded-only default, HLS-native/report-only, and unlabelled mixed modes. See [[10-recommendation-matrix]].

## Recommendation

Adopt **Dual-Mode Queue Semantics With Elastic Reference And Bounded HLS Lane**.

The Rust simulator and HLS flow should expose two named policies:

1. `compat_scalagen_elastic`
2. `bounded_hls_cycle`

`compat_scalagen_elastic` is the default reference lane for legacy functional comparison and ScalaGen parity. It should preserve current value behavior: source/IR-order execution, unbounded host queue/stack growth, advisory `isFull`/`isAlmostFull`, `invalid` on empty reads, no overflow failure, no cycle claim, and no deadlock claim. This lane can report `elastic_over_capacity` and high-water marks as diagnostics, but those diagnostics do not make the run a bounded-protocol proof.

`bounded_hls_cycle` is the HLS-facing validation lane. It uses finite capacity from staged size, explicit stream policy, or host ABI manifest; full writes and empty reads become stalls or explicit contract errors, never silent growth or invalid payload values. Same-cycle multi-lane enqueue/dequeue and push/pop should use a deterministic snapshot/commit rule: evaluate requests against pre-state, commit only the set of lanes whose aggregate reads and writes fit the bounded state transition, and record accepted versus blocked lanes. This avoids making bounded behavior depend on textual ScalaGen order.

Bounded mode may claim hardware/liveness meaning only when it also declares:

- `queue_policy`
- `scheduler_policy`
- `same_cycle_rule`
- `capacity_source`
- `overflow_policy`
- `underflow_policy`
- `deadlock_policy`
- `max_cycles`
- `delayline_policy`
- `ii_source`
- `ii_value`

For D-15, bounded queue transfer must be tied to the HLS-semantic peer/barrier stepping model rather than the serial ScalaGen model. For D-20, delay lines advance only on the same accepted controller step as bounded queues. For D-21, any queue trace must state whether its II came from selected/compiler II, an emitted HLS request, or parsed HLS accepted II. A bounded trace generated before reports exist remains a labelled estimate; after reports exist, stale or mismatched II should relabel the trace rather than silently bless it.

HLS lowering should use bounded semantics. Local FIFO should lower to a bounded channel or explicit ring buffer with overflow/underflow diagnostics. Local LIFO should lower to an explicit bounded stack or be rejected as unsupported for the target; it should not be mapped to FIFO-ordered streams. External StreamIn/StreamOut should use the host ABI and endpoint protocol, with ready/valid or equivalent back-pressure. HLS reports and co-simulation are final evidence for scheduled artifacts, but they do not replace the need for pre-HLS bounded diagnostics.

### Rejected alternatives

Reject **elastic-only as the D-22 answer**. It is the correct ScalaGen compatibility oracle, but it hides overflow, starvation, and deadlock, and cannot support HLS liveness claims.

Reject **bounded-only as the default Rust simulator**. It would break legacy ScalaGen value goldens and would prematurely force D-15/D-20/D-21 timing choices into every functional test.

Reject **HLS-native/report-only**. Reports and co-simulation are post-tool evidence, but they arrive too late to explain which source-level queue operation first blocked, overflowed, or deadlocked.

Reject **unlabelled mixed semantics**. A run must not say "passed" without declaring whether it passed as Scalagen value reference, bounded protocol diagnostic, or HLS report-backed artifact.

## Decision

Status: awaiting-user-confirmation.

If confirmed, follow-up implementation tasks:
1. Define `queue_policy` values: `compat_scalagen_elastic`, `bounded_hls_cycle`, `hls_report_authority`, and `incomparable_policy`.
2. Implement Rust elastic FIFO/LIFO/stream behavior matching ScalaGen collections, including advisory full and invalid-on-empty reads.
3. Add optional elastic diagnostics for high-water mark and `elastic_over_capacity` without changing value-reference behavior.
4. Implement bounded FIFO ring buffers, bounded LIFO stacks, and bounded stream endpoints with explicit capacity, occupancy, and blocked-event records.
5. Define the snapshot/commit rule for same-cycle multi-lane enqueue/dequeue and push/pop.
6. Add deadlock/timeout diagnostics with wait-for sets, last-progress cycle, blocked channel, producer, consumer, and controller path.
7. Require bounded mode to declare D-15 scheduler policy, D-20 delay-line policy, and D-21 II source/value.
8. In HLS lowering, map FIFO to bounded channel/ring-buffer forms, map LIFO to bounded stack or reject, and map external streams through the host ABI.
9. Add fixtures for overflow, underflow, depth-one same-cycle transfer, multi-lane atomicity, faster producer/consumer, cyclic deadlock, LIFO order, and stream host stalls.
10. Label every simulator result and HLS artifact as `scalagen_value_reference`, `bounded_protocol_diagnostic`, `hls_report_authority`, or `incomparable_policy`.

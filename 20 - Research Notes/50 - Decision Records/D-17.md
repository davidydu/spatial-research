# D-17 — Canonical Unbiased Rounding Policy

## Question

Choose the canonical unbiased rounding policy.

Source: `spatial/emul/src/emul/FixedPoint.scala:232-241`; [[20 - Numeric Reference Semantics]]

Decision criteria: User decision: unbiased rounding matches nondeterministic Spatial behavior, becomes seedable, or is replaced by deterministic rounding.

## Research findings

- D-17 is the language-level companion to [[D-09]], not a duplicate Rust-simulator decision. D-09 already recommends **Seeded Stable Stochastic Rounding** for Rust simulation and rejects JVM RNG sequence compatibility, but it deliberately leaves the canonical meaning of `Unbiased` to D-17. See [[01-d09-handoff-scope]].

- The current call surface is stochastic and broader than one helper. `*&`, `/&`, `*&!`, `/&!`, `FixToFixUnb`, and `FixToFixUnbSat` all reach `FixedPoint.unbiased` through Scalagen or the Scala executor, while constant folding has known mismatches. The helper always assumes four guard bits, computes a sixteenth-granularity remainder, calls `scala.util.Random.nextFloat()`, then wraps or saturates the chosen result. See [[02-current-spec-call-surface]].

- Existing hardware evidence supports stochastic intent, not deterministic arithmetic rounding. ChiselGen lowers unbiased multiply, divide, and fixed casts to `Unbiased`, while Fringe documents `Unbiased` as LFSR stochastic rounding and implements fractional shrink by adding PRNG low bits before shaving. However, current PRNG seed and advance policy are not canonical: seeds are chosen during elaboration, and some paths use literal random folding rather than runtime stochastic rounding. See [[03-hardware-chisel-fringe-rounding]].

- Deterministic alternatives are viable only as explicit redefinitions. Round-to-nearest-even is the cleanest deterministic candidate and maps well to common HLS fixed-point libraries, while half-up, half-away, and truncation are weaker candidates for something named `Unbiased`. Stable-hash stochastic rounding is the strongest deterministic-in-replay option because it preserves the probability law while removing sequence nondeterminism. See [[04-deterministic-alternatives-implications]].

- Tests and apps constrain distributional behavior, not JVM sequence parity. `SpecialMath` samples `*&` repeatedly and checks mean error plus saturation endpoints. SGD/SVRG-style apps use unbiased fixed arithmetic in convergence checks rather than exact bit comparisons. Coverage gaps remain for `/&!`, unbiased casts, signed edge cases, seed replay, and event/lane independence. See [[05-tests-apps-statistical-expectations]].

- HLS libraries make deterministic rounding easy but do not make stochastic `Unbiased` a free type-parameter choice. A portable HLS path needs an explicit Spatial primitive such as a seeded hash, valid-gated LFSR, or equivalent stochastic source, with seed and advance policy recorded. Otherwise unsupported stochastic rounding should be rejected or marked as a non-canonical fallback. See [[06-hls-library-vendor-support]].

- A cross-backend provenance schema is the necessary bridge from D-09 to D-17. The recommended schema records `rounding_policy`, seed mode/value/scope, algorithm name/version/guard bits, signed-negative behavior, stochastic source, event identity, backend capability, and fallback. Its canonical v1 label is `seeded_stochastic_legacy_away`. See [[07-cross-backend-provenance-schema]].

- Migration cost favors seedable stochastic rounding. Keeping ambient nondeterminism is cheap only for Scala emul and expensive everywhere else. Deterministic rounding is cheap for new backends but costly to justify, rename, and retest. Seeded stable stochastic requires metadata and HLS primitive work, but preserves current semantics while making CI and debugging sane. See [[08-migration-update-cost]].

- Risk analysis identifies silent numeric divergence as the worst failure mode. Ambient nondeterminism risks irreproducible failures; deterministic replacement risks semantic camouflage under the familiar `Unbiased` name; HLS fallback risks simulator/kernel mismatch unless declared. Seeded stable stochastic has more moving parts, but it makes the policy auditable. See [[09-risk-failure-modes]].

- The recommendation matrix converges on **seeded stable stochastic rounding** as canonical. Ambient JVM nondeterminism is rejected as a language rule, deterministic modes are rejected as the meaning of `Unbiased`, and HLS approximations are allowed only as declared non-canonical fallbacks. See [[10-recommendation-matrix]].

## Recommendation

Adopt **Canonical Seeded Stable Stochastic Rounding**, with v1 policy label:

`rounding_policy = seeded_stochastic_legacy_away`

This policy preserves the current Spatial probability law: unbiased operations carry four guard fractional bits, compute the discarded remainder, and round by comparing that remainder against a random threshold. It also preserves the current v1 signed behavior: when the stochastic threshold fires, nonnegative shifted values move to `biased + 1` and negative shifted values move to `biased - 1`. That legacy-away rule is not mathematically pretty, but it is the behavior actually exposed by `FixedPoint.unbiased`, so changing it should be a later versioned numeric migration rather than a hidden cleanup.

The random threshold must become a pure, replayable function of declared provenance:

`(seed, algorithm_version, static_op_id, dynamic_instance_id, lane_or_element_id)`

Every backend artifact should record the rounding schema version, seed mode and value, algorithm name/version, guard-bit count, stochastic source, signed-negative behavior, backend capability, and fallback. Rust simulation should implement the canonical mode with stable event-derived thresholds. Scala compatibility may keep the old ambient JVM stream only under a legacy label. HLS and RTL should either implement the canonical stochastic rule, declare a statistical-equivalent stochastic source with seed and advance policy, or reject/mark fallback behavior.

This intentionally ratifies D-09 at the language level while extending it beyond Rust. D-09 selected seeded stable stochastic behavior as the Rust simulator default; D-17 makes the same statistical rule the canonical meaning of `Unbiased`, with cross-backend provenance and capability gates.

### Rejected alternatives

Reject **ambient JVM nondeterminism** as canonical. It preserves an implementation artifact, not a language rule. It makes failures hard to replay, depends on evaluation order, and does not map cleanly to HLS or hardware.

Reject **JVM sequence compatibility**. The desired compatibility is statistical Spatial parity and replayable provenance, not matching a process-global Scala RNG stream.

Reject **round-to-nearest-even, half-up, half-away, or truncation** as the default meaning of `Unbiased`. Round-to-nearest-even may be an excellent separate deterministic mode or future language migration, but adopting it here would redefine current Scalagen, emul, and Fringe behavior. Half-up and truncation are even weaker because they do not preserve the mean-preserving stochastic intent.

Reject **silent HLS deterministic fallback**. A target that cannot implement canonical stochastic rounding may provide a named fallback such as `deterministic_rne`, but generated artifacts and comparison tools must treat that as non-canonical.

## Decision

Status: awaiting-user-confirmation.

If confirmed, follow-up implementation tasks:
1. Define the `spatial.rounding.v1` provenance schema with `rounding_policy`, seed fields, algorithm version, signed-negative behavior, stochastic source, event identity, backend capability, and fallback.
2. Implement `seeded_stochastic_legacy_away` in the Rust simulator as a stable event-derived threshold rule.
3. Preserve the four-guard-bit probability law, wrap/saturate split, invalid propagation, and legacy negative threshold behavior in v1.
4. Assign stable event identities for unbiased arithmetic and fixed-cast sites, including dynamic controller/iteration and lane or element identity.
5. Add run metadata and mismatch reporting for resolved seed, algorithm, event identity, backend capability, and fallback.
6. Add Scala compatibility support: either context-aware seeded `FixedPoint.unbiased` or an explicit `legacy_host_global_rng` mode.
7. Add HLS/RTL lowering support for canonical stochastic rounding, or capability-gated rejection/fallback with visible diagnostics.
8. Add tests for fixed-seed replay, changed-seed variation, order/thread independence, signed negative cases, saturation, division, casts, and HLS fallback/rejection.
9. Keep deterministic RNE/truncation as explicit non-default modes, not as silent replacements for `Unbiased`.
10. Cross-reference D-19 so fused/unfused arithmetic choices define where stochastic rounding events occur.

# RuntimeModel equations and HLS parity limits

## Controller shape

Spatial emits a standalone Scala app whose `build_model()` returns a `ControllerModel`; the generated app initializes ask maps, applies retuning maps, prints structure/results, executes the model, and records total cycles (`src/spatial/model/RuntimeModelGenerator.scala:111-156`). Each controller carries an `id`, `CtrlLevel`, schedule, one or more counter chains, body latency `L`, initiation interval `II`, source context, and optional `bitsPerCycle` for transfers (`models/src/models/RuntimeModel.scala:192-200`). The schedule vocabulary is Spatial-specific: `Sequenced`, `Pipelined`, `Streaming`, `ForkJoin`, `Fork`, plus memory transfers modeled as schedules such as `DenseLoad`, `DenseStore`, `GatedDenseStore`, `SparseLoad`, and `SparseStore` (`models/src/models/RuntimeModel.scala:29-40`). Context records the Spatial symbol, source line, source snippet, and IR statement, so the model is meant to explain compiler IR structure rather than consume vendor report rows (`models/src/models/RuntimeModel.scala:76-84`).

The generator emits different controller nodes for accel scopes, foreach/reduce/unit pipes, mem-reduces, transfers, FSMs, switches, and switch cases (`src/spatial/model/RuntimeModelGenerator.scala:261-414`). Parent/child hierarchy is reconstructed by visiting blocks and registering children after each body is emitted (`src/spatial/model/RuntimeModelGenerator.scala:266-268`, `src/spatial/model/RuntimeModelGenerator.scala:285-286`).

## Cycle equations

Counter chains determine per-lane iterations. A counter rounds iteration count up to a multiple of parallelism and divides by `par`; a chain multiplies counter `N`s and treats unroll as the product of `par` values except in final mode (`models/src/models/RuntimeModel.scala:165-188`). The runtime then computes `cycsPerParent` by controller level and resolved schedule (`models/src/models/RuntimeModel.scala:312-338`). Outer sequenced controllers sum children per iteration with startup/shutdown, child sync, and sequence-advance overheads; multilevel chains use `cchain.head.N` as the outer count and `cchain.last.N` as inner work (`models/src/models/RuntimeModel.scala:315-320`). Pipelined outer controllers use `maxChild` over repeated iterations plus a one-time sum of child costs and `metaSync`; streaming/fork-join use max child; fork uses user-asked duty-cycle percentages (`models/src/models/RuntimeModel.scala:318-327`).

Inner controllers are where emitted `L` and `II` matter: sequenced inner control is `cchainIters * L + startup + shutdown`, while other inner schedules are `(cchainIters - 1) * II + L + startup + shutdown + dpMask` (`models/src/models/RuntimeModel.scala:334-337`). Execution is a DFS: each child inherits `parent.num_iters * parent.cchainIters`, then `num_cycles = cycsPerParent * num_iters` (`models/src/models/RuntimeModel.scala:340-347`).

## `bodyLatency`, `II`, and transfer model

For most non-transfer control nodes, generated `L` is `scrubNoise(lhs.bodyLatency.sum)` only if the node is inner control; outer controls get `0`. Generated `II` becomes `1` when `lhs.II <= 1` or the node is outer control, otherwise `scrubNoise(lhs.II)` (`src/spatial/model/RuntimeModelGenerator.scala:262-265`, `src/spatial/model/RuntimeModelGenerator.scala:279-284`, `src/spatial/model/RuntimeModelGenerator.scala:328-333`). FSMs, switches, and switch cases add an unexplained `+ 2` to both latency and II, explicitly marked as “invisible latency” (`src/spatial/model/RuntimeModelGenerator.scala:373-379`, `src/spatial/model/RuntimeModelGenerator.scala:398-411`).

Dense transfers are modeled either from lowered transfer metadata or `DenseTransfer` nodes. Lowered transfers add a last counter from `loweredTransferSize`, set `bitsPerCycle` from metadata, and use dense/gated load/store schedules (`src/spatial/model/RuntimeModelGenerator.scala:270-278`, `src/spatial/model/RuntimeModelGenerator.scala:312-318`). The runtime counts competing dense loads, stores, and gated stores through the controller tree, scaling by ancestor unroll and excluding sequential siblings (`models/src/models/RuntimeModel.scala:226-241`). `congestionModel` burst-aligns transfer length to 512 bits, looks up curve-fit coefficients by resolved schedule, and returns the max of a floor expression and the fitted estimate (`models/src/models/RuntimeModel.scala:244-276`). Sparse transfers are placeholders returning `1` cycle (`models/src/models/RuntimeModel.scala:328-332`).

## Tunables, asks, and generated assumptions

Tuneables are Spatial tile sizes, parallelization params, and pipeline params (`src/spatial/model/RuntimeModelGenerator.scala:31-35`). Tuneable schedules become a `Right(Tuneable(...))` that resolves to `Sequenced` when the lookup string is `"false"`, otherwise `Pipelined` (`src/spatial/model/RuntimeModelGenerator.scala:279-284`, `models/src/models/RuntimeModel.scala:214-224`). Unknown counter bounds, arg-ins, and params become `Ask(...)` entries, with generated defaults in `InitAskMap`; one TODO says the fallback default of `100` should be chosen/warned more deliberately (`src/spatial/model/RuntimeModelGenerator.scala:198-243`, `src/spatial/model/RuntimeModelGenerator.scala:159-168`). Runtime lookup can be interactive, noninteractive CLI-driven, or retuned by map (`models/src/models/RuntimeModel.scala:94-139`).

## HLS parity takeaway

Transferable pieces for HLS DSE are the controller-tree abstraction, loop tripcount/parallelism math, and the inner-loop `latency + (N - 1) * II` equation, provided HLS can supply equivalent tripcounts, hierarchy, latency, and II (unverified). The transfer/congestion equations are less portable: they depend on Spatial lowered-transfer metadata, a 512-bit burst assumption, `bitsPerCycle`, dense/gated schedule classes, and `ModelData.curve_fit` coefficients keyed by Spatial schedules (`src/spatial/model/RuntimeModelGenerator.scala:270-278`, `models/src/models/RuntimeModel.scala:244-276`). Non-transfer outer-control equations may transfer conceptually, but constants like `startup`, `shutdown`, `seqSync`, `metaSync`, `seqAdvance`, and `dpMask` are fixed Spatial/Chisel-runtime assumptions rather than HLS-report facts (`models/src/models/RuntimeModel.scala:205-212`). The major non-parity risks are explicit TODOs for last-level transfer counters, sparse transfers, upper-chain interpretation, tuneable symbol emission, arbitrary ask defaults, and invisible `+2` latency patches (`src/spatial/model/RuntimeModelGenerator.scala:40-45`, `src/spatial/model/RuntimeModelGenerator.scala:271-289`, `models/src/models/RuntimeModel.scala:296-299`).

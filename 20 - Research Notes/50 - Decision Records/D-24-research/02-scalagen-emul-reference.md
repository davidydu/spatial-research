# Scalagen and emul FixedPoint Reference

## Reference Surface

Scalagen does not emit native Scala integers for Spatial fixed point. It remaps every `Fix` value to `emul.FixedPoint` and quotes constants as `FixedPoint(BigDecimal("$c"), FixFormat(s,i,f))` (`spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:12-23`). The format itself is raw scaled-integer state: `FixFormat.bits = ibits + fbits`, with signed min/max in the same scaled domain (`spatial/emul/src/emul/FixFormat.scala:3-11`). This matters for D-24 because the semantic reference is not C++ `double`; it is `FixedPoint(value: BigInt, valid: Boolean, fmt: FixFormat)` (`spatial/emul/src/emul/FixedPoint.scala:3`).

## Construction and Text

Integer constructors shift by `fmt.fbits` before wrapping, so `FixedPoint(3, Qm.n)` stores `3 << fbits` (`spatial/emul/src/emul/FixedPoint.scala:150-154`). Float, double, BigDecimal, and string constructors multiply by `2^fbits` and enter `clamped(BigDecimal, ...)` (`spatial/emul/src/emul/FixedPoint.scala:156-159`), which converts to `BigInt` before wrapping and returns invalid only when parsing/number conversion throws `NumberFormatException` (`spatial/emul/src/emul/FixedPoint.scala:162-167`). Scalagen's stream parser and `TextToFix` both route strings through that same constructor (`spatial/src/spatial/codegen/scalagen/ScalaGenStream.scala:23-25`; `spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:122-124`). `toString` prints `"X"` when invalid, otherwise prints decimal `toPlainString` for fractional formats and the raw stored `value.toString` for `fbits == 0` (`spatial/emul/src/emul/FixedPoint.scala:105-112`); `FixToText` emits `$x.toString` (`spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:119-120`).

## Rounding, Invalid, and Casts

Ordinary fixed arithmetic is wraparound `clamped`: add/sub use raw integer addition, multiply shifts the product right by `fmt.fbits`, and divide shifts the numerator left before integer division (`spatial/emul/src/emul/FixedPoint.scala:14-17`). `valueOrX` catches thrown failures and returns `FixedPoint.invalid(fmt)` (`spatial/emul/src/emul/FixedPoint.scala:102-104`), implemented as raw value `-1` plus `valid=false` (`spatial/emul/src/emul/FixedPoint.scala:161`). Validity is ANDed through arithmetic and comparisons (`spatial/emul/src/emul/FixedPoint.scala:14-32`); `Bool` follows the same `(value, valid)` convention and prints invalid as `X` (`spatial/emul/src/emul/Bool.scala:3-17`). Saturating ops clip to format min/max (`spatial/emul/src/emul/FixedPoint.scala:218-222`) and Scalagen maps `Sat*` to `+!`, `-!`, `*!`, `/!` (`spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:92-95`). Unbiased ops consume four extra fractional bits, compare a 4-bit remainder with `scala.util.Random.nextFloat()`, then clamp or saturate (`spatial/emul/src/emul/FixedPoint.scala:232-241`; `spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:96-99`). Plain `FixToFix` shifts left or right by fractional-bit delta and re-clamps, so narrowing fractional precision truncates by arithmetic right shift (`spatial/emul/src/emul/FixedPoint.scala:90-94`; `spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:107-108`). The explicit saturating/unbiased casts instead call helpers on `$x.value` as emitted (`spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:109-114`).

## Bit Layout and Raw Helpers

Fixed bits are LSB first: `bits(i) = value.testBit(i)` (`spatial/emul/src/emul/FixedPoint.scala:45`). `fromBits(Array[Bool], fmt)` treats index `fmt.bits - 1` as the sign bit, sign-extends from `BigInt(-1)` when it is set, masks to `fmt.bits`, and sets validity to `bits.forall(_.valid)` (`spatial/emul/src/emul/FixedPoint.scala:171-188`). Bytes are also little-bit-order within each byte (`spatial/emul/src/emul/FixedPoint.scala:190-195`; `spatial/emul/src/emul/Number.scala:13-19`). Scalagen's `DataAsBits`, `BitsAsData`, `DataAsVec`, and `VecAsData` delegate to `.bits` and `FixedPoint.fromBits`, preserving that layout (`spatial/src/spatial/codegen/scalagen/ScalaGenBits.scala:47-74`). Cppgen mirrors the low-bit-first packing style in `DataAsBits`/`BitsAsData`, but it first calls `toTrueFix` for fractional values (`spatial/src/spatial/codegen/cppgen/CppGenArray.scala:111-151`). Struct `toRaw()` also shifts fields into increasing bit positions (`spatial/src/spatial/codegen/cppgen/CppGenArray.scala:175-188`).

## Rust Bit-Exact Implication

A Rust reference that claims emul parity should store `BigInt` raw values plus `valid` and `FixFormat`, implement `clamped`, `saturating`, `unbiased`, `fromBits`, `bits`, `toString`, and `toFixedPoint` from the cited Scala lines, and parse strings/doubles through a Scala-compatible decimal path rather than host `f64` scaling. Cppgen is a separate compatibility mode: fractional `FixPt` remaps to `double`, while `toTrueFix` multiplies by `1 << f` and casts to an integer, and `toApproxFix` divides raw integers back to approximate fixed values (`spatial/src/spatial/codegen/cppgen/CppGenCommon.scala:36-49`, `spatial/src/spatial/codegen/cppgen/CppGenCommon.scala:75-89`). Host args repeat that approximate conversion: set multiplies into `int64_t`; get sign-extends and divides (`spatial/src/spatial/codegen/cppgen/CppGenInterface.scala:42-80`). Therefore D-24's bit-exact path should be named `scalagen_emul_reference`, while the Cppgen path should be named as a double/shift ABI compatibility policy. D-17 remains the hard caveat: unbiased rounding is only bit-exact if Rust also controls or reproduces the Scala RNG draw sequence.

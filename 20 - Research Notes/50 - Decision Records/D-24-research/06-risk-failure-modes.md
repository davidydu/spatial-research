# Risk and failure modes

## Silent corruption and reproducibility

D-24 is not a cosmetic host API choice: the queue asks whether Rust host fixed-point conversion should match Cppgen's double approximation or enforce bit-exact shifted integers (`Spatial Research/20 - Research Notes/40 - Decision Queue.md:103-105`). The silent-corruption risk comes from Cppgen using two representations at once. Fractional `FixPtType` becomes `double` in ordinary host expressions (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenCommon.scala:75-89`), but `toTrueFix` multiplies by `1 << f` and casts to an integer transport type (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenCommon.scala:36-49`). Scalar `setArg` repeats that shift into `int64_t`, while `getArg` sign-extends and divides by `1 << f` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenInterface.scala:42-80`). A Cppgen-compatible default therefore preserves old C++ behavior but can lose raw bits whenever a fractional value needs more precision than `double`, then look numerically plausible on readback. A bit-exact shifted-integer default is more reproducible because the scaled integer is the value; the main cost is ergonomic, not semantic.

## Test breakage and compatibility

Existing tests mostly validate numeric values, not raw encodings. `FixPtArgInOut` parses `-1.5`, sends it through `ArgIn`, computes a gold value on the host, and compares numeric equality (`/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/unit/ArgInOut.scala:64-90`). `SmallTypeTransfers` initializes `FixPt[FALSE,_8,_8]` and `FixPt[FALSE,_4,_4]` arrays with `0.5` multiples, transfers them, and compares host arrays (`/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/transfers/SmallTypeTransfers.scala:20-24`, `/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/transfers/SmallTypeTransfers.scala:47-69`). Those tests would catch gross read/write failures but not every double-rounding, overflow, or packing mismatch. Switching the default to raw integer values could break CLI-style expectations unless the Rust host still accepts decimal text and exposes an ergonomic typed wrapper. Cppgen's ergonomics come from `InputArguments` as strings plus `std::stod` for fractional fixed text conversion (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenArray.scala:93-104`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenDebug.scala:26`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenCommon.scala:102-119`). Backwards compatibility favors a Cppgen-compatible adapter, not necessarily a Cppgen-compatible semantic default.

## Rounding and saturation drift

Scalagen's reference type is `FixedPoint`, not `double` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenFixPt.scala:12-14`). Normal casts call `toFixedPoint`, which shifts or truncates scaled integers (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:90-95`); wrap and saturation are distinct constructors (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:203-221`); unbiased conversion consumes four guard bits and an RNG before clamping or saturating (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:232-240`). Cppgen collapses ordinary and unbiased fixed casts to C++ casts, with saturation implemented as host comparisons against type min/max (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenMath.scala:120-135`). Chisel instead lowers fixed casts through `Math.fix2fix` with explicit `Truncate`/`Unbiased` and `Wrapping`/`Saturating` modes (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenMath.scala:75-78`), and Fringe implements fractional shrink plus saturation in hardware logic (`/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/math/Converter.scala:37-43`, `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/math/Converter.scala:59-78`). Target-native/vendor-native conversion is therefore risky unless labelled as non-parity.

## Packing, bitcasts, and structs

Sub-byte payloads are a separate failure plane. Cppgen packs integer widths below 8 into `uint8_t` lanes with element `j` at `j * width`, unpacks the same way, and throws for fractional fixed values below 8 bits (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenInterface.scala:95-103`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenInterface.scala:120-128`). The corpus exercises `Bit`, `UInt2`, and `UInt4` packed transfers (`/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/transfers/TransferSubByteTypes.scala:13-29`, `/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/transfers/TransferSubByteTypes.scala:60-63`), but not fractional sub-byte recovery. Bit views also depend on low-bit-first reconstruction: Scalagen maps fixed bits through `bits`/`fromBits` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenBits.scala:47-56`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenBits.scala:70-72`), while Cppgen shifts fixed bits out of `toTrueFix` and rebuilds with `<< i` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenArray.scala:111-151`). Packed structs amplify the risk: Cppgen stores fields as raw integer types, setters call `toTrueFix`, `toString` calls `toApproxFix`, and `toRaw` concatenates fields by bit offset (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenArray.scala:153-192`). Chisel packs structs as reversed `ConvAndCat` fields and slices fields back by computed offsets (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenStruct.scala:17-20`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenStruct.scala:39-42`). Tests move fractional fixed structs through DRAM but even mark one checksum as expected to fail (`/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/transfers/TransferStruct.scala:29-42`, `/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/transfers/TransferStruct.scala:81-92`).

## HLS co-sim containment

HLS co-sim needs one manifest-labelled conversion policy per scalar, memory element, stream payload, struct field, and bitcast, not a global assumption. Chisel sign-extends signed fixed HostIO/ArgOut writes to the 64-bit register path (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:73-99`) and emits a separate ArgAPI ordering artifact (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:156-187`). A target-native HLS wrapper that uses vendor fixed-point assignment may pass vendor co-sim while diverging from Scalagen or legacy Cppgen. Lowest-risk policy is dual-mode with manifest labels: `bit_exact_scaled_integer_v1` for canonical reproducible parity, `cppgen_fractional_double_shift_v1` only for legacy adapters, and explicit `target_native_*` labels that cannot be compared as bit-exact goldens.

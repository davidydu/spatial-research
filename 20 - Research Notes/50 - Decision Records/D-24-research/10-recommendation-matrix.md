# D-24 Angle 10: Recommendation Matrix

## Evidence Baseline

Cppgen compatibility is not the same thing as fixed-point correctness. Fractional `FixPtType` remaps to host `double`, while `toTrueFix` multiplies by `1 << f` and casts to a raw integer, and `toApproxFix` divides raw storage back to an approximate host value (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenCommon.scala:36-49`; `/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenCommon.scala:75-90`). Register writes repeat that policy by shifting before `setArg`; reads sign-extend the raw value and divide by `1 << f` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenInterface.scala:42-80`). The reference side is raw scaled state: `FixedPoint(value: BigInt, valid, fmt)`, with `FixFormat.bits = ibits + fbits`, integer constructors shifting by `fbits`, decimal constructors multiplying by `2^fbits`, and casts shifting raw values between fractional widths (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:3`; `/Users/david/Documents/David_code/spatial/emul/src/emul/FixFormat.scala:3-11`; `/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:150-159`; `/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:90-94`). D-23 already has the right carrier: `numeric_formats` include fixed widths, packing, and `conversion_policy`, with D-24 choosing the default (`20 - Research Notes/50 - Decision Records/D-23.md:52-65`).

## Test Implications

Existing tests mostly validate logical values, not raw encodings: `FixPtArgInOut` parses `-1.5`, computes host gold, and compares equality, while `SmallTypeTransfers` uses exactly representable half-step arrays (`/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/unit/ArgInOut.scala:64-90`; `/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/transfers/SmallTypeTransfers.scala:20-69`). D-24 should add raw golden fixtures for scalar set/get, DRAM set/get, binary file payloads, struct `toRaw`, `DataAsBits`, signed negative readback, fractional subbyte rejection, exact decimal parsing, and Cppgen loss cases. Stochastic unbiased arithmetic remains out of scope except for recording provenance; current Scala uses `Random.nextFloat`, while Fringe uses a PRNG for `Unbiased` (`/Users/david/Documents/David_code/spatial/emul/src/emul/FixedPoint.scala:232-241`; `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/math/RoundingMode.scala:3-8`).

## Decisive Recommendation and Rejections

Choose dual-mode with `bit_exact_scaled_integer_v1` as the Rust default and `cppgen_fractional_double_shift_v1` as an explicit legacy compatibility mode. This gives D-24 a reproducible source contract, gives D-23 a concrete policy label, and keeps old Cppgen behavior available without letting it define correctness. Reject Cppgen-compatible default because it preserves approximation as policy. Reject bit-exact-only because migration would be needlessly sharp. Reject target-native/vendor-native because it is a backend projection, not a portable Spatial host contract.

# Cppgen conversion source survey

## 1. Host type map and helpers

[[10 - Cppgen]] frames Cppgen as a distributed host-driver backend, and `CppGenCommon` is the conversion hinge. `remap` maps `FixPtType(s,d,f)` with `f == 0` to sized signed or unsigned integers, but maps any fractional fixed point to `double` (`src/spatial/codegen/cppgen/CppGenCommon.scala:75-90`). `asIntType` separately chooses raw signed storage from `d+f`, down to `bool` for one-bit payloads (`CppGenCommon.scala:51-72`). The helper split is explicit: `toTrueFix` multiplies a host-visible value by `1 << f` and casts it to the raw integer type, while `toApproxFix` divides raw integer storage by `1 << f` and casts back to the logical host type (`CppGenCommon.scala:36-49`). Text parsing leans approximate: `conv` returns `stod` for fractional fixed and `sto*` for integer fixed, rejecting string parse for 128-bit integers (`CppGenCommon.scala:102-119`). Fixed constants print as raw literals, with only an `L` suffix for wide values (`CppGenCommon.scala:125-130`).

## 2. Scalar register boundary

`SetReg` shows the mixed representation clearly. A fractional fixed host variable remains in the `double` representation, but the register write is `(int64_t)(v * ((int64_t)1 << f))` (`src/spatial/codegen/cppgen/CppGenInterface.scala:42-51`). Integer fixed and other scalars are passed directly; custom floats are copied into an `int64_t`, masked to `g+e` bits, and written (`CppGenInterface.scala:52-60`). `GetReg` reverses fixed transport with an integer temp, signedness test, sign extension, and `(${lhs.tp}) tmp / ((int64_t)1 << f)` (`CppGenInterface.scala:64-80`). This is Cppgen-compatible scaled-register conversion, not a bit-exact fixed host type: after readback, a fractional fixed value is again a `double` because `remap` chose `double`.

## 3. Memories and files

`SetMem` and `GetMem` use `asIntType`, `bitWidth`, and `fracBits` to choose transport. Fractional fixed vectors of width at least 8 allocate a raw vector, multiply each host element by `1 << f`, and bulk-copy it; reads copy raw values back and divide each element to a host `double` (`CppGenInterface.scala:85-119`). Integer sub-byte vectors pack and unpack `uint8_t` lanes using element `j * width`; fractional sub-byte fixed is explicitly unsupported (`CppGenInterface.scala:95-103`, `CppGenInterface.scala:120-128`). Otherwise the path is raw `memcpy` (`CppGenInterface.scala:104-108`, `CppGenInterface.scala:129-131`). Binary files mirror memory: `ReadBinaryFile` reads bytes into `vector<rawtp>`, then fixed payloads loop through `toApproxFix`; `WriteBinaryFile` emits `raw = value * (1 << f)` before writing `rawtp` (`src/spatial/codegen/cppgen/CppGenFileIO.scala:40-68`, `CppGenFileIO.scala:73-86`). CSV is different: `ReadTokens` only produces strings and `WriteTokens` streams `token.result` plus a delimiter, so numeric CSV conversion is outside FileIO itself (`CppGenFileIO.scala:98-132`).

## 4. Structs, arrays, and bit views

Arrays mostly keep the host expression type. `emitUpdate` only special-cases integer fixed, casting through a signed intermediate; fractional fixed falls through to direct assignment (`src/spatial/codegen/cppgen/CppGenArray.scala:43-67`). Structs are more raw: `SimpleStruct` declares fields as `asIntType`, prints them through `toApproxFix`, and emits setters that store `toTrueFix(x)` (`CppGenArray.scala:153-170`). `toRaw` packs fields by left shifting stored raw fields at increasing offsets, and the raw-bit constructor calls setters on shifted slices (`CppGenArray.scala:172-188`). `FieldApply`, however, returns `$struct.$field` directly into `${lhs.tp}` without `toApproxFix` (`CppGenArray.scala:200-206`), so struct reads are a conversion-policy hazard. Bit views are LSB-first: `DataAsBits` and `DataAsVec` use `toTrueFix` then right shift from index zero; `BitsAsData` and `VecAsData` reconstruct by adding chunks shifted left (`CppGenArray.scala:111-151`). Float bit extraction reinterprets a native `float`; inverse float bitcasts throw (`CppGenArray.scala:113-126`, `CppGenArray.scala:135-147`).

## 5. Math, casts, and D-24 implication

Math reinforces the double-host model. Arithmetic on fractional fixed emits ordinary C++ expressions such as `x + y`, `x * y`, and `x / y`; casts are plain C casts, with saturating casts comparing against type min/max and unbiased casts also plain casts (`src/spatial/codegen/cppgen/CppGenMath.scala:26-34`, `CppGenMath.scala:120-138`). Raw-integer helpers appear only where an operator needs bits: `FixInv` applies `~` to `toTrueFix` and returns through `toApproxFix`, while `FixMod` computes positive modulo on raw scaled integers then approximates back (`CppGenMath.scala:20-21`, `CppGenMath.scala:55-56`). Shifts split by fractionalness: `FixSLA` and `FixSRA` on fractional fixed use multiply or divide by `pow(2., y)`, integer fixed uses `<<` or `>>`, `FixDivSRA` always divides by `pow`, and `FixSRU` emits a TODO `>>>` expression that is not valid C++ (`CppGenMath.scala:144-156`). D-24 therefore has two real choices: preserve `cppgen_fractional_double_shift_v1` as legacy behavior, including struct, CSV, and shift quirks, or define `bit_exact_scaled_integer_v1` as a new host conversion contract and keep Cppgen compatibility as a D-23 manifest projection (`D-23.md:61-65`).

# Chisel, RTL, and HLS Behavior for DelayLine

## Hardware Path

For [[D-20]], `DelayLine` is a real hardware timing artifact, not just an IR wrapper. The node is a `Primitive` whose only local rewrite is `size == 0` pass-through (`/Users/david/Documents/David_code/spatial/src/spatial/node/DelayLine.scala:8`, `/Users/david/Documents/David_code/spatial/src/spatial/node/DelayLine.scala:9`). User code can request the same node through `retime(delay, payload)`: negative delays throw, zero delays return the payload, and positive delays stage `DelayLine` and mark `userInjectedDelay` (`/Users/david/Documents/David_code/spatial/src/spatial/lang/api/MiscAPI.scala:23`, `/Users/david/Documents/David_code/spatial/src/spatial/lang/api/MiscAPI.scala:29`, `/Users/david/Documents/David_code/spatial/src/spatial/lang/api/MiscAPI.scala:30`). The compiler also inserts lines automatically: `RetimingTransformer.delayLine` stages `DelayLine(size, data.view[Bits])` and names it from the producer (`/Users/david/Documents/David_code/spatial/src/spatial/transform/RetimingTransformer.scala:68`, `/Users/david/Documents/David_code/spatial/src/spatial/transform/RetimingTransformer.scala:71`), while `computeDelayLines` computes producer-consumer latency mismatch and returns `ValueDelay`s when the mismatch is nonzero (`/Users/david/Documents/David_code/spatial/src/spatial/util/modeling.scala:570`, `/Users/david/Documents/David_code/spatial/src/spatial/util/modeling.scala:632`, `/Users/david/Documents/David_code/spatial/src/spatial/util/modeling.scala:634`). This matches [[C0 - Retiming]] and [[70 - Timing Model]]: `DelayLine` exists to align cycles inside pipelined hardware.

## Chisel Lowering

`ChiselGenDelay` preserves that timing. It ignores unbound broadcast addresses, updates `maxretime`, emits a typed Chisel `Wire`, and assigns each scalar or vector lane from `DL(data.r, delay)` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenDelay.scala:17`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenDelay.scala:19`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenDelay.scala:26`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenDelay.scala:28`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenDelay.scala:29`). `ChiselCodegen` imports `fringe.templates.retiming._`, so the generated Scala sees the retiming wrappers (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselCodegen.scala:79`). The helper `DL` is the key semantic bridge: for bits it emits `DS(latency, rr, flow)`, otherwise `getRetimed(name, latency, flow)`, where `flow` is `backpressure & forwardpressure` inside a controller and `true.B` outside one (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:288`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:291`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:292`). For controllers that do not halt if starved, forwardpressure is not part of this flow gate; for halt-if-starved controllers, it is.

The fringe template then becomes RTL state. `RetimeWrapper` instantiates `RetimeShiftRegister`, wires clock/reset/flow/init/in/out, and the black box exposes `WIDTH` and `STAGES` parameters (`/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/retiming/RetimeShiftRegister.scala:8`, `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/retiming/RetimeShiftRegister.scala:15`, `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/retiming/RetimeShiftRegister.scala:18`, `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/retiming/RetimeShiftRegister.scala:45`, `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/retiming/RetimeShiftRegister.scala:51`). The copied SystemVerilog is a register array: reset fills every stage with `init`; when `flow` is high, stage 0 takes `in` and later stages shift; output is `sr[STAGES-1]` (`/Users/david/Documents/David_code/spatial/resources/synth/aws.hw-resources/RetimeShiftRegister.sv:14`, `/Users/david/Documents/David_code/spatial/resources/synth/aws.hw-resources/RetimeShiftRegister.sv:22`, `/Users/david/Documents/David_code/spatial/resources/synth/aws.hw-resources/RetimeShiftRegister.sv:25`, `/Users/david/Documents/David_code/spatial/resources/synth/aws.hw-resources/RetimeShiftRegister.sv:28`, `/Users/david/Documents/David_code/spatial/resources/synth/aws.hw-resources/RetimeShiftRegister.sv:31`, `/Users/david/Documents/David_code/spatial/resources/synth/aws.hw-resources/RetimeShiftRegister.sv:38`). Therefore RTL observes cycles, and it holds the line under downstream stalls because the shift register advances only on the Chisel flow predicate.

The flow predicate is not isolated to delay nodes. Chiselgen derives stream forward pressure from readable streams being valid and back pressure from writable streams being ready or not full (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:249`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:251`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:266`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:270`). Controllers latch those predicates into each state-machine object (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenController.scala:308`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenController.scala:309`), memory writes include `$backpressure` in their flow enable (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenMem.scala:45`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenMem.scala:50`), and stream-out valid is masked by backpressure (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenStream.scala:39`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenStream.scala:42`). [Inference] The RTL contract is cycle-aware and stall-aware at the same time: `DelayLine` state, memory side effects, and stream handshakes advance together.

## Backend Contrast

The non-Chisel codegens do not preserve `DelayLine` time. Scalagen emits no code for constant delay lines and aliases non-constants as `val lhs = data` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenDelays.scala:10`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenDelays.scala:12`). Cppgen and Roguegen do the same value-only collapse (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenDebug.scala:55`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/roguegen/RogueGenDebug.scala:33`). PIR's delay generator has its `DelayLine` cases commented out (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/pirgen/PIRGenDelays.scala:9`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/pirgen/PIRGenDelays.scala:12`). The build pipeline confirms the current synthesis path is Chisel plus host Cpp/Rogue, not a separate HLS kernel backend (`/Users/david/Documents/David_code/spatial/src/spatial/Spatial.scala:245`, `/Users/david/Documents/David_code/spatial/src/spatial/Spatial.scala:248`; `/Users/david/Documents/David_code/spatial/src/spatial/targets/HardwareTarget.scala:10`). So the HLS equivalent is an architectural choice, as flagged in [[30 - Chisel-Specific]] and [[40 - Open HLS Questions]]: either elide `DelayLine` to match value-only simulation, or emit explicit cycle state such as static shift registers gated by accepted II/valid-ready flow. Silent aliasing would match Scalagen but not RTL backpressure behavior.

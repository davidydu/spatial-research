# D-20 Angle 3: Tests, Apps, and Usage

## Search Scope

I searched source tests, app tests, `apps/src`, and the vault specs for `DelayLine`, `retime`, `retimeGate`, retiming, forced latency, line buffers, streaming controllers, explicit `II`, `haltIfStarved`, and `--nostreamify`. The decision frame is [[D-20]] / Q-148: Rust simulation must either match value-only Scalagen elision or model cycle-aware retiming registers (`20 - Research Notes/40 - Decision Queue.md:87-89`; `20 - Research Notes/20 - Open Questions.md:2103-2113`). The specs already define the fork: [[70 - Timing Model]] says Scalagen ignores delays while RTL timing uses metadata and inserted lines (`10 - Spec/20 - Semantics/70 - Timing Model.md:61-65`), and [[60 - Counters and Primitives]] says `DelayLine(3, x)` is the same value as `x` in ScalaGen (`10 - Spec/50 - Code Generation/20 - Scalagen/60 - Counters and Primitives.md:78-96`).

## Direct Delay and Retiming Tests

The strongest direct test is `SimpleRetimePipe`: it imports `DelayLine`, stages `d := a * b + c`, and asserts exactly three `DelayLine` nodes in `checkIR` (`test/spatial/tests/feature/retiming/SimpleRetimePipe.scala:5`, `test/spatial/tests/feature/retiming/SimpleRetimePipe.scala:14-23`). This is an IR-shape sentinel, not a cycle-behavior sentinel; the runtime value assertion is not about phasing. `RetimeNestedPipe` and `SimplePipeRetime2` are useful final-value checks for nested arithmetic and mixed latency arithmetic, but neither checks `DelayLine` state or cycle traces (`test/spatial/tests/feature/retiming/RetimeNestedPipe.scala:15-27`; `test/spatial/tests/feature/retiming/SimplePipeRetime2.scala:19-26`).

User-directed timing appears in three forms. `ForcedLatency` verifies retiming metadata by requiring a `RegWrite` data symbol to have `hasForcedLatency` and `fullDelay == 0.0` (`test/spatial/tests/compiler/ForcedLatency.scala:19-33`). `retimeGate()` is used to separate stream/FIFO events in `ContinualStreaming`, `PriorityDeq`, `PriorityDeq2In1`, `DeqDependencyCtrl`, and `BufferedRF` (`test/spatial/tests/compiler/ContinualStreaming.scala:23-41`; `test/spatial/tests/feature/dynamic/PriorityDeq.scala:226-267`; `test/spatial/tests/feature/dynamic/PriorityDeq.scala:285-312`; `test/spatial/tests/feature/memories/regfile/ShiftRegs.scala:138-153`). The only positive explicit delay in the searched tests is `retime(10, data)` in `PriorityDeqDependencyCtrl`; it is meant to expose a payload/active timing mismatch, but the assertion only checks the final dequeued value (`test/spatial/tests/feature/dynamic/PriorityDeq.scala:330-358`). That makes it a good fixture seed, not a finished D-20 discriminator.

`IterationDiffs` is adjacent rather than direct. It imports `DelayLine` but primarily exercises loop-carried dependence distances, body-latency comments, and final SRAM matrices (`test/spatial/tests/compiler/InitiationIntervals.scala:6`, `test/spatial/tests/compiler/InitiationIntervals.scala:121-145`, `test/spatial/tests/compiler/InitiationIntervals.scala:147-237`, `test/spatial/tests/compiler/InitiationIntervals.scala:242-287`). It should stay in the D-21/D-06 orbit unless a new fixture records retiming-created delays alongside expected II.

## Apps and Streaming Pressure

Line-buffer apps are the main realistic cycle-pressure surface. `Stencil2D` and `Stencil3D` are marked `ReviveMe (LineBuffer)` but still encode exact sliding-window state with `LineBuffer`, shift-register windows, and matrix/tensor gold checks (`test/spatial/tests/apps/MachSuite.scala:561`, `test/spatial/tests/apps/MachSuite.scala:607-618`, `test/spatial/tests/apps/MachSuite.scala:642-643`, `test/spatial/tests/apps/MachSuite.scala:717-733`, `test/spatial/tests/apps/MachSuite.scala:757-759`). `Convolution_FPGA` and `Convolution_FPGA2` also use line buffers and shift registers, but they collapse correctness to checksum sums, which can hide per-window phasing bugs (`test/spatial/tests/feature/dense/Convolution_FPGA.scala:29-67`, `test/spatial/tests/feature/dense/Convolution_FPGA.scala:123-128`, `test/spatial/tests/feature/dense/Convolution_FPGA.scala:155-177`, `test/spatial/tests/feature/dense/Convolution_FPGA.scala:233-238`). `StridedRowConvolution` and CNN inference cover strided line buffers and exact elementwise output comparisons (`test/spatial/tests/feature/dense/StridedRowConvolution.scala:19-24`, `test/spatial/tests/feature/dense/StridedRowConvolution.scala:57-60`; `test/spatial/tests/apps/Inference.scala:632-651`, `test/spatial/tests/apps/Inference.scala:737-740`).

Streaming pipelines matter because `DelayLine` only becomes honest under a tick/flow policy. `DotProductStream` explicitly comments that paired DRAM loads must be serviced equally to avoid stalling the `Stream` controller (`test/spatial/tests/feature/dense/DotProduct.scala:109-128`). `MatMult_systolic` combines `Stream.Foreach`, FIFO drains, shift-register waves, `computeDone`, and an `II(1)` comment that is only safe under a particular wavefront schedule; its check is checksum-only (`test/spatial/tests/feature/dense/MatMult_systolic.scala:34-60`, `test/spatial/tests/feature/dense/MatMult_systolic.scala:63-68`, `test/spatial/tests/feature/dense/MatMult_systolic.scala:97-108`). The compiler streaming tests add producer/consumer buffering pairs and `--nostreamify` variants, but still mostly assert final values (`test/spatial/tests/compiler/streaming/SRAMBuffering.scala:21-49`; `test/spatial/tests/compiler/streaming/RegBuffering.scala:18-36`; `test/spatial/tests/compiler/streaming/HierarchicalSlowdown.scala:20-34`).

## Gaps and Needed Fixtures

Current tests do not distinguish value-only `DelayLine` elision from cycle-aware shift-register behavior. Standard regression lists are especially thin: mini runs only two math tests, and nightly adds `feature.memories.*`, not `feature.retiming`, compiler streaming, dynamic priority-dequeue, or app stencils (`regressions/mini.list:1-2`; `regressions/nightly.list:1-2`).

D-20 needs five fixtures. First, a deterministic explicit-`retime` trace test for delays 0, 1, 3, and 10 that records every logical cycle under both value-only and cycle-aware expected outputs. Second, an automatic-retiming fixture derived from `SimpleRetimePipe` that checks both `DelayLine` count and Chisel/Rust cycle alignment. Third, a minimized `retimeGate` FIFO fixture from `PriorityDeq2In1` or `DeqDependencyCtrl`, with no randomness and expected event order. Fourth, line-buffer fixtures that assert every window at row/stride/swap boundaries, using the emul circular state as reference (`emul/src/emul/LineBuffer.scala:14-25`, `emul/src/emul/LineBuffer.scala:43-60`). Fifth, a streaming starvation fixture based on `ContinualStreaming` that records stalls, enqueue/dequeue cycles, and final values, explicitly linked to [[D-15]] and [[D-21]] so cycle-aware `DelayLine` does not become an accidental HLS scheduler.

# Risk and Failure Modes for DelayLine Semantics

## Decision Risk Frame

[[D-20]] is a simulator-truth decision, not just a codegen detail. `DelayLine(size, data)` is a Spatial primitive whose size-zero case rewrites back to the payload, while `retime(delay, payload)` can stage a user-injected line and mark `userInjectedDelay` (`/Users/david/Documents/David_code/spatial/src/spatial/node/DelayLine.scala:8-14`; `/Users/david/Documents/David_code/spatial/src/spatial/lang/api/MiscAPI.scala:23-31`). The compiler also creates automatic delay lines from retiming metadata: `RetimingAnalyzer` writes `fullDelay` and cycle membership, then `RetimingTransformer` computes missing delay, creates or extends `ValueDelay`, and substitutes delayed inputs into consumers (`/Users/david/Documents/David_code/spatial/src/spatial/traversal/RetimingAnalyzer.scala:26-55`; `/Users/david/Documents/David_code/spatial/src/spatial/util/modeling.scala:570-649`; `/Users/david/Documents/David_code/spatial/src/spatial/transform/RetimingTransformer.scala:132-154`). So the choice is whether Rust simulation treats those nodes as invisible value aliases or observable pipeline state. This follows the classification in [[02-retiming-scheduling-role]] and the dual-mode frame in [[05-simulator-semantics-options]].

## Value-Only Risks

Value-only is the safest compatibility lane. Scalagen currently elides constants and emits non-constant delay lines as `val lhs = data`, so a Rust value simulator that does the same preserves the existing golden oracle (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/scalagen/ScalaGenDelays.scala:9-14`; [[open-questions-scalagen#Q-scal-06 - Delay line semantics under --sim]]). Other non-RTL backends already follow the same broad shape: C++ and Rogue debug codegen emit direct assignments for `DelayLine` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenDebug.scala:55`; `/Users/david/Documents/David_code/spatial/src/spatial/codegen/roguegen/RogueGenDebug.scala:33`). This mode also avoids fabricating cycle precision where HLS may not expose one; [[C0 - Retiming]] says Chisel delay-line insertion has no direct Vitis HLS analogue and HLS instead relies on pipeline pragmas and scheduler-inserted registers (`10 - Spec/40 - Compiler Passes/C0 - Retiming.md:307-311`).

The cost is blind spots. A value-only simulator can pass final-value tests while hiding retiming bugs: wrong line size, wrong producer sharing, missing switch-case trailing delay, or bad `RetimeGate` boundary behavior. These are real implementation surfaces: delay sharing is memoized in `ValueDelay.value()`, `precomputeDelayLines` materializes lines for sibling blocks, and `retimeSwitchCase` may add a trailing line to align case outputs (`/Users/david/Documents/David_code/spatial/src/spatial/metadata/retiming/ValueDelay.scala:13-18`; `/Users/david/Documents/David_code/spatial/src/spatial/transform/RetimingTransformer.scala:111-130`; `/Users/david/Documents/David_code/spatial/src/spatial/transform/RetimingTransformer.scala:156-178`). Value-only also weakens user-injected `retime`: the model records a real `d` only for user-injected lines in `NodeParams`, but final-value simulation still cannot show the cycle when that state should appear (`/Users/david/Documents/David_code/spatial/src/spatial/targets/NodeParams.scala:41`).

## Cycle-Aware Risks

Cycle-aware simulation is attractive because it can expose bad data/valid alignment, stale register state, and liveness failures that value-only erases. Chisel does build real retimed state: `ChiselGenDelay` emits a wire whose `.r` comes from `DL(..., delay)`, and `DL` lowers to `getRetimed` gated by controller backpressure and sometimes forward pressure (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenDelay.scala:18-30`; `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:288-293`). Fringe's retime wrapper is explicitly stateful, with `flow`, `in`, and `out` ports around a `RetimeShiftRegister` (`/Users/david/Documents/David_code/spatial/fringe/src/fringe/utils/package.scala:88-97`; `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/retiming/RetimeShiftRegister.scala:8-21`).

The danger is false precision. Modeling values alone is not cycle-aware enough: controller progress depends on `backpressure`, masks, done bits, and synchronization, and streaming/fork logic gates state advancement on those signals (`/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/Controllers.scala:95-118`; `/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/Controllers.scala:141-170`). A partial cycle model can invent deadlocks, miss real deadlocks, or report off-by-one data because the simulator delayed payloads but not enables, masks, empty/full status, or ready/valid handshakes. It also risks breaking Scalagen parity immediately: tests such as `SimpleRetimePipe` structurally expect three `DelayLine` nodes but still assert only final value, while larger retiming tests compare arrays and one retiming offset test is disabled (`/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/retiming/SimpleRetimePipe.scala:14-24`; `/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/retiming/RetimeLoop.scala:22-24`; `/Users/david/Documents/David_code/spatial/test/spatial/tests/feature/retiming/RetimeOffset.scala:5-7`).

## Guardrails

The lowest-risk split is provenance, not pretending one mode answers every question. Keep `compat_value_elided` as the default regression oracle and label its evidence as value-only. Add `cycle_shift_register` or `retime_cycle_debug` only when it records delay-line identities, sizes, producer traces, flow/backpressure gates, queue or stream occupancy, first divergent cycle, and timeout wait reasons. HLS results should be reconciled through [[D-21]]: [[70 - Timing Model]] already warns that Rust should store compiler-requested and backend-accepted II separately, and a simulator claiming cycle accuracy must state whether it matches Scalagen delay elision or RTL delay lines (`10 - Spec/20 - Semantics/70 - Timing Model.md:57-65`). Success in value mode should not bless HLS liveness; success in cycle mode should not rewrite legacy Scalagen goldens.

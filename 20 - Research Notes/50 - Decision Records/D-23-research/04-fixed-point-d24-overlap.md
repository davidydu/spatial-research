# Fixed-point and D-24 overlap

## Scope

D-23 should describe the host ABI surfaces, not resolve D-24. The queue separates the host ABI manifest for scalar args, DRAM pointers, ArgIO/ArgOuts, counters, exits, and streams from the later decision about Cppgen-compatible versus bit-exact fixed-point host conversion (`20 - Research Notes/40 - Decision Queue.md:99-105`). Cppgen itself proves why: its type map makes integer `FixPtType` a sized C integer, but maps fractional fixed-point to `double`; it maps `FloatType`/`FltPtType` to native `float`, `DoubleType` to `double`, `Bit` to `bool`, `Vec` and `host.Array` to `vector<...>`, and `Reg` to the element type (`src/spatial/codegen/cppgen/CppGenCommon.scala:75-99`). Therefore the manifest needs separate logical type, host representation, and transport representation fields.

## Scalar registers

Scalar ABI slots are collected as `argIns`, `argIOs`, `argOuts`, and `drams` when `ArgInNew`, `HostIONew`, `ArgOutNew`, and `DRAMHostNew` are seen (`src/spatial/codegen/cppgen/CppGenInterface.scala:20-40`), then emitted in ArgIns, DRAM pointers, ArgIOs, ArgOuts, instrumentation, and early-exit order (`src/spatial/codegen/cppgen/CppGenInterface.scala:138-164`). For scalar writes, fractional fixed values are multiplied by `1 << f` and cast to `int64_t`; integer fixed values are passed directly; custom floats are `memcpy`-copied into `int64_t` and masked to `g+e` bits; everything else is sent as-is (`src/spatial/codegen/cppgen/CppGenInterface.scala:42-61`). For reads, fixed results are fetched with `getArg` or `getArg64`, sign-extended when signed, then divided by `1 << f`; custom floats are copied back from an integer temporary; other types are cast from the register value (`src/spatial/codegen/cppgen/CppGenInterface.scala:64-80`). Chisel also treats ArgIn as a 64-bit `UInt` port and sign-extends signed fixed HostIO/ArgOut writes when the logical width is under 64 (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:405-409`; `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:72-86`; `src/spatial/codegen/chiselgen/ChiselGenInterface.scala:93-99`). Each scalar entry should record role, index, register width, signedness, logical width, fractional bits, and readback extension policy.

## Memory and vectors

`asIntType` chooses raw host storage from fixed `d+f` or general `bitWidth`, using wider signed integer types down to `bool` (`src/spatial/codegen/cppgen/CppGenCommon.scala:51-72`). DRAM writes rawify fractional fixed vectors of width at least 8 by multiplying each element by `1 << f`; integer subbyte vectors are packed into `uint8_t` lanes with element `j` shifted by `j * width`; fractional subbyte payloads throw; otherwise Cppgen bulk-copies the vector (`src/spatial/codegen/cppgen/CppGenInterface.scala:85-108`). Reads reverse those cases, dividing fixed raw integers by `1 << f`, unpacking subbyte lanes, or bulk-copying (`src/spatial/codegen/cppgen/CppGenInterface.scala:109-131`). Binary file I/O repeats the same raw buffer pattern: read bytes into `vector<rawtp>` and convert fixed with `toApproxFix`; write fixed by multiplying into `rawtp` before `fstream.write` (`src/spatial/codegen/cppgen/CppGenFileIO.scala:40-68`; `src/spatial/codegen/cppgen/CppGenFileIO.scala:73-86`). Manifest entries for DRAM, vectors, and files need element type, raw element type, element storage bits, byte length, byte order, bit order, lanes per byte, and conversion stage.

## Structs and bit views

Structs are another ABI surface. `SimpleStruct` emits packed C++ structs, stores scalar fields as `asIntType`, calls setters that use `toTrueFix`, prints through `toApproxFix`, and builds `toRaw` by shifting fields at increasing bit positions (`src/spatial/codegen/cppgen/CppGenArray.scala:153-192`). Data/bit views use the same low-bit-first convention: `DataAsBits` and `DataAsVec` push bits or chunks by right-shifting from index zero; fixed values pass through `toTrueFix`, while float views reinterpret a native `float` as `int32_t` (`src/spatial/codegen/cppgen/CppGenArray.scala:111-144`). The inverse Cppgen paths support fixed and one-bit bool but throw for floating bit reinterpretation (`src/spatial/codegen/cppgen/CppGenArray.scala:124-151`). The IR/emul rewrite agrees that bit arrays are indexed LSB first for fixed and float reconstruction (`argon/src/argon/node/Bits.scala:35-43`; `emul/src/emul/FixedPoint.scala:171-188`; `emul/src/emul/FloatPoint.scala:435-459`). D-23 should include struct field order, field offsets, packed size, and nested vector/struct element schemas.

## Manifest fields

Recommended fields: `numeric_format` (`fix`, `float`, `bool`, `struct`, `vector`, `text`), `storage_bits`, `register_bits`, `memory_bits`, `signedness`, `integer_bits`, `fractional_bits`, `mantissa_bits`, `exponent_bits`, `host_repr`, `transport_repr`, `endian_packing`, and `rounding_saturation_policy_ref`. The last field should point at a policy label, not inline semantics: ordinary fixed conversion in Scalagen shifts/truncates through `toFixedPoint`, while saturating and unbiased variants call distinct helpers (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:107-114`; `emul/src/emul/FixedPoint.scala:90-95`; `emul/src/emul/FixedPoint.scala:203-241`). Add `conversion_policy` as a required enum with values such as `cppgen_fractional_double_shift_v1`, `bit_exact_scaled_integer_v1`, `scalagen_emul_reference_v1`, `native_float32_masked_v1`, `raw_memcpy_v1`, `packed_subbyte_lsb0_v1`, and `unsupported_fractional_subbyte_v1`. D-24 can choose which fixed-point policy becomes default; D-23 only ensures every scalar, memory, struct, vector, bool, float, and bitcast site declares the policy it uses.

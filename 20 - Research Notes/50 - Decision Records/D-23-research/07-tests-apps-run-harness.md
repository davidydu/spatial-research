# D-23 Angle 7: Tests, Apps, And Run Harness Consumers

## 1. CLI And Harness Shape

The manifest must preserve positional argv behavior, not just typed accelerator ports. Tests define compile-time and runtime argument lists separately, and each backend composes its own compiler flags, make command, run command, and model command (`argon/src/argon/DSLTest.scala:16-45`, `argon/src/argon/DSLTest.scala:95-114`, `src/spatial/SpatialTest.scala:33-63`). At compile time, backend args and per-test compile args are concatenated with `-v --test`; at run time, every `runtimeArgs` string is appended to the backend run command (`argon/src/argon/DSLTest.scala:138-150`, `argon/src/argon/DSLTest.scala:217-223`). Generated run wrappers then forward those values differently: Scala uses `sbt -batch "run $args"`, PIR uses `runMain AccelMain`, and synth/VCS invokes `./Top $@` while setting DRAM simulation environment knobs (`resources/scalagen/run.sh:1-4`, `resources/pirgen/run.sh:1-5`, `resources/synth/run.sh:17-31`). CLI names are inferred from `args(i)` consumers, including sparse indices: `CLITest` names `0,1,2,4` while its runtime string has four tokens (`test/spatial/tests/compiler/CLITest.scala:7-18`, `src/spatial/traversal/CLINaming.scala:24-30`). Therefore include `argv_examples`, `argv_index_names`, missing-index policy, backend wrapper, environment, and max-cycle/run-mode flags.

## 2. Scalars, HostIO, And DRAM Transfers

Tests and apps exercise a mixed scalar ABI: `HelloSpatial` sets ArgIns and HostIO from CLI, writes ArgOut/HostIO in hardware, then reads them back (`apps/src/HelloSpatial.scala:6-17`, `apps/src/HelloSpatial.scala:53-58`, `apps/src/HelloSpatial.scala:73-78`). Unit tests cover scalar int, HostIO loopback, fixed-point `-1.5`, and float args (`test/spatial/tests/feature/unit/ArgInOut.scala:5-30`, `test/spatial/tests/feature/unit/ArgInOut.scala:34-60`, `test/spatial/tests/feature/unit/ArgInOut.scala:64-120`). The source API exposes `setArg`, `getArg`, `setMem`, `getMem`, `setFrame`, and `getFrame` as host transfer nodes (`src/spatial/lang/api/TransferAPI.scala:9-45`). Cppgen then allocates DRAM, registers the pointer as an arg, and copies whole host vectors with type-dependent rawification or packing (`src/spatial/codegen/cppgen/CppGenInterface.scala:36-40`, `src/spatial/codegen/cppgen/CppGenInterface.scala:85-131`). `SmallTypeTransfers` and `FloatTransfer` show why D-24 must remain attached: host transfers span fractional fixed, sub-byte-ish packed fixed, and float payloads (`test/spatial/tests/feature/transfers/SmallTypeTransfers.scala:5-24`, `test/spatial/tests/feature/transfers/SmallTypeTransfers.scala:47-69`, `test/spatial/tests/feature/transfers/FloatTransfer.scala:5-18`, `test/spatial/tests/feature/transfers/FloatTransfer.scala:35-50`).

## 3. Outputs, Counters, And Exits

Expected output is currently a harness convention, not an ABI artifact. Backends parse `PASS: 1` or `PASS: true` as success and `PASS: 0` or `PASS: false` as failure, but `runApp` defaults unknown output to pass after commands finish (`argon/src/argon/DSLTest.scala:120-127`, `argon/src/argon/DSLTest.scala:217-223`). Regression scripts separately grep logs for `Assertion` and `TIMEOUT`, then run instrumentation on success (`resources/synth/scripts/regression_run.sh:11-20`, `resources/synth/scripts/regression_run.sh:76-84`). Many tests compare getArg/getMem data directly rather than emitting a normalized result record, for example `PipeMergerTest` and `DeviceMemcpy` (`test/spatial/tests/compiler/PipeMergerTest.scala:63-74`, `test/spatial/tests/feature/host/DeviceMemcpy.scala:28-35`). Instrumentation and exits are real ABI slots: Cppgen sets arg counts before `run`, reads cycles/iters/stalled/idle after, and reads early-exit bits with source messages (`src/spatial/codegen/cppgen/CppGenAccel.scala:24-36`, `src/spatial/codegen/cppgen/CppGenAccel.scala:38-55`, `src/spatial/codegen/cppgen/CppGenAccel.scala:71-99`). Chisel emits the corresponding ArgAPI offsets and breakpoint counts (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:167-186`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:537-561`). Preserve expected-output mode, assertion policy, counter ids, counter groups, and exit context.

## 4. Streams, Frames, And Host Files

Streams split into interactive Scala simulation, AXI/fringe hardware, and missing file fixtures. Scala stream endpoints prompt for filenames, parse digit-bearing input lines, enqueue values, and dump StreamOut queues on accelerator exit (`emul/src/emul/Stream.scala:5-24`, `emul/src/emul/Stream.scala:27-50`, `src/spatial/codegen/scalagen/ScalaGenStream.scala:60-82`, `src/spatial/codegen/scalagen/ScalaGenStream.scala:84-97`). File buses exist, including EOF-by-last-bit semantics, but I found no live test/app that binds `StreamIn` or `StreamOut` to `FileBus`; live external-stream fixtures use AXI streams and frames (`src/spatial/lang/Bus.scala:59-75`, `test/spatial/tests/feature/transfers/LoadStoreUnitTests.scala:8-27`, `test/spatial/tests/feature/transfers/LoadStoreUnitTests.scala:33-47`, `test/spatial/tests/feature/transfers/LoadStoreUnitTests.scala:51-73`). Chisel records AXI widths, TID/TDEST filters, sideband defaults, and a one-input/one-output limit (`src/spatial/codegen/chiselgen/ChiselGenStream.scala:17-32`, `src/spatial/codegen/chiselgen/ChiselGenStream.scala:81-131`, `src/spatial/codegen/chiselgen/ChiselGenStream.scala:150-167`, `src/spatial/codegen/chiselgen/ChiselGenStream.scala:181-197`). `setFrame/getFrame` fixtures exist, but the visible host emission is Rogue-only (`src/spatial/node/Frame.scala:11-35`, `src/spatial/codegen/roguegen/RogueGenInterface.scala:33-45`, `src/spatial/codegen/roguegen/RogueGenInterface.scala:75-80`).

## 5. Missing ABI Fixtures

Add fixtures for: noninteractive FileBus/FileEOFBus stream replay and dump; multiple AXI streams past the current Chisel limit; full AXI struct sidebands versus payload-only defaults; Cppgen/Scalagen/HLS `setFrame/getFrame`; ArgIO host-set plus accelerator-write ordering across multiple Accel scopes; explicit stdout-only failure versus assertion failure; fractional sub-byte fixed transfer rejection; and manifest readback of generated run command, argv names, ArgAPI offsets, DRAM pointer slots, counter slots, and early-exit slots.

# D-23 Angle 6: ABI Manifest Schema Proposal

## 1. Identity, Target, And Compatibility

Define one canonical `abi_manifest_v1` artifact, serialized as JSON with a TOML/debug rendering allowed only as a derived view. The root fields should be `kernel`, `abi`, `target`, `build_options`, `host_abi`, `memories`, `streams`, `host_files`, `numeric_formats`, `packing`, `lifecycle`, `diagnostics`, and `compatibility`. `kernel` records `name`, `entry_symbol`, `source_module`, `source_hash`, `ir_hash`, and producer/runtime versions. `abi` records `schema_version`, `abi_version`, `min_host_runtime`, and `manifest_hash`. `target` records `backend = hls`, HLS vendor/tool version, part/board/shell, clock, address width, data width, stream word width, memory-channel count, and target capability table. `build_options` carries retiming, modular/perpetual mode, channel assignment, latency-model provenance, and any HLS report authority. This makes explicit the facts now hidden behind `globals.target`, target-derived widths, mutable build flags, and generated `SpatialIP` target selection (`fringe/src/fringe/globals.scala:12-31`; `fringe/src/fringe/SpatialIP.scala:18-39`; [[D-05-research/03-target-fringe-coupling]]).

## 2. Host Register ABI And Control Results

`host_abi` should be an ordered list of entries, not regenerated from traversal order. Each entry records `id`, `name`, `kind = scalar_in|dram_ptr|arg_io|scalar_out`, `direction`, `type_ref`, `width_bits`, `host_word_bits`, `offset`, `init`, `is_host_io`, and `logical_count_group`. Preserve the current order: scalar ArgIns, DRAM pointers, ArgIOs, then ArgOuts (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:156-166`; `src/spatial/codegen/cppgen/CppGenInterface.scala:139-147`). Also separate logical counts from padded hardware counts because Chisel pads top-level arg vectors to at least one (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:197-245`). Add `instrumentation` entries for cycles, iterations, stalled, and idle counters, keyed by controller path and ArgOut offset, plus `control_exits` for `ExitIf`, asserts, and breakpoints with source context and valid-bit semantics (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:167-186`; `src/spatial/codegen/chiselgen/ChiselGenDebug.scala:22-39`; `src/spatial/codegen/chiselgen/ChiselGenController.scala:398-409`).

## 3. Remote Memories, Transfers, And Host Files

`memories` should cover both host DRAM pointers and accelerator-managed remote memories. Fields: `id`, `name`, `kind = host_dram|accel_dram|lock_dram`, `element_type`, `numeric_format`, `rank`, `shape_exprs`, `layout_order = row_major_flat`, `strides`, `byte_stride`, `base_arg`, `allocation = host_bound|kernel_allocated`, `allocator_id`, `lifecycle`, and `transfer_directions`. Host DRAM construction already carries dimension lists, and Cppgen allocates a flat product of those dimensions before setting the pointer arg (`src/spatial/lang/DRAM.scala:40-54`; `src/spatial/node/DRAM.scala:11-39`; `src/spatial/codegen/cppgen/CppGenInterface.scala:36-40`). Transfer records should name `direction = host_to_device|device_to_host|kernel_load|kernel_store`, starts, lengths, strides, parallelism, alignment, bytes per word, packing policy, and request length; dense transfer computes these explicitly before lowering to command/data/ack streams (`src/spatial/node/DenseTransfer.scala:79-129`, `:163-190`, `:324-364`). `host_files` should record file path, mode, encoding, endian, element type, parser/serializer, and attached stream or DRAM because CSV/binary/file-bus behavior is currently split between FileIO nodes and Scala/Cpp emitters (`src/spatial/node/FileIO.scala:7-55`; `src/spatial/codegen/scalagen/ScalaGenFileIO.scala:51-102`; `src/spatial/codegen/cppgen/CppGenFileIO.scala:35-96`).

## 4. Streams, Numeric Formats, And Packing

`streams` should include both memory streams and external streams. Memory stream entries need `class = load|store|gather|scatter`, `cmd_stream`, `data_stream`, optional `ack_stream`, element width, lane count, direction per channel, capacity source, and D-22 fields: `queue_policy`, `backpressure_policy`, `same_cycle_rule`, `ii_source`, overflow/underflow policy, and deadlock policy. ChiselGen already records load/store/gather/scatter `StreamParInfo(element_bit_width, parallelism, 0)` and drives stream pressure from ready/valid or FIFO empty/full (`src/spatial/codegen/chiselgen/ChiselGenCommon.scala:247-278`, `:416-451`; [[D-22]]). External stream entries should record `protocol = axi4_stream|file|pin`, width, direction, TID/TDEST/TUSER/TKEEP/TSTRB/TLAST policy, payload-only versus full-packet mode, and endpoint-count limit; AXI stream width and sideband behavior are visible in `Bus.scala` and `ChiselGenStream` (`src/spatial/lang/Bus.scala:28-87`; `src/spatial/codegen/chiselgen/ChiselGenStream.scala:17-32`, `:181-198`; [[05-stream-file-hostio-d22-overlap]]).

`numeric_formats` should be a registry referenced by args, memories, streams, and files: `bool`, signed/unsigned integers, `fixpt`, `float`, and structs/vectors. For fixed point, record signedness, integer bits, fractional bits, total bits, rounding, overflow, validity representation, and `host_conversion_policy = bit_exact_shifted_int|cppgen_double_approx`. D-24 exists because Cppgen currently converts fractional fixed-point host args and DRAM data through doubles or shifted integers depending on path, while ScalaGen uses `FixFormat`/`FixedPoint` (`src/spatial/codegen/cppgen/CppGenCommon.scala:36-49`, `:75-117`; `src/spatial/codegen/cppgen/CppGenInterface.scala:42-80`, `:85-131`; `src/spatial/codegen/scalagen/ScalaGenFixPt.scala:18-28`; [[40 - Decision Queue#D-24 - Q-153 Choose bit-exact or Cppgen-compatible fixed-point host conversion]]).

## 5. Lifecycle And Diagnostics

`lifecycle` should make the host sequence auditable: bind scalar args, bind DRAM pointers, allocate accelerator DRAMs, copy in, start, poll/wait, read ArgIO/ArgOut/counters/exits, copy out, close files/streams, free. `diagnostics` should attach source location, static op id, dynamic instance id, controller path, queue/transfer id, policy label, HLS report id, and first-failure records for OOB, stream stalls, capacity overflow, bad sideband, stale report, and numeric conversion mismatch. D-16 already argues that memory diagnostics need stable event ids and addresses, and D-22 requires queue policy labels rather than unqualified pass/fail ([[D-16-research/06-rust-mode-design-diagnostics]]; [[D-22-research/08-validation-fixtures-diagnostics]]). `compatibility` should list required features, deprecated fields, legacy layout names, and supported modes such as `scalagen_value_reference`, `cppgen_host_approx`, `bit_exact_host`, and `bounded_hls_cycle`.

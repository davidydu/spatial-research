# Chisel/Fringe Host Interface Survey

## Generated ABI Surfaces

The Chisel path has no single manifest file. It spreads the host ABI across generated Scala, Fringe globals, and the host `ArgAPI` ordinal table. `ChiselGenInterface` first records scalar symbols as `argIns`, `argIOs`, and `argOuts`, assigning each the current collection length when the node is seen (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:18, /Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:23, /Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:32). Its footer emits `Instantiator.scala` counts and comments, stream metadata lists, and host DRAM count (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:121-138). It also emits matching `AccelWrapper` fields (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:141-153) and `ArgAPI.scala`, ordered as ArgIns, DRAM pointers, ArgIOs, ArgOuts, counters, then early exits (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:156-187). `ChiselCodegen` then combines register args, memory pointer args, ArgIOs, instrumentation args, and breakpoints before constructing `CustomAccelInterface` (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselCodegen.scala:196-217, /Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselCodegen.scala:238-245).

## Scalar Registers And Control

Fringe exposes a 64-bit scalar bus with `raddr`, `wen`, `waddr`, `wdata`, and `rdata`, plus accelerator `enable`, `done`, and `reset` (/Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:36-48). The register file is sized from generated counts, with two extra registers for command and status (/Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:113-116). Command is register 0 and status is register 1; command bit 0 enables the accelerator, command bit 1 resets it, and status carries `done`, `timeout`, heap alloc/dealloc, and size/address payloads (/Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:21-32, /Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:132-138). User ArgIns are read from register file outputs after the two control slots (/Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:151). ArgOuts are decoupled valid/bits channels that write back through the register file; status occupies the first ArgOut slot and accelerator outputs follow it (/Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:165-181).

The VCS host context makes the software side explicit. `setArg` writes `arg + 2`, and `getArg` reads `2 + arg`, except for the zero-ArgIn fallback (/Users/david/Documents/David_code/spatial/resources/synth/vcs.sw-resources/FringeContextVCS.h:422-434). `run()` clears status, pulses reset through command value 2, drops command to 0, then starts with command value 1 and polls status until done or timeout (/Users/david/Documents/David_code/spatial/resources/synth/vcs.sw-resources/FringeContextVCS.h:349-399).

## DRAMs And Streams

Host DRAMs are ABI arguments, not separate top-level pointer ports. `ChiselGenDRAM` records each `DRAMHostNew` in `hostDrams`, wires it as a 64-bit fixed-point value, and reads the pointer from `accelUnit.io.argIns(api.<dram>_ptr)` (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenDRAM.scala:17-23). `ArgAPI.scala` places those DRAM pointer slots immediately after scalar ArgIns (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:159-164). Cppgen mirrors this by allocating device memory with `c1->malloc`, then calling `c1->setArg(<dram>_ptr, ...)` ([[10 - Cppgen]]; /Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenInterface.scala:36-40).

Memory traffic itself uses stream descriptors. Fringe IO contains `memStreams`, physical DRAM channels, and heap allocators (/Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:54-58). `AppStreams` groups load, store, gather, and scatter channels, each parameterized by `StreamParInfo(w, v, memChannel)` (/Users/david/Documents/David_code/spatial/fringe/src/fringe/FringeBundles.scala:39-88). `ChiselGenInterface` marks tile load/store, gather, scatter, and aligned/unaligned properties as it sees Fringe nodes (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:102-116), then emits sorted stream-info lists into both generated top surfaces (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:133-152). The arbiter consumes those lists to instantiate load/store/gather/scatter stream controllers (/Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/dramarbiter/DRAMArbiter.scala:13-19, /Users/david/Documents/David_code/spatial/fringe/src/fringe/templates/dramarbiter/DRAMArbiter.scala:58-84).

## Counters, Exits, And Metadata

Instrumentation and exits are appended ArgOut-like slots. `ChiselGenCommon` owns `instrumentCounters` and `earlyExits` beside the arg maps (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:34-44). `ChiselGenInterface` emits per-controller cycle, iteration, stalled, and idle arg IDs, then one breakpoint count and exit arg ID per early exit (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:167-185). The controller generator drives those ArgOut channels and sets `done_latch` either on root completion or on any breakpoint (/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenController.scala:381-407, /Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenController.scala:497-532). Cppgen supplies the contrasting host behavior: before launch it calls `setNumArgIns`, `setNumArgIOs`, `setNumArgOuts`, `setNumArgOutInstrs`, and `setNumEarlyExits`, then reads exit and counter values by those generated IDs after `run()` (/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenAccel.scala:24-36, /Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenAccel.scala:38-100).

## Rust/HLS Manifest Implication

The Rust/HLS host ABI manifest should be the single source for the current distributed facts: ordered scalar slots, pointer slots, ArgIO/ArgOut slots, counter slots, exit slots, command/status semantics, stream descriptors, AXI stream descriptors, allocator count, target widths, channel assignment, and provenance. Fringe globals show the hidden state to replace: arg counts, loopback map, four memory stream lists, AXI stream lists, and allocator count are mutable globals today (/Users/david/Documents/David_code/spatial/fringe/src/fringe/globals.scala:45-60). The manifest should be generated once and consumed by host code, simulator bindings, HLS top signature, and shell integration, rather than requiring Cppgen and Chisel/Fringe to rediscover the same ordinal contract independently.

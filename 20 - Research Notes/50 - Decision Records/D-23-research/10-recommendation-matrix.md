# D-23 Angle 10: Recommendation Matrix

## Evidence Baseline

The current ABI is real but not manifest-shaped. Cppgen accumulates `argIns`, `drams`, `argIOs`, and `argOuts` in mutable buffers, allocates host DRAMs, and emits `ArgAPI.hpp` in ArgIns, DRAM pointers, ArgIOs, ArgOuts, counters, and early-exit order (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenCommon.scala:31-34`; `/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenInterface.scala:20-40`; `/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenInterface.scala:138-164`). Before launch it separately tells the runtime the same counts (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/cppgen/CppGenAccel.scala:24-29`). Chiselgen repeats the ordinal contract, adds wrapper stream lists, and writes generated globals (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:18-39`; `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenInterface.scala:121-152`; `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselCodegen.scala:196-216`). Fringe then hardens the runtime ABI as 64-bit command/status registers plus scalar ArgIn/ArgOut lanes (`/Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:21-25`; `/Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:36-52`; `/Users/david/Documents/David_code/spatial/fringe/src/fringe/Fringe.scala:113-181`).

## Decisive Recommendation

Adopt the explicit versioned manifest-first ABI. D-23 should define `abi_manifest_v1` as the source of truth, then require lossless Cppgen-compatible and Chisel/Fringe-compatible projections. The manifest must freeze ordinal order, logical counts versus padded hardware counts, register width, pointer args, ArgIO/ArgOut direction, instrumentation counter slots, early-exit slots, stream endpoint descriptors, D-22 stream/backpressure labels, and D-24 numeric conversion labels. Target-native HLS signatures should be generated from the manifest, not treated as the source ABI.

## Rejected Alternatives

Reject no manifest because it preserves hidden coupling. Reject Cppgen-compatible as the main ABI because it cannot explain ignored streams or fixed-point ambiguity. Reject Chisel/Fringe-compatible as the sole ABI because it overfits current globals and 64-bit register semantics. Reject target-native HLS ABI because it is a backend projection, not a portable Rust/HLS host contract.

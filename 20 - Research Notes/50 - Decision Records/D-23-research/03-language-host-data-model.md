# Language/API Surface and Host Data Model

## Scalar Register Surface

`setArg` and `getArg` are the visible scalar ABI hooks in [[60 - Host and IO]]. The API lowers literal and staged scalar writes to `SetReg`, and scalar reads to `GetReg` (`src/spatial/lang/api/TransferAPI.scala:9-25`). The language allocators split the logical directions: `ArgIn` stages `ArgInNew`, `ArgOut` stages `ArgOutNew`, and `HostIO` stages `HostIONew` (`src/spatial/lang/Reg.scala:101-110`). The IR keeps these as zero-dimensional register allocations, while `GetReg` is a reader with no address or enables and `SetReg` is a writer with no address or enables (`src/spatial/node/Reg.scala:13-15`, `src/spatial/node/Reg.scala:49-61`).

Cppgen turns that into a flat register map. It records `ArgInNew`, `HostIONew`, and `ArgOutNew` in `argIns`, `argIOs`, and `argOuts` (`src/spatial/codegen/cppgen/CppGenInterface.scala:20-28`). `SetReg` writes through `c1->setArg`; fixed-point fractional values are shifted by `1 << f`, and floating values are bit-copied and masked before the write (`src/spatial/codegen/cppgen/CppGenInterface.scala:42-60`). `GetReg` chooses `getArg64` for wider values, sign-extends fixed-point payloads, divides by `1 << f`, and bit-copies floating payloads back (`src/spatial/codegen/cppgen/CppGenInterface.scala:64-80`). Chiselgen shows the hardware side: `ArgIn` reads from `accelUnit.io.argIns`, while `HostIO` and `ArgOut` are `MultiArgOut` decoupled outputs, with `ArgOut` offset after the ArgIO lanes (`src/spatial/codegen/chiselgen/ChiselGenInterface.scala:18-39`).

## DRAM and Frame Transfers

Host DRAMs carry rank and dimensions at allocation. `DRAM.apply` for ranks 1 through 5 stages `DRAMHostNew` with the dimension sequence (`src/spatial/lang/DRAM.scala:40-54`), and the node records those `dims` as a `MemAlloc` (`src/spatial/node/DRAM.scala:9-15`). `setMem` stages `SetMem`; `getMem` allocates `Tensor1.empty(dram.dims.prodTree)`, stages `GetMem`, and returns that host array (`src/spatial/lang/api/TransferAPI.scala:37-45`). The IR effects are asymmetric: `SetMem` writes the remote DRAM, while `GetMem` writes the host tensor (`src/spatial/node/DRAM.scala:34-39`). Cppgen allocates bytes with `c1->malloc`, registers the pointer with `setArg(<handle>_ptr, ...)`, and bulk-copies with `c1->memcpy` (`src/spatial/codegen/cppgen/CppGenInterface.scala:36-40`, `src/spatial/codegen/cppgen/CppGenInterface.scala:85-131`).

`Frame` is a separate host object, not just DRAM with a different name. Its source comment says it is stream-only, AXI4-Stream facing, and can be read or written once (`src/spatial/lang/Frame.scala:11-13`). `Frame.apply` links the allocation to a `StreamIn` or `StreamOut` (`src/spatial/lang/Frame.scala:43-50`), and the IR node stores that stream symbol (`src/spatial/node/Frame.scala:9-12`). `setFrame` and `getFrame` mirror DRAM at the API level (`src/spatial/lang/api/TransferAPI.scala:27-35`), but Roguegen implements them as `sendFrame` and `getFrame` on connected PyRogue streams (`src/spatial/codegen/roguegen/RogueGenInterface.scala:33-43`, `src/spatial/codegen/roguegen/RogueGenInterface.scala:75-80`).

## Streams and Files

Streams are both local memories and remote endpoints: `StreamIn` permits reads but rejects writes, while `StreamOut` permits writes but rejects reads (`src/spatial/lang/StreamIn.scala:9-24`, `src/spatial/lang/StreamOut.scala:10-25`). Their allocation nodes carry a `Bus` and a unit dimension (`src/spatial/node/StreamIn.scala:7-14`, `src/spatial/node/StreamOut.scala:7-15`). The bus model covers pin buses, AXI streams with 64/256/512-bit widths and `tid`/`tdest`, DRAM burst/gather/scatter buses, and file-backed stream buses (`src/spatial/lang/Bus.scala:13-26`, `src/spatial/lang/Bus.scala:28-50`, `src/spatial/lang/Bus.scala:57-87`). Chiselgen maps AXI stream allocations into `axiStreamsIn` and `axiStreamsOut`, stores width metadata, checks `tid`/`tdest` on reads, and currently errors above one AXI stream in either direction (`src/spatial/codegen/chiselgen/ChiselGenStream.scala:18-31`, `src/spatial/codegen/chiselgen/ChiselGenStream.scala:153-167`, `src/spatial/codegen/chiselgen/ChiselGenStream.scala:181-197`).

Files are host-side resources rather than accelerator memories. `CSVFile` and `BinaryFile` are mutable host reference shells (`src/spatial/lang/host/CSVFile.scala:7-9`, `src/spatial/lang/host/BinaryFile.scala:7-9`). The file API parses constants immediately, stages open/read/write/close operations for CSV and binary files, and includes NumPy placeholders plus ASCII-text DRAM preload (`src/spatial/lang/api/FileIOAPI.scala:11-39`, `src/spatial/lang/api/FileIOAPI.scala:41-67`, `src/spatial/lang/api/FileIOAPI.scala:98-136`).

## Tensor Flattening

The host tensor data model is flat row-major storage plus shape fields. `Matrix` stores `data`, `rows`, and `cols`; indexing is `i*cols + j`; `flatten` returns `data`; and `tabulate` maps a linear index back through division and modulo (`src/spatial/lang/host/Matrix.scala:11-20`, `src/spatial/lang/host/Matrix.scala:28-35`, `src/spatial/lang/host/Matrix.scala:77-85`). `Tensor3`, `Tensor4`, and `Tensor5` repeat this pattern with explicit dimension fields and row-major formulas (`src/spatial/lang/host/Tensor3.scala:11-35`, `src/spatial/lang/host/Tensor4.scala:11-42`, `src/spatial/lang/host/Tensor5.scala:11-45`). Transfer overloads flatten `Tensor2` through `Tensor5` to `.data` on `setMem` and reconstruct shaped views from DRAM dimensions on `getMatrix`/`getTensorN` (`src/spatial/lang/api/TransferAPI.scala:60-86`). The manifest should therefore describe tensors as host-shape metadata plus one flat payload, not nested buffers.

## Manifest Implications

The Rust/HLS manifest should model at least these records: scalar entries with logical kind (`ArgIn`, `ArgOut`, `HostIO`/ArgIO, instrumentation, exit), stable handle, register or output-lane index, direction, bit width, signedness, fractional bits, float packing, reset/default, and whether host writes, accelerator writes, or both are legal. It should separately list DRAM buffers with pointer-arg index, rank, dims, element type, flattened element count, byte width, sub-byte packing, and fixed-point rawification policy. Frames need size, element type, direction, linked stream symbol, stream protocol, and host API method. Streams need endpoint direction, bus kind, width, `tid`/`tdest`, file name when file-backed, and boundedness/back-pressure policy. Files need resource path, mode, parser/format, and whether they are staging-time constants or runtime host IO. Finally, counts and ordering must be explicit: cppgen sets num ArgIns as arg-ins plus DRAM pointers plus ArgIOs, then separate ArgIO, ArgOut, instrumentation, and early-exit counts (`src/spatial/codegen/cppgen/CppGenAccel.scala:24-29`); ArgAPI emits ArgIns, DRAM pointers, ArgIOs, ArgOuts, counters, and exits in that order (`src/spatial/codegen/cppgen/CppGenInterface.scala:138-164`). Chisel emits the same scalar counts plus stream-info globals for the hardware wrapper (`src/spatial/codegen/chiselgen/ChiselCodegen.scala:196-216`).

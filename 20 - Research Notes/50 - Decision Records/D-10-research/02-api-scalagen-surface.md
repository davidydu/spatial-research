# D-10 / Q-130: API and Scalagen Surface

## 1. Public Staged Surface

The generic Spatial math API exposes `pow`, `abs`, `ceil`, `floor`, `exp`, `ln`, `sqrt`, `sin`, `cos`, `tan`, `sinh`, `cosh`, `tanh`, `asin`, `acos`, `atan`, and `sigmoid` through the `Num` typeclass, with overloads for staged `Sym` and lifted values (`src/spatial/lang/api/MathAPI.scala:28-48`, `src/spatial/lang/api/MathAPI.scala:50-65`). Fixed point implements that same set as `FixPow`, `FixExp`, `FixLn`, `FixSqrt`, the circular and hyperbolic trig nodes, inverse trig nodes, and `FixSigmoid` (`argon/src/argon/lang/Fix.scala:134-147`). Floating point implements the parallel staged set as `FltPow`, `FltExp`, `FltLn`, `FltSqrt`, trig, hyperbolic trig, inverse trig, and `FltSigmoid` (`argon/src/argon/lang/Flt.scala:65-78`). There is no public staged `log2` next to this list: the generic API goes from `exp`/`ln` through `sigmoid`, and the fixed/float type APIs do the same (`src/spatial/lang/api/MathAPI.scala:36-48`, `argon/src/argon/lang/Fix.scala:135-147`, `argon/src/argon/lang/Flt.scala:66-78`).

## 2. IR Nodes and Rewrites

The IR mirrors the public set: fixed nodes are `FixPow`, `FixExp`, `FixLn`, `FixSqrt`, `FixSin`, `FixCos`, `FixTan`, `FixSinh`, `FixCosh`, `FixTanh`, `FixAsin`, `FixAcos`, `FixAtan`, `FixRecip`, `FixRecipSqrt`, and `FixSigmoid` (`argon/src/argon/node/Fix.scala:254-311`). Float nodes are `FltPow`, `FltExp`, `FltLn`, `FltSqrt`, the same trig families, `FltRecip`, `FltRecipSqrt`, and `FltSigmoid` (`argon/src/argon/node/Flt.scala:39-73`). Both `pow` nodes rewrite exponent `0.5` to `sqrt`, `-0.5` to reciprocal square root, and `-1` to reciprocal (`argon/src/argon/node/Fix.scala:260-262`, `argon/src/argon/node/Flt.scala:45-47`). The main fixed-only asymmetry is internal `log2`: fixed multiply/divide strength reductions call `Number.log2` for power-of-two constants, but there is no staged `FixLog2` or `FltLog2` node in the API/node lists above (`argon/src/argon/node/Fix.scala:119-122`, `argon/src/argon/node/Fix.scala:144-145`, `src/spatial/lang/api/MathAPI.scala:36-48`).

## 3. Scalagen Lowering

Scalagen maps fixed values to emulation `FixedPoint` and float values to emulation `FloatPoint` (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:12-15`, `src/spatial/codegen/scalagen/ScalaGenFltPt.scala:12-15`). Fixed constants are emitted with `FixedPoint(BigDecimal(...), FixFormat(...))`, and float constants are emitted with `FloatPoint(..., FltFormat(...))`, including NaN, infinities, and negative zero cases (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:17-23`, `src/spatial/codegen/scalagen/ScalaGenFltPt.scala:17-29`). For transcendental lowering, Scalagen emits `Number.ln`, `Number.exp`, `Number.sqrt`, `Number.sin`, `Number.cos`, `Number.tan`, `Number.sinh`, `Number.cosh`, `Number.tanh`, `Number.asin`, `Number.acos`, `Number.atan`, and `Number.pow` for both fixed and float (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:137-149`, `src/spatial/codegen/scalagen/ScalaGenFltPt.scala:107-119`). `FixRecipSqrt` lowers as `one / Number.sqrt(x)`, while `FltRecipSqrt` lowers directly to `Number.recipSqrt(x)` (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:151`, `src/spatial/codegen/scalagen/ScalaGenFltPt.scala:121`). Both fixed and float sigmoid lower as `one / (Number.exp(-x) + one)` rather than as `Number.sigmoid` (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:152`, `src/spatial/codegen/scalagen/ScalaGenFltPt.scala:122`).

## 4. Runtime and Hardware Contrast

`emul.Number` is explicitly Double-backed for these operations: the file says the fixed and float transcendental blocks "all rely on Double implementation right now," and the implementations call `Math.sqrt`, `Math.exp`, `Math.log`, `Math.pow`, and the trig functions on `x.toDouble` before rebuilding a value in the original format (`emul/src/emul/Number.scala:96-114`, `emul/src/emul/Number.scala:139-155`). `Number.log2` exists for `FixedPoint`, but the float block lists no `FloatPoint` `log2` peer between `recip` and `sigmoid` (`emul/src/emul/Number.scala:102`, `emul/src/emul/Number.scala:140-155`). The Scala executor's generic `NaryResolver` runs `Unary` and `Binary` nodes through each node's `unstaged` function, so these IR nodes inherit the `Number.*` semantics during execution (`src/spatial/executor/scala/resolvers/NaryResolver.scala:30-39`). Hardware is a narrower and partly placeholder surface: Chisel lowers fixed `sqrt`, `sin`, `cos`, `atan`, `sinh`, and `cosh`, but the fixed `Math.sin`/`cos`/`atan`/`sinh`/`cosh` helpers are TODO pass-throughs, and float trig/tanh/sigmoid cases are commented out (`src/spatial/codegen/chiselgen/ChiselGenMath.scala:179-184`, `fringe/src/fringe/templates/math/Math.scala:74-87`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:215-224`).

## 5. Implication for Rust Reference Parity

The Rust numeric reference should match Scalagen/emul `Number` semantics for D-10 parity, not MPFR/exact math and not synthesized hardware behavior, because the Scala-generated CPU path lowers the staged API to `Number.*` and `Number.*` is Double-backed then re-quantized to the source format (`src/spatial/codegen/scalagen/ScalaGenFixPt.scala:137-149`, `src/spatial/codegen/scalagen/ScalaGenFltPt.scala:107-119`, `emul/src/emul/Number.scala:96-114`, `emul/src/emul/Number.scala:139-155`). MPFR or exact per-format transcendental evaluation would be a deliberate new oracle, not a match to existing Scalagen behavior (`emul/src/emul/Number.scala:96-114`, `emul/src/emul/Number.scala:139-155`). Matching synthesized units would under-cover the staged surface because hardware lowering only handles a subset and leaves several floating transcendental cases commented or unconnected (`src/spatial/codegen/chiselgen/ChiselGenMath.scala:207`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:215-224`). The Rust surface should therefore implement the shared `pow/exp/ln/sqrt/trig/hyperbolic/inverse-trig/sigmoid` set, preserve the fixed-only internal `log2` helper distinction, and explicitly document `FixRecipSqrt` and `sigmoid` as Scalagen expression lowerings rather than single primitive calls (`argon/src/argon/lang/Fix.scala:134-147`, `argon/src/argon/lang/Flt.scala:65-78`, `emul/src/emul/Number.scala:102`, `src/spatial/codegen/scalagen/ScalaGenFixPt.scala:151-152`, `src/spatial/codegen/scalagen/ScalaGenFltPt.scala:121-122`).

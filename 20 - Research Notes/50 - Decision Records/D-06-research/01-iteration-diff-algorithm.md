# Iteration-Diff Algorithm

## Decision Context

D-06 asks whether an HLS scheduler should reuse Spatial's `iterDiff`, keep it as diagnostics, or replace it with an HLS-native dependence/schedule model. The source evidence says `IterationDiffAnalyzer` is not a general dependence pass: it is part of Spatial's banking analysis pipeline after retime analysis and access analysis, before memory analysis/allocation (`src/spatial/Spatial.scala:140-143`). Its result is consumed by initiation-interval calculation, memory banking, unrolling, and Chisel reporting (`src/spatial/traversal/InitiationAnalyzer.scala:29-39`, `src/spatial/traversal/banking/MemoryConfigurer.scala:258`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:240-309`, `src/spatial/codegen/chiselgen/ChiselGenMem.scala:73-95`).

## Cycle Finder Scope

The pass visits each `Control`, scans its bodies and blocks, and asks `findAccumCycles(block.stms).accums` for read/write cycles (`src/spatial/traversal/IterationDiffAnalyzer.scala:17-22`, `src/spatial/traversal/IterationDiffAnalyzer.scala:187-210`). `findAccumCycles` groups readers and writers by memory, excludes argument inputs and lock SRAMs, then reports `AccumTriple(mem, rd, wr)` only when read and write are distinct, no retime gate breaks the path, and `rd.independentOf(wr)` is false (`src/spatial/util/modeling.scala:152-179`). It also requires a nonempty dataflow path between the read and write before adding the triple (`src/spatial/util/modeling.scala:183-194`). The iteration-diff logic further narrows this to local memories whose conflicts have been waived with `ignoreAllConflicts` (`src/spatial/traversal/IterationDiffAnalyzer.scala:25-28`). That waiver maps to Spatial memory metadata: `ignoreAllConflicts` means all raw-rank dimensions are in `shouldIgnoreConflicts`, and `.conflictable` sets those flags for SRAMs and regs (`src/spatial/metadata/memory/package.scala:66-68`, `src/spatial/lang/SRAM.scala:112-116`, `src/spatial/lang/Reg.scala:29-33`).

## Tick Model

For eligible cycles, the algorithm requires both reader and writer to have affine access matrices (`src/spatial/traversal/IterationDiffAnalyzer.scala:28`). Those matrices are produced by `AccessAnalyzer`, which turns address patterns into unrolled `AccessMatrix` metadata (`src/spatial/traversal/AccessAnalyzer.scala:215-228`, `src/spatial/metadata/access/package.scala:104-107`). `AccessExpansion` is where Spatial parallel lanes enter the model: it reads the access iterators, their `ctrParOr1`, then enumerates `multiLoop(ps)` to create one matrix per lane, scaling iterator coefficients by parallelization and adding lane-specific constants (`src/spatial/traversal/AccessExpansion.scala:74-83`, `src/spatial/traversal/AccessExpansion.scala:87-124`). Counter metadata supplies `ctrStart`, `ctrEnd`, `ctrStep`, and `ctrParOr1` (`src/spatial/metadata/control/package.scala:1078-1083`).

The analyzer identifies accumulation iterators by dropping leading block iterators until one appears in the writer matrix keys (`src/spatial/traversal/IterationDiffAnalyzer.scala:31-34`). For each iterator, it computes "personal ticks to exhaust" from static `Expect(start/end/step)` divided by parallel lanes, and a stride of `step * par`; unknown bounds either become a huge value under `--looseIterDiffs` or user II, or conservatively become `1` (`src/spatial/traversal/IterationDiffAnalyzer.scala:34-58`, `src/spatial/Spatial.scala:440-442`). Global ticks are the product of inner iterators' personal ticks (`src/spatial/traversal/IterationDiffAnalyzer.scala:60-68`).

## Overlap and Metadata Writes

For each write/read matrix pairing, `ticksToCoverDist` subtracts read from write, finds which iterators contribute to each access dimension, picks the innermost contributing iterator as the gap-closing iterator, and converts distance to ticks by `dist * ticks / stride` (`src/spatial/traversal/IterationDiffAnalyzer.scala:76-87`). It also detects an innermost iterator absent from the read dimension and uses that as an invariant-repeat period (`src/spatial/traversal/IterationDiffAnalyzer.scala:79-90`). The repeat distance is the smaller nonzero value between distance closure and invariant repetition (`src/spatial/traversal/IterationDiffAnalyzer.scala:90-100`). It evaluates four lane "corners" - first/last writer against first/last reader - and keeps the smallest positive overlap tick, or zero if all are zero (`src/spatial/traversal/IterationDiffAnalyzer.scala:103-113`). That value is written to `reader.iterDiff`, `writer.iterDiff`, and `mem.iterDiff`, taking the minimum when an existing nonzero value is already present (`src/spatial/traversal/IterationDiffAnalyzer.scala:115-117`). Special cases override this: `OpMemReduce` marks accumulator/load/store as `iterDiff = 0`, while `OpReduce` marks them as `1` (`src/spatial/traversal/IterationDiffAnalyzer.scala:193-204`).

## Segment Mapping and HLS Fit

If there are multiple reader lanes, the pass builds a lane-to-segment map for intra-iteration dependencies. Lane 0 starts in segment 0; later reader lanes search earlier-or-equal writer lanes for identical collapsed address differences and advance one segment beyond the dependency, preserving any prior segment assignment (`src/spatial/traversal/IterationDiffAnalyzer.scala:119-168`). `SegmentMapping` is explicit metadata for "later lanes of an unrolled read depend on earlier lanes of the unrolled write" (`src/spatial/metadata/memory/AccumulatorData.scala:89-93`). Consumers treat segments as scheduling and banking facts: `AccessMatrix.segmentAssignment` reads the map, banking only conflicts accesses in the same segment, `pipeLatencies` pushes segmented reads after prior-segment writers using a hard-coded SRAM load latency, and unrolling fractures vector accesses by segment (`src/spatial/metadata/access/AffineData.scala:126-130`, `src/spatial/traversal/banking/MemoryConfigurer.scala:327-354`, `src/spatial/util/modeling.scala:433-453`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:240-309`).

The Spatial-specific assumptions are strong: local `LocalMem` classification, user conflict waivers, Spatial counter-chain/parallel-lane metadata, affine matrices that encode unrolled lanes, `Expect` static bounds, retime gates, SRAM timing constants, and Chisel-oriented memory/codegen consumers (`src/spatial/metadata/memory/package.scala:335-346`, `src/spatial/metadata/access/AffineData.scala:133-141`, `src/spatial/util/modeling.scala:401-453`). For D-06, exact reuse would require preserving those contracts in HLS; otherwise `iterDiff` is safer as a diagnostic input while the scheduler uses an HLS-native dependence and resource schedule model (unverified).

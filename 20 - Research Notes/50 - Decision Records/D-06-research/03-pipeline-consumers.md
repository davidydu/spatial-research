Special reduce forms seed known `iterDiff` values of 0 for mem-reduce accumulator cycles and 1 for ordinary reductions (`/Users/david/Documents/David_code/spatial/src/spatial/traversal/IterationDiffAnalyzer.scala:195`, `/Users/david/Documents/David_code/spatial/src/spatial/traversal/IterationDiffAnalyzer.scala:202`). Outer controls use child `II`s and set `compilerII` equal to that interval (`/Users/david/Documents/David_code/spatial/src/spatial/traversal/InitiationAnalyzer.scala:17`, `/Users/david/Documents/David_code/spatial/src/spatial/traversal/InitiationAnalyzer.scala:20`). ## Scheduling Consumers

The direct scheduling path is real: `iterDiff` changes `compilerII`, and `compilerII` can change actual `II` unless user schedule/II overrides apply (`/Users/david/Documents/David_code/spatial/src/spatial/traversal/InitiationAnalyzer.scala:37`, `/Users/david/Documents/David_code/spatial/src/spatial/traversal/InitiationAnalyzer.scala:39`). `segmentMapping` also feeds scheduling indirectly: `latencyAndInterval` sums WAR-cycle lengths across segmented memories and returns the larger interval, while path scheduling pushes segmented readers after prior-segment writers (`/Users/david/Documents/David_code/spatial/src/spatial/util/modeling.scala:118`, `/Users/david/Documents/David_code/spatial/src/spatial/util/modeling.scala:123`, `/Users/david/Documents/David_code/spatial/src/spatial/util/modeling.scala:433`, `/Users/david/Documents/David_code/spatial/src/spatial/util/modeling.scala:445`). Thus Spatial is not merely reporting iteration-diff; it uses it to set controller cadence. Whether an HLS scheduler should reuse this exact model is a D-06 policy question (unverified).

Downstream scheduling consumers read `II` and `bodyLatency`, not `iterDiff` directly. Chisel controller generation passes latency into the state-machine module and creates an `IICounter` from `II` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenController.scala:337`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenController.scala:357`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenController.scala:358`). Stream pressure logic delays FIFO/merge/blackbox pressure by `sym.s.get.II - 1` (`/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:252`, `/Users/david/Documents/David_code/spatial/src/spatial/codegen/chiselgen/ChiselGenCommon.scala:270`). The Scala executor gates inner loop pipeline enqueue every `ctrl.II` cycles (`/Users/david/Documents/David_code/spatial/src/spatial/executor/scala/ControlExecutor.scala:307`, `/Users/david/Documents/David_code/spatial/src/spatial/executor/scala/ControlExecutor.scala:330`; `/Users/david/Documents/David_code/spatial/src/spatial/executor/scala/ControlExecutor.scala:795`, `/Users/david/Documents/David_code/spatial/src/spatial/executor/scala/ControlExecutor.scala:819`).

## Segmentation And Unrolling Consumers

Segmentation has a separate unrolling/banking life. Banking maps each affine unroll lane to `segmentAssignment`, then treats conflicts within the same segment differently from conflicts across segments (`/Users/david/Documents/David_code/spatial/src/spatial/metadata/access/AffineData.scala:126`, `/Users/david/Documents/David_code/spatial/src/spatial/metadata/access/AffineData.scala:130`; `/Users/david/Documents/David_code/spatial/src/spatial/traversal/banking/MemoryConfigurer.scala:353`, `/Users/david/Documents/David_code/spatial/src/spatial/traversal/banking/MemoryConfigurer.scala:354`). If bank-grouping sees positive segment mappings on an unrolled access, it errors and asks to disable `--bank-groupUnroll` (`/Users/david/Documents/David_code/spatial/src/spatial/traversal/banking/MemoryConfigurer.scala:370`, `/Users/david/Documents/David_code/spatial/src/spatial/traversal/banking/MemoryConfigurer.scala:372`).

`MemoryUnrolling` fractures a mapped access into per-segment banked accesses, rewrites port/castgroup/broadcast metadata per segment, writes each new access as `Map(0 -> segment)`, and copies `iterDiff` forward (`/Users/david/Documents/David_code/spatial/src/spatial/transform/unrolling/MemoryUnrolling.scala:240`, `/Users/david/Documents/David_code/spatial/src/spatial/transform/unrolling/MemoryUnrolling.scala:251`, `/Users/david/Documents/David_code/spatial/src/spatial/transform/unrolling/MemoryUnrolling.scala:286`, `/Users/david/Documents/David_code/spatial/src/spatial/transform/unrolling/MemoryUnrolling.scala:301`, `/Users/david/Documents/David_code/spatial/src/spatial/transform/unrolling/MemoryUnrolling.scala:309`). Full mem-reduce unrolling can explicitly remove both segment and iter-diff metadata for accumulator/load/store paths (`/Users/david/Documents/David_code/spatial/src/spatial/transform/unrolling/MemReduceUnrolling.scala:298`, `/Users/david/Documents/David_code/spatial/src/spatial/transform/unrolling/MemReduceUnrolling.scala:301`, `/Users/david/Documents/David_code/spatial/src/spatial/transform/unrolling/MemReduceUnrolling.scala:307`).

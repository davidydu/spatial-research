**Local scheduler facts.**

`IterationDiffAnalyzer` is not a general dependence model. It walks each controller body, finds accumulation cycles, and then narrows to `AccumTriple(mem, reader, writer)` where the memory is local, reader and writer differ, and the memory has `ignoreAllConflicts` set (`src/spatial/traversal/IterationDiffAnalyzer.scala:17`, `src/spatial/traversal/IterationDiffAnalyzer.scala:21`, `src/spatial/traversal/IterationDiffAnalyzer.scala:26`). It only proceeds when reader and writer both have affine access matrices (`src/spatial/traversal/IterationDiffAnalyzer.scala:28`). The computed value is a lower-bound-ish "ticks before repeat/overlap" scalar: it derives per-iterator exhaustion ticks and strides from static `Expect(start/end/step)` when available, falls back to conservative or warning paths when not, and treats explicit `userII` as a reason the uncertainty is acceptable (`src/spatial/traversal/IterationDiffAnalyzer.scala:37`, `src/spatial/traversal/IterationDiffAnalyzer.scala:45`, `src/spatial/traversal/IterationDiffAnalyzer.scala:49`). It then compares writer/read affine matrices, checks corner pairings, keeps the smallest nonzero overlap distance, and writes the same `iterDiff` onto reader, writer, and memory (`src/spatial/traversal/IterationDiffAnalyzer.scala:76`, `src/spatial/traversal/IterationDiffAnalyzer.scala:103`, `src/spatial/traversal/IterationDiffAnalyzer.scala:111`, `src/spatial/traversal/IterationDiffAnalyzer.scala:115`).

**How Spatial consumes it.**

`InitiationAnalyzer` treats `iterDiff` as an input to Spatial's own II estimate. For an inner control it computes block latency and interval, includes blackbox II, collects all statement-level `iterDiff`s, uses nonpositive diffs to force II 1, otherwise divides interval by the minimum positive diff when the diff is greater than 1, and finally lets `userII` override the computed value unless the schedule is explicitly `Sequenced` (`src/spatial/traversal/InitiationAnalyzer.scala:23`, `src/spatial/traversal/InitiationAnalyzer.scala:27`, `src/spatial/traversal/InitiationAnalyzer.scala:29`, `src/spatial/traversal/InitiationAnalyzer.scala:37`, `src/spatial/traversal/InitiationAnalyzer.scala:38`). This means `iterDiff` has already lost the dependence edge identity by the time II is set: it is a collapsed scalar over statements, not a recurrence graph. Spatial also has a separate `compilerII` metadata slot, while user-specified II is distinct user metadata (`src/spatial/metadata/control/ControlData.scala:162`, `src/spatial/metadata/control/ControlData.scala:170`; `src/spatial/metadata/control/package.scala:832`, `src/spatial/metadata/control/package.scala:835`, `src/spatial/metadata/control/package.scala:838`).

**What HLS scheduling needs.**

An HLS backend should carry a schedule contract, not just a Spatial II scalar. The useful fields are: per-loop requested II, Spatial-estimated II, per-edge dependence type, source/sink ops, memory, affine address relation, distance vector, latency, lane/segment relation, and whether the dependence is asserted, waived, or tool-discovered. Specific HLS pragma spellings and report schemas are tool-specific (unverified), but the backend must distinguish "requested II" from "backend accepted II" because HLS may legally return a larger achieved II due to recurrences, resource conflicts, memory ports, or conservative alias/dependence assumptions (unverified). Spatial already exposes the user request through `CtrlOpt.ii`, which sets `x.userII`, and existing reporting can highlight when `compilerII` differs from the chosen `II` (`src/spatial/lang/control/CtrlOpt.scala:8`, `src/spatial/lang/control/CtrlOpt.scala:21`; `src/spatial/codegen/treegen/TreeGen.scala:126`, `src/spatial/codegen/treegen/TreeGen.scala:129`). The HLS path should extend that split to `requested_ii`, `spatial_estimated_ii`, `emitted_ii_pragma`, and `hls_achieved_ii`.

**Partitioning and dependence waivers.**

Array partitioning is part of the schedule model, not a side annotation. Spatial banking already reads affine matrices from all readers/writers, groups accesses by concurrent-port and address-conflict tests, and treats segment assignments as conflict-relevant (`src/spatial/traversal/banking/MemoryConfigurer.scala:75`, `src/spatial/traversal/banking/MemoryConfigurer.scala:78`, `src/spatial/traversal/banking/MemoryConfigurer.scala:252`, `src/spatial/traversal/banking/MemoryConfigurer.scala:258`). It also rejects some grouping when segment mappings imply inter-iteration dependency (`src/spatial/traversal/banking/MemoryConfigurer.scala:370`). `IterationDiffAnalyzer` creates those segment mappings for inter-lane conflicts by mapping later lanes to later segments when a lane reads an address written by an earlier lane, then writes that mapping to reader, writer, and memory (`src/spatial/traversal/IterationDiffAnalyzer.scala:154`, `src/spatial/traversal/IterationDiffAnalyzer.scala:157`, `src/spatial/traversal/IterationDiffAnalyzer.scala:166`). HLS array partition/reshape pragmas change apparent banks and ports (unverified), so the backend must model partition factors and dimensions before deciding whether a recurrence is real, port-limited, or waived.

Dependence pragmas are especially risky. Spatial's `.conflictable`-style waiver becomes `shouldIgnoreConflicts`, and `ignoreAllConflicts` is exactly the predicate that enables this analyzer's accumulation-cycle path (`src/spatial/lang/SRAM.scala:54`, `src/spatial/lang/SRAM.scala:116`; `src/spatial/metadata/memory/package.scala:66`, `src/spatial/metadata/memory/package.scala:68`; `src/spatial/traversal/IterationDiffAnalyzer.scala:26`). HLS dependence pragmas can suppress tool-inferred hazards (unverified); therefore they should only be emitted from explicit user/IR waiver metadata, with a diagnostic showing the local affine evidence and the backend report result. `iterDiff` can help explain why Spatial expected an II, but it should not by itself authorize a dependence pragma.

**Decision posture.**

For D-06, replace `iterDiff` as the HLS scheduler authority with an HLS-facing dependence/schedule model. Reuse the current `IterationDiffAnalyzer` facts diagnostically: seed recurrence-distance hints, display "Spatial expected II" notes, and flag mismatches between Spatial's `compilerII` and HLS achieved II. Do not directly reuse collapsed `iterDiff` to drive HLS scheduling, because it is scoped to waived local accumulation conflicts, collapses edge detail, and predates backend effects such as partitioning, resource binding, and report feedback (`src/spatial/traversal/IterationDiffAnalyzer.scala:26`, `src/spatial/traversal/IterationDiffAnalyzer.scala:115`; `src/spatial/traversal/InitiationAnalyzer.scala:37`, `src/spatial/traversal/InitiationAnalyzer.scala:40`).

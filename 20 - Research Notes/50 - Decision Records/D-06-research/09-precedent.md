# Precedent Survey: Recurrence Distance and II Selection

## 1. Local Problem Shape

D-06 is not deciding whether recurrence distance matters; Spatial already uses it to choose initiation interval. `IterationDiffAnalyzer` scans each controller body, finds accumulation cycles, then narrows to local memories with distinct reader/writer nodes and `ignoreAllConflicts` (`src/spatial/traversal/IterationDiffAnalyzer.scala:17`, `src/spatial/traversal/IterationDiffAnalyzer.scala:21`, `src/spatial/traversal/IterationDiffAnalyzer.scala:26`). It only proceeds when both sides have affine matrices (`src/spatial/traversal/IterationDiffAnalyzer.scala:28`). The computed value is a "ticks before repeat/overlap" scalar: static counter bounds become exhaustion ticks and strides, unknown bounds either warn, rely on explicit user II, or fall back conservatively (`src/spatial/traversal/IterationDiffAnalyzer.scala:31`, `src/spatial/traversal/IterationDiffAnalyzer.scala:37`, `src/spatial/traversal/IterationDiffAnalyzer.scala:45`, `src/spatial/traversal/IterationDiffAnalyzer.scala:49`). The pass then compares write/read affine matrices, checks first/last lane corner pairings, takes the smallest positive overlap tick, and writes that number to reader, writer, and memory `iterDiff` metadata (`src/spatial/traversal/IterationDiffAnalyzer.scala:76`, `src/spatial/traversal/IterationDiffAnalyzer.scala:103`, `src/spatial/traversal/IterationDiffAnalyzer.scala:111`, `src/spatial/traversal/IterationDiffAnalyzer.scala:115`).

## 2. Recurrence Distance Precedent

The general HLS precedent is to express loop pipelining as a constrained scheduling problem in which dependence/recurrence distance limits the minimum legal II (unverified). That resembles Spatial's intent but not its representation. Spatial does not retain a recurrence graph or per-edge distance in the II pass; `InitiationAnalyzer` merely collects statement-level `iterDiff`s, finds the minimum positive diff, treats nonpositive diff as a force-II-1 special case, and computes `ceil(interval / minIterDiff)` when the diff exceeds one (`src/spatial/traversal/InitiationAnalyzer.scala:29`, `src/spatial/traversal/InitiationAnalyzer.scala:30`, `src/spatial/traversal/InitiationAnalyzer.scala:31`, `src/spatial/traversal/InitiationAnalyzer.scala:37`). The local interval being divided already comes from Spatial's own latency/cycle model (`src/spatial/traversal/InitiationAnalyzer.scala:25`, `src/spatial/traversal/InitiationAnalyzer.scala:27`; `src/spatial/util/modeling.scala:112`, `src/spatial/util/modeling.scala:116`). In a DSL-to-HLS flow, the closest precedent is therefore not "reuse the scalar"; it is "preserve dependence evidence until backend scheduling/reporting can explain the accepted II" (unverified).

## 3. User II and Requested II

HLS tools commonly expose a user-requested pipeline II pragma or directive, while the tool may still achieve a larger II if dependences, resources, or memory ports require it (unverified). Spatial already has the same conceptual split. User intent enters through `CtrlOpt.ii`, which stores `x.userII` as user metadata (`src/spatial/lang/control/CtrlOpt.scala:8`, `src/spatial/lang/control/CtrlOpt.scala:11`, `src/spatial/lang/control/CtrlOpt.scala:18`, `src/spatial/lang/control/CtrlOpt.scala:21`). The compiler-owned and user-owned II channels are distinct: `InitiationInterval` and `CompilerII` are analysis metadata, while `UserII` is `SetBy.User` (`src/spatial/metadata/control/ControlData.scala:153`, `src/spatial/metadata/control/ControlData.scala:168`, `src/spatial/metadata/control/ControlData.scala:177`). The getters preserve that distinction as `II`, `userII`, and `compilerII` (`src/spatial/metadata/control/package.scala:832`, `src/spatial/metadata/control/package.scala:835`, `src/spatial/metadata/control/package.scala:838`). For HLS, this supports a four-way record: requested II, Spatial estimated/compiler II, emitted HLS directive, and HLS achieved II (unverified).

## 4. Backend Reports and Diagnostics

Backend precedent also favors report feedback, not silent trust in frontend estimates (unverified). Spatial already reports II mismatches in its tree view: it renders `II`, compares `compilerII` against chosen `II`, and highlights `CompilerII` when they differ (`src/spatial/codegen/treegen/TreeGen.scala:126`, `src/spatial/codegen/treegen/TreeGen.scala:128`, `src/spatial/codegen/treegen/TreeGen.scala:129`, `src/spatial/codegen/treegen/TreeGen.scala:130`). The Chisel backend then emits an `IICounter` from the chosen controller `II`, defaulting to one for outer controls or disabled retiming (`src/spatial/codegen/chiselgen/ChiselGenController.scala:337`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:338`, `src/spatial/codegen/chiselgen/ChiselGenController.scala:358`). HLS schedule reports usually explain failed requested II through recurrence, memory port, or resource bottlenecks (unverified). The equivalent Spatial-HLS diagnostic should attach local evidence: affine matrices, `iterDiff`, memory/segment metadata, requested II, emitted pragma, and achieved II.

## 5. D-06 Implication

General precedent points away from making `iterDiff` the HLS scheduler authority (unverified). Spatial's value is narrowed by local-memory eligibility, conflict waivers, static-bound fallbacks, lane-corner heuristics, and a scalar writeback that loses edge identity (`src/spatial/traversal/IterationDiffAnalyzer.scala:26`, `src/spatial/traversal/IterationDiffAnalyzer.scala:37`, `src/spatial/traversal/IterationDiffAnalyzer.scala:70`, `src/spatial/traversal/IterationDiffAnalyzer.scala:107`, `src/spatial/traversal/IterationDiffAnalyzer.scala:115`). Its consumers also treat segment mapping and conflicts as Spatial-specific scheduling/banking facts (`src/spatial/traversal/IterationDiffAnalyzer.scala:154`, `src/spatial/traversal/IterationDiffAnalyzer.scala:166`; `src/spatial/traversal/banking/MemoryConfigurer.scala:252`, `src/spatial/traversal/banking/MemoryConfigurer.scala:258`). The strongest D-06 posture is therefore: replace `iterDiff` as the HLS scheduling model, but reuse it diagnostically to seed recurrence-distance explanations and to flag differences between Spatial `compilerII` and backend achieved II (`src/spatial/traversal/InitiationAnalyzer.scala:37`, `src/spatial/traversal/InitiationAnalyzer.scala:40`).

## Math object surface.

`Math` declares itself as a common arithmetic API whose implementations should live in BigIP, although the object also contains wrappers and inline operators (`templates/math/Math.scala:9-13`). Scalar signatures are `log2(UInt, Option[Double], Bool, String): UInt`, `mul/div/mod(UInt|SInt, UInt|SInt, Option[Double], Bool, String): UInt|SInt`, and `singleCycleDivide/singleCycleModulo(UInt|SInt, UInt|SInt): UInt|SInt` (`templates/math/Math.scala:16-55`). Fixed signatures include `sqrt(FixedPoint, Option[Double], Bool, String): FixedPoint`, placeholder `sin/cos/atan/sinh/cosh(FixedPoint, Option[Double], String): FixedPoint`, `floor/ceil(FixedPoint, String): FixedPoint`, `add/sub/mul/div/mod(FixedPoint, FixedPoint, Option[Double], Bool, RoundingMode, OverflowMode, String): FixedPoint`, bitwise/shift/compare helpers, `neg`, `fix2flt`, `fix2fix` overloads, `frand`, `fixrand`, `min/max`, and `fma(FixedPoint, FixedPoint, FixedPoint, Option[Double], Bool, String): FixedPoint` (`templates/math/Math.scala:59-88`, `templates/math/Math.scala:105-543`, `templates/math/Math.scala:680-817`, `templates/math/Math.scala:829-868`). Floating signatures include `fadd/fsub/fmul/fdiv(FloatingPoint, FloatingPoint, Option[Double], Bool, String): FloatingPoint`, `flt/flte/feql/fneq(...): Bool`, unary `fabs/fsqrt/exp/tanh/sigmoid/ln/recip/recip_sqrt`, `accum(FloatingPoint, Bool, Bool, Option[Double], Bool, String): FloatingPoint`, `fma(FloatingPoint, FloatingPoint, FloatingPoint, Option[Double], Bool, String): FloatingPoint`, and `flt2fix/flt2flt` conversions (`templates/math/Math.scala:548-701`, `templates/math/Math.scala:775-799`). Direct callers use these through `FloatingPoint` arithmetic/comparison/conversion operators, `FixedPoint` arithmetic/bitwise/comparison/conversion operators, memory accumulator `fma`/`mul`, and memory bank/address modulo/divide helpers (`templates/math/FloatingPoint.scala:47-83`, `templates/math/FixedPoint.scala:52-158`, `templates/memory/Accum.scala:86-175`, `templates/memory/MemPrimitives.scala:411-425`, `templates/memory/MemPrimitives.scala:875-917`).

## Dispatch protocol.

`BigIP` accepts operands plus `latency`, `flow`, and `myName`, returns plain `UInt`/`SInt`/`Bool` values, and models unsupported default methods by throwing `Unimplemented` (`BigIP.scala:6-8`, `BigIP.scala:15-23`, `BigIP.scala:31-43`, `BigIP.scala:45-104`). Integer `log2`, `mul`, `div`, and `mod` route to `globals.bigIP.log2`, `.multiply`, `.divide`, and `.mod`; default fixed latencies come from `globals.target.fix*` times operand width except `log2`, which defaults to zero (`templates/math/Math.scala:19-49`). Fixed `sqrt` delegates integer roots to `bigIP.sqrt`; the odd-fraction branch computes `denSqrt` but then divides by `(1.U << (num.f/2))`, so the intended use of `denSqrt` is ambiguous (`templates/math/Math.scala:59-72`). Fixed `mul`, `div`, and `mod` are wrapper modules that upcast/downcast and call integer `Math.mul`/`Math.div`/`Math.mod`, while `add`, `sub`, bitwise ops, shifts, comparisons, `neg`, `min`, and `max` are implemented inline with Chisel operations and/or `getRetimed` (`templates/math/Math.scala:118-157`, `templates/math/Math.scala:159-373`, `templates/math/Math.scala:375-543`, `templates/math/Math.scala:829-841`). Floating methods map to BigIP as `fadd/fsub/fmul/fdiv`, `flt/fle/feq/fne`, `fabs/fsqrt/fexp/ftanh/fsigmoid/fln/frec/frsqrt/fltaccum/ffma`, and `fix2flt/fix2fix/flt2fix/flt2flt` (`templates/math/Math.scala:548-701`, `templates/math/Math.scala:703-799`). Fixed `fma` is not a direct BigIP op; it multiplies with `Math.mul` and retimes the addend before adding (`templates/math/Math.scala:843-868`).

## Caller assumptions.

Callers assume result type mirrors surface type: `FloatingPoint` operators return `FloatingPoint` or `Bool` and use `None` latency plus `true.B` flow, while `FixedPoint` operators return `FixedPoint`/`Bool` and also default to `None` plus `true.B` (`templates/math/FloatingPoint.scala:47-83`, `templates/math/FixedPoint.scala:61-158`). Conversion constructors assign only `.r` from `Math.fix2flt`/`Math.fix2fix`, so they assume usable raw bits rather than a result/status bundle (`templates/math/FloatingPoint.scala:99-112`, `templates/math/FixedPoint.scala:172-178`). Accumulators treat latency as a scheduling contract: write enables and resets are retimed by `fmaLatency` or `opLatency` alongside the `Math.fma`/`Math.mul` result (`templates/memory/Accum.scala:61-94`, `templates/memory/Accum.scala:123-159`). The retiming helper uses `flow` only as the retime wrapper advance input and returns the same data type, so callers have no separate valid channel from `flow` alone (`utils/package.scala:88-97`). `MemPrimitives` calls `singleCycleModulo` and `singleCycleDivide` directly in bank/address arithmetic, assuming immediate Chisel `%`/`/` behavior rather than BigIP latency (`templates/memory/MemPrimitives.scala:411-425`, `templates/memory/MemPrimitives.scala:875-917`).

## Implications for D-01.

A data-plane "not supported" signal is harder under this dispatch contract than placeholders, because Math and BigIP return plain `UInt`/`SInt`/`Bool`/wrapped numeric values with no validity or support field, and callers immediately assign `.r` or use the result in `Mux`/address logic (`BigIP.scala:15-23`, `BigIP.scala:33-104`, `templates/math/FloatingPoint.scala:47-83`, `templates/memory/Accum.scala:147-176`). Elaboration-time rejection is easier than a signal because `BigIP` already models unsupported defaults as `throw Unimplemented(...)`, while placeholder behavior exists in `Math` for some functions by returning the input unchanged (`BigIP.scala:6-8`, `BigIP.scala:23-31`, `templates/math/Math.scala:74-88`). If D-01 requires runtime/composable rejection, the current contract would need either a new result type or a sideband API propagated through Math, FixedPoint/FloatingPoint operators, and latency-aware callers (`templates/math/Math.scala:548-701`, `templates/math/FixedPoint.scala:61-158`, `templates/memory/Accum.scala:86-175`).

**Override matrix.**

The base `BigIP` contract makes integer divide/mod/multiply and core FP add/sub/mul/div/compare abstract, while defaults for `sqrt`, `log2`, trig, extended FP, and conversions throw `Unimplemented` (`src/fringe/BigIP.scala:15`, `src/fringe/BigIP.scala:23`, `src/fringe/BigIP.scala:31`, `src/fringe/BigIP.scala:64`, `src/fringe/BigIP.scala:104`).

| BigIP method | targets that override | targets that inherit default |
|---|---|---|
| `divide`, `mod`, `multiply` (`UInt`/`SInt`) | `BigIPZynq`, `BigIPCXP`, `BigIPASIC`, and `BigIPSim` each implement these bodies differently (`targets/zynq/BigIPZynq.scala:12`, `targets/cxp/BigIPCXP.scala:12`, `targets/asic/BigIPASIC.scala:8`, `targets/BigIPSim.scala:13`). | Thin or selected targets inherit one of those paths: AWS/ZedBoard/VCU1525 BigIP classes extend Zynq-style classes (`targets/aws/BigIPAWS.scala:5`, `targets/zedboard/BigIPZedBoard.scala:5`, `targets/vcu1525/BigIPVCU1525.scala:5`), AWS_Sim/VCS/Arria10 extend `BigIPSim` (`targets/aws/BigIPAWS_Sim.scala:10`, `targets/vcs/BigIPVCS.scala:10`, `targets/arria10/BigIPArria10.scala:6`), KCU1500/ZCU select `BigIPZynq` (`targets/kcu1500/KCU1500.scala:11`, `targets/zcu/ZCU.scala:6`), and DE1SoC selects `BigIPSim` (`targets/de1soc/DE1SoC.scala:7`). |
| `sqrt(UInt)` | Zynq/CXP use `SquareRooter`, and Sim uses `SqrtSim` (`targets/zynq/BigIPZynq.scala:90`, `targets/cxp/BigIPCXP.scala:90`, `targets/BigIPSim.scala:77`). | ASIC inherits the base throwing default because `BigIPASIC` ends after core FP comparisons (`targets/asic/BigIPASIC.scala:143`, `src/fringe/BigIP.scala:23`). |
| `log2`, `sin`, `cos`, `atan`, `sinh`, `cosh` | `BigIPSim` overrides `log2` with `Log2Sim` and trig/hyperbolic methods as TODO identity stubs (`targets/BigIPSim.scala:71`, `targets/BigIPSim.scala:82`). | Zynq/CXP/ASIC do not define these methods in their concrete files and therefore use the base throwing defaults (`targets/zynq/BigIPZynq.scala:274`, `targets/cxp/BigIPCXP.scala:274`, `targets/asic/BigIPASIC.scala:143`, `src/fringe/BigIP.scala:25`). |
| Core FP `fadd`/`fsub`/`fmul`/`fdiv` and `flt`/`feq`/`fgt`/`fge`/`fle`/`fne` | Zynq/CXP/ASIC/Sim all implement the abstract methods, but with different IP families (`targets/zynq/BigIPZynq.scala:97`, `targets/cxp/BigIPCXP.scala:97`, `targets/asic/BigIPASIC.scala:75`, `targets/BigIPSim.scala:136`). | Only thin targets inherit their selected parent implementation, as above (`targets/aws/BigIPAWS.scala:5`, `targets/arria10/BigIPArria10.scala:6`). |
| Extended FP and conversions | Zynq/CXP override `fabs`, `fexp`, `fln`, `fsqrt`, `frec`, `frsqrt`, `ffma`, `fix2flt`, `fix2fix`, `flt2fix`, `flt2flt`, and `fltaccum`; Sim overrides `fsqrt`, `fix2fix`, `fix2flt`, and `flt2fix` (`targets/zynq/BigIPZynq.scala:168`, `targets/zynq/BigIPZynq.scala:217`, `targets/cxp/BigIPCXP.scala:168`, `targets/BigIPSim.scala:125`, `targets/BigIPSim.scala:237`). | ASIC inherits the base throwing defaults for these non-core methods (`targets/asic/BigIPASIC.scala:143`, `src/fringe/BigIP.scala:64`, `src/fringe/BigIP.scala:92`). |

Fully shared methods are sparse: `getConst` is the shared concrete helper on the base class, and `ftanh`/`fsigmoid` remain base throwing defaults in the base contract (`src/fringe/BigIP.scala:9`, `src/fringe/BigIP.scala:70`).

**Vendor-IP usage patterns.**

Zynq-family targets instantiate Xilinx integer and FP wrappers in `BigIPZynq`: `Divider` (`targets/zynq/BigIPZynq.scala:15`, `targets/zynq/BigIPZynq.scala:25`), `Modulo` (`targets/zynq/BigIPZynq.scala:35`, `targets/zynq/BigIPZynq.scala:45`), `Multiplier` (`targets/zynq/BigIPZynq.scala:63`, `targets/zynq/BigIPZynq.scala:82`), `SquareRooter` (`targets/zynq/BigIPZynq.scala:91`), FP wrappers (`targets/zynq/BigIPZynq.scala:98`, `targets/zynq/BigIPZynq.scala:162`), and extended/conversion wrappers (`targets/zynq/BigIPZynq.scala:169`, `targets/zynq/BigIPZynq.scala:266`). Those wrappers generate Vivado `div_gen`, `mult_gen`, `cordic`, and `floating_point` IP (`targets/zynq/ZynqBlackBoxes.scala:81`, `targets/zynq/ZynqBlackBoxes.scala:140`, `targets/zynq/ZynqBlackBoxes.scala:187`, `targets/zynq/ZynqBlackBoxes.scala:440`). CXP mirrors the same wrapper pattern and Vivado IP emission (`targets/cxp/BigIPCXP.scala:15`, `targets/cxp/BigIPCXP.scala:266`, `targets/cxp/CXPBlackBoxes.scala:81`, `targets/cxp/CXPBlackBoxes.scala:440`).

ASIC instantiates DesignWare-style integer blackboxes for divider/modulo/multiply through `designware_divmod` and `designware_mult`, and its core FP wrappers still emit Vivado `floating_point` IP for add/sub/mul/div/compare (`targets/asic/ASICBlackBoxes.scala:24`, `targets/asic/ASICBlackBoxes.scala:43`, `targets/asic/ASICBlackBoxes.scala:80`, `targets/asic/ASICBlackBoxes.scala:282`, `targets/asic/ASICBlackBoxes.scala:759`). Sim-family targets use Chisel/HardFloat modules for FP (`MulAddRecFN`, `DivSqrtRecFN_small`, `CompareRecFN`, `INToRecFN`, `RecFNToIN`) and simulation blackboxes for `log2`/integer sqrt (`targets/BigIPSim.scala:126`, `targets/BigIPSim.scala:137`, `targets/BigIPSim.scala:178`, `targets/BigIPSim.scala:276`, `targets/BigIPSim.scala:286`, `targets/BigIPSim.scala:103`, `targets/BigIPSim.scala:116`). Arria10 and DE1SoC expose Avalon interfaces but their BigIP paths are simulation-based, not Altera math IP (`targets/arria10/Arria10Interface.scala:13`, `targets/de1soc/DE1SoCInterface.scala:12`, `targets/arria10/BigIPArria10.scala:6`, `targets/de1soc/DE1SoC.scala:7`).

**Common idioms.**

Most implementations special-case constants with `getConst` before falling back to hardware or retimed operations (`targets/zynq/BigIPZynq.scala:12`, `targets/zynq/BigIPZynq.scala:52`, `targets/cxp/BigIPCXP.scala:12`, `targets/BigIPSim.scala:14`, `targets/asic/BigIPASIC.scala:51`). The Xilinx-style implementations consistently pass `flow` into `aclken` and name generated modules with `suggestName`, while Sim uses `getRetimed(..., flow)` (`targets/zynq/BigIPZynq.scala:15`, `targets/zynq/ZynqBlackBoxes.scala:27`, `targets/BigIPSim.scala:15`). The target layer often reuses a family implementation instead of making a new BigIP class: AWS/ZedBoard/KCU1500/ZCU reuse Zynq behavior, and AWS_Sim/VCS/Arria10/DE1SoC reuse Sim behavior (`targets/aws/BigIPAWS.scala:5`, `targets/zedboard/BigIPZedBoard.scala:5`, `targets/kcu1500/KCU1500.scala:11`, `targets/zcu/ZCU.scala:6`, `targets/aws/BigIPAWS_Sim.scala:10`, `targets/arria10/BigIPArria10.scala:6`, `targets/de1soc/DE1SoC.scala:7`). VCU1525 has a Zynq-style BigIP class, but the active target selects `BigIPSim` (`targets/vcu1525/BigIPVCU1525.scala:5`, `targets/vcu1525/VCU1525.scala:8`). No `zcu/BigIPZCU*.scala` matched the requested glob; the active ZCU target selects `BigIPZynq` instead (`targets/zcu/ZCU.scala:6`).

**Implications for D-01.**

The evidence favors both shared structure and target-family separation for an HLS port. A shared HLS-facing facade matches the stable `BigIP` method contract and existing reuse by KCU1500/ZCU/AWS/ZedBoard (`src/fringe/BigIP.scala:15`, `targets/kcu1500/KCU1500.scala:11`, `targets/zcu/ZCU.scala:6`, `targets/aws/BigIPAWS.scala:5`). Separate backends still look necessary where the bodies bind different IP ecosystems: Zynq/CXP emit Vivado IP, ASIC wraps DesignWare plus Vivado FP, and Sim uses HardFloat/native retiming (`targets/zynq/ZynqBlackBoxes.scala:81`, `targets/asic/ASICBlackBoxes.scala:52`, `targets/BigIPSim.scala:137`). A reasonable D-01 posture is therefore shared HLS semantics with per-family lowering hooks, rather than one target-specific HLS file per board or one monolithic implementation for every target (`targets/zedboard/BigIPZedBoard.scala:5`, `targets/aws/BigIPAWS_Sim.scala:10`).

# 09 - Precedent in DSL/RTL/HLS Designs

## Local Shape: Mux Plus Hidden Group

Spatial's current `priorityDeq` shape already resembles a standard select operation in the value plane, but not in the effect/control plane. The API stages one `FIFOPriorityDeq` per candidate FIFO, then returns one `PriorityMux` over non-empty candidates (`src/spatial/lang/api/PriorityDeqAPI.scala:15`, `src/spatial/lang/api/PriorityDeqAPI.scala:20`). The conditional overload repeats the pattern with side conditions, placing the actual value choice in a `PriorityMux` whose selectors include `!f.isEmpty && c` (`src/spatial/lang/api/PriorityDeqAPI.scala:44`, `src/spatial/lang/api/PriorityDeqAPI.scala:50`). `roundRobinDeq` computes rotated priorities from `(iter + idx) % fifos.size`, suppresses candidates with lower rotated priority, and still emits per-FIFO `FIFOPriorityDeq` nodes plus a final `PriorityMux` (`src/spatial/lang/api/PriorityDeqAPI.scala:71`, `src/spatial/lang/api/PriorityDeqAPI.scala:84`, `src/spatial/lang/api/PriorityDeqAPI.scala:96`, `src/spatial/lang/api/PriorityDeqAPI.scala:101`).

That is good precedent for making the data select explicit, but poor precedent for leaving arbitration identity implicit. `PriorityMux` is a first-class primitive node, and Scala codegen lowers it to ordered `if` / `else if` selection (`src/spatial/node/Mux.scala:39`, `src/spatial/codegen/scalagen/ScalaGenBits.scala:39`, `src/spatial/codegen/scalagen/ScalaGenBits.scala:42`). Chisel codegen likewise lowers it to a Chisel `PriorityMux` (`src/spatial/codegen/chiselgen/ChiselGenMath.scala:230`, `src/spatial/codegen/chiselgen/ChiselGenMath.scala:232`). By contrast, the grouped destructive-read event is not a first-class operation. The only common identity is `prDeqGrp`, assigned from the first FIFO's string hash in each API call (`src/spatial/lang/api/PriorityDeqAPI.scala:17`, `src/spatial/lang/api/PriorityDeqAPI.scala:47`, `src/spatial/lang/api/PriorityDeqAPI.scala:97`).

## Arbiter Precedent

In RTL and hardware DSLs, arbitration is usually represented as an explicit arbiter/scheduler with request, grant, selected data, and backpressure behavior, while mux/select nodes only choose data once the grant is known (unverified). The local Fringe code follows that style. `StreamArbiter` is a named module; it computes `cmdValids`, uses `PriorityEncoder`, holds `cmdIdx`, and comments that it services a full request before switching streams (`fringe/src/fringe/templates/dramarbiter/StreamArbiter.scala:10`, `fringe/src/fringe/templates/dramarbiter/StreamArbiter.scala:17`, `fringe/src/fringe/templates/dramarbiter/StreamArbiter.scala:18`, `fringe/src/fringe/templates/dramarbiter/StreamArbiter.scala:19`, `fringe/src/fringe/templates/dramarbiter/StreamArbiter.scala:21`). It then gates each app's `cmd.ready` by the decoded selected stream and routes responses by decoded stream id (`fringe/src/fringe/templates/dramarbiter/StreamArbiter.scala:60`, `fringe/src/fringe/templates/dramarbiter/StreamArbiter.scala:64`, `fringe/src/fringe/templates/dramarbiter/StreamArbiter.scala:67`).

This is much closer to an explicit grouped-dequeue IR than to hash compatibility. The arbiter has a structural input set, an arbitration policy, and a selected output. The selection key is not reconstructed later from unrelated object names or hashes.

## Handshake and Scheduler Precedent

Handshake channels also argue for structural grouping. Fringe stream bundles use Decoupled command/data interfaces with valid/ready on the channel boundary (`fringe/src/fringe/FringeBundles.scala:41`, `fringe/src/fringe/FringeBundles.scala:42`, `fringe/src/fringe/FringeBundles.scala:50`, `fringe/src/fringe/FringeBundles.scala:52`). Stream controllers propagate those valid/ready relationships explicitly through FIFOs and DRAM-facing streams (`fringe/src/fringe/templates/dramarbiter/StreamController.scala:40`, `fringe/src/fringe/templates/dramarbiter/StreamController.scala:42`, `fringe/src/fringe/templates/dramarbiter/StreamController.scala:43`, `fringe/src/fringe/templates/dramarbiter/StreamController.scala:44`, `fringe/src/fringe/templates/dramarbiter/StreamController.scala:57`, `fringe/src/fringe/templates/dramarbiter/StreamController.scala:59`). General HLS scheduling similarly treats a channel read as an operation with readiness, side effects, and scheduling constraints, not just as a data mux (unverified).

Spatial already needs group-aware scheduling behavior downstream. `PriorityDeqGroup` is documented specifically as the tag for figuring out which priority dequeues are grouped together (`src/spatial/metadata/access/AccessData.scala:37`, `src/spatial/metadata/access/AccessData.scala:43`). Memory unrolling lowers scalar `FIFOPriorityDeq` to `FIFOBankedPriorityDeq` and copies the group tag (`src/spatial/transform/unrolling/MemoryUnrolling.scala:303`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:304`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:523`, `src/spatial/transform/unrolling/MemoryUnrolling.scala:524`). Control metadata then excludes banked priority dequeues from ordinary read streams and groups inbound FIFOs by `prDeqGrp.get` (`src/spatial/metadata/control/package.scala:1360`, `src/spatial/metadata/control/package.scala:1367`, `src/spatial/metadata/control/package.scala:1374`). Chisel forward pressure uses the semantic contract directly: OR within each priority group, AND across groups (`src/spatial/codegen/chiselgen/ChiselGenCommon.scala:258`, `src/spatial/codegen/chiselgen/ChiselGenCommon.scala:260`, `src/spatial/codegen/chiselgen/ChiselGenCommon.scala:263`).

## Rewrite Implication

Precedent points toward preserving source-level compatibility, not hash-level compatibility. The Rust/HLS rewrite should model priority and round-robin dequeue as an explicit grouped-dequeue operation: candidate FIFOs, candidate enables/conditions, arbitration policy, optional round-robin base, selected value, and per-candidate destructive read effects. It can still lower to per-FIFO dequeue effects plus a priority/select mux when that matches existing scheduling, and it can emit legacy group metadata as a compatibility bridge for old-style passes.

Hash exactness is the wrong center of gravity. The hash is not user-visible API; it is an implementation tag used because the real arbiter group was missing. Known HDL/HLS patterns make arbitration, channel readiness, and scheduling policy explicit (unverified), and Spatial's own backend already consumes the grouping semantically. For D-02, the rewrite should introduce explicit grouped-dequeue IR while preserving the observed semantics: one logical arbitration event, one selected value, OR-within-group forward pressure, codegen-time empty guarding, and distinct groups for distinct source-level arbitration events.
